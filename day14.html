Remotes  ·  The Bridge

What's a Remote Repository?

A remote is a copy of your Git repo hosted on another computer (usually GitHub's servers). It's the bridge between your local work and the outside world.

Local repo

On your machine. You commit here. Only you can see it.

$ git init
$ git add .
$ git commit -m "first commit"

Remote repo (GitHub)

On GitHub's servers. You push/pull from here. Everyone on the team can see it.

github.com/you/project.git

This URL is your remote's address.

2. Link: git remote add

After creating an empty repo on GitHub, connect it to your local repo:

$ git remote add origin git@github.com:your-username/project.git

origin = the conventional name for your primary remote.

Verify it worked:

$ git remote -v
origin  git@github.com:your-username/project.git (fetch)
origin  git@github.com:your-username/project.git (push)
⚠️ Two URL formats

git@github.com:... = SSH (preferred, no password each time).
https://github.com/... = HTTPS (needs token or password). Set up SSH keys first!

3. Upload: git push

Sends your local commits to GitHub. Your code is now backed up and shareable.

$ git push -u origin main
  • -u = --set-upstream. Tells Git: "main on local = main on origin."
  • After -u, future pushes can just be git push.

What happens behind the scenes:

Counting objects: 15, done.
Writing objects: 100% (15/15), 2.1 MiB ...
To github.com:your-username/project.git
   * [new branch]  main -> main

4. Download: git pull

Fetches changes from GitHub and merges them into your local branch. Use this when:

  • A teammate pushed new code
  • You edited on GitHub.com (or another machine)
  • You're starting work on a shared branch
$ git pull
Already up to date.  # nothing new on remote

$ git pull
Updating a1b2c3d..e4f5g6h
Fast-forward
 README.md | 2 ++
 1 file changed, 2 insertions(+)  # got new stuff

git pull = git fetch + git merge. Do this before you start coding on a shared branch.

5. Copy: git clone

Downloads an existing GitHub repo to your machine — sets up the remote automatically.

$ git clone git@github.com:some-user/some-project.git
Cloning into 'some-project'...
remote: Enumerating objects: 42, done.
Receiving objects: 100% (42/42), 1.2 MiB, done.

After clone, you have:

  • The full repo history locally
  • A remote called origin already configured
  • The default branch checked out
💡 Clone vs. Init + Remote Add

Use git clone when the repo exists on GitHub first. Use git init + git remote add when you start locally and then create the GitHub repo.

The Complete Workflow

Putting It All Together

Create a local project and push it to GitHub — start to finish.

# 1. Create a new project locally
$ mkdir my-first-project && cd my-first-project
$ echo "# My First Project" > README.md
$ git init
$ git add README.md
$ git commit -m "initial commit"

# 2. Create a NEW repo on GitHub (no README, no .gitignore)
#    Copy the SSH URL from the GitHub page

$ git remote add origin git@github.com:you/my-first-project.git
$ git push -u origin main

# 3. Verify on GitHub — refresh the page, your code is there!

This is the pattern you'll use for every project, including your co-op projects.

Practice Exercises

Exercise 1 · Push a project

Create a new local repo called co-op-prep with a README.md. Create an empty repo on GitHub with the same name. Connect and push. Verify it shows on GitHub.com.

Exercise 2 · Clone and inspect

Find an open-source project on GitHub that interests you. Clone it. Run git log --oneline -5, git remote -v, and git branch -a. What do each of these tell you?

Exercise 3 · Pull simulation

Partner up (or use a second account/browser): one person edits the README on GitHub.com and commits. The other runs git pull locally. What do you see?

💡 Stuck?

For Ex.1, create the GitHub repo without initializing it with a README or .gitignore — otherwise push will fail. Watch the GitHub UI carefully.

Bonus: SSH Key Setup

Skip password prompts forever. Generate and register an SSH key.

$ ssh-keygen -t ed25519 -C "your_email@example.com"
# Press Enter to accept defaults

$ cat ~/.ssh/id_ed25519.pub
# Copy the full line starting with "ssh-ed25519 ..."

Then add it on GitHub:

  • GitHub.com → Settings → SSH and GPG keys → New SSH key
  • Paste the public key, give it a name (e.g. "My Laptop")

Test it:

$ ssh -T git@github.com
Hi your-username! You've successfully authenticated.
