Concept

What is Express.js?

Express is a minimal, unopinionated web framework for Node.js. It gives you:

  • Routing — map URLs to handler functions
  • Middleware — process requests through a pipeline
  • HTTP helpers — send JSON, files, status codes easily

Think of it as the glue between your JavaScript logic and the internet. Everything you build in Phase III rides on Express.

Walkthrough

Scaffold the Project

Open a terminal and run these commands step by step:

# 1. Create the project folder
mkdir my-first-api && cd my-first-api

# 2. Initialize package.json (accept defaults)
npm init -y

# 3. Install Express
npm install express

# 4. Verify it worked — check package.json for "express"
cat package.json | grep express

✅ After step 4 you should see "express": "^4.21.x" in the dependencies. If not, try npm install express again.

Code

Your First GET Endpoint

Create index.js with this:

const express = require('express');
const app = express();
const PORT = 3000;

app.get('/', (req, res) => {
  res.json({ message: 'Hello, world!' });
});

app.listen(PORT, () => {
  console.log(`Server running at http://localhost:${PORT}`);
});

What's going on?

  • express() creates the app object
  • app.get(path, handler) registers a route for GET requests
  • res.json(data) sends a JSON response with the right headers
  • app.listen() starts the server on a port

Walkthrough

Test Your API

1. Start the server

node index.js

You should see: Server running at http://localhost:3000

2. Open in browser

Go to http://localhost:3000 in your browser. You'll see:

{"message":"Hello, world!"}

3. Test with curl

curl http://localhost:3000

Same JSON, no browser needed.

4. Pretty-print (optional)

curl http://localhost:3000 | jq

If you have jq installed — gives syntax-highlighted output.

⚠️ Keep the server running in one terminal window. Open a second terminal for curl commands.

Code

Add More Routes

Replace index.js with a multi-route API:

const express = require('express');
const app = express();
const PORT = 3000;

// GET / returns a greeting
app.get('/', (req, res) => {
  res.json({ message: 'Welcome to my API!' });
});

// GET /time returns the current time
app.get('/time', (req, res) => {
  res.json({ serverTime: new Date().toISOString() });
});

// GET /hello/:name uses a route parameter
app.get('/hello/:name', (req, res) => {
  res.json({ greeting: `Hello, ${req.params.name}!` });
});

app.listen(PORT, () => {
  console.log(`🚀 Server running at http://localhost:${PORT}`);
});

Route parameters: :name in the path captures whatever is in that position as req.params.name. Try /hello/Alice → {"greeting":"Hello, Alice!"}

Demo

Test the Extended API

With the updated server running, try these in a browser or curl:

# Root route
curl http://localhost:3000
# {"message":"Welcome to my API!"}

# Time endpoint
curl http://localhost:3000/time
# {"serverTime":"2026-06-30T12:00:00.000Z"}

# Route parameter — change "Bob" to your own name
curl http://localhost:3000/hello/Bob
# {"greeting":"Hello, Bob!"}

Each curl should return JSON immediately. If you get connection refused, make sure node index.js is still running.

Practice

Exercises

Try these on your own. Read the hint if you get stuck.

Exercise 1: Random Quote

Add a GET /quote endpoint that returns a random quote from a hardcoded array of 3-5 quotes.

const quotes = ["The only limit is your imagination.", "Code is poetry.", "First, solve the problem."];
app.get('/quote', (req, res) => {
  const i = Math.floor(Math.random() * quotes.length);
  res.json({ quote: quotes[i] });
});

Exercise 2: Route Echo

Add a GET /echo/:word endpoint that returns the word back to the user along with its character count.

app.get('/echo/:word', (req, res) => {
  res.json({ word: req.params.word, length: req.params.word.length });
});

Test: curl http://localhost:3000/echo/express → {"word":"express","length":7}

Exercise 3: Status Quiz

Create GET /status/:code that takes a number and returns its HTTP status description. Support at least 200 (OK), 404 (Not Found), 500 (Internal Server Error). Return an error if the code isn't recognized.

const statusMap = { '200': 'OK', '404': 'Not Found', '500': 'Internal Server Error' };
app.get('/status/:code', (req, res) => {
  const desc = statusMap[req.params.code];
  if (!desc) res.status(404).json({ error: 'Unknown status code' });
  else res.json({ code: +req.params.code, description: desc });
});

Key Takeaways

01 Express turns Node.js into a web server in ~5 lines

02 app.get(path, handler) is your bread and butter for GET routes

03 res.json() sends JSON — always use it, not res.send()

04 Route params (:name) let you build dynamic endpoints
