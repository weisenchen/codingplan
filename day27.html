🌳 What Is the DOM?

The Document Object Model (DOM) is the browser's tree-shaped representation of your HTML. Every tag becomes a node you can read, write, or delete with JavaScript.

HTML Source
<!-- static file -->
<div id="app">
  <h1>Hello</h1>
  <p class="desc">World</p>
</div>
DOM Tree (in memory)
document
 └─ div#app
     ├─ h1       // "Hello"
     └─ p.desc   // "World"
Key insight: document is a global JS object. Everything on the page lives inside it. When you change the DOM, the page immediately re-renders — no refresh needed.

🎯 Selecting Elements

Before you can change an element, you need a reference to it. Two methods cover 95% of real use:

getElementById
// HTML: <h1 id="title">...</h1>
const title = document
  .getElementById("title");

// ✅ returns one element or null
// 🏁 fastest selector — use for IDs

🔑 IDs must be unique on the page. Direct, fast, simple.

querySelector
// HTML: <p class="desc">...</p>
const desc = document
  .querySelector(".desc");

// ✅ any CSS selector: #id, .class, tag
// returns FIRST match or null
// querySelectorAll → NodeList

🔍 Use when you don't have an ID or need CSS selector power.

Rule of thumb: Have an ID? → getElementById. Using a class/tag/attribute? → querySelector.

📝 textContent vs innerHTML

Two ways to read/write content — but they behave very differently:

.textContent ✅ Safe
const el = document
  .getElementById("output");

// Read (strips all HTML tags)
console.log(el.textContent);

// Write (treated as plain text)
el.textContent = "Hello <b>World</b>";
// → shows: "Hello <b>World</b>" literally

🛡️ Escapes HTML — no XSS risk. Always prefer this for user-submitted text.

.innerHTML ⚠️ Powerful
el.innerHTML = "Hello <b>World</b>";
// → shows: "Hello World"

// Read (returns raw HTML string)
console.log(el.innerHTML);

// ⚠️ DANGER with user input:
el.innerHTML = userInput;
// userInput="<script>stealCookies()</script>"
// → XSS attack!
🚨 XSS warning: Never use innerHTML with data from users, URLs, or APIs. If you need HTML (e.g., formatting), sanitize it first or use DOM methods like document.createElement.

🛠️ Walkthrough: Live Preview Editor

Let's build a mini editor — type in a box, see the preview update instantly.

<!-- HTML structure -->
<input id="nameInput"
       placeholder="Your name" />
<h2 id="greeting">Hello, guest</h2>

<!-- JS -->
<script>
  const input = document
    .getElementById("nameInput");
  const greeting = document
    .getElementById("greeting");

  input.addEventListener("input", () => {
    greeting.textContent =
      "Hello, " + input.value || "guest";
  });
</script>
Try it: Copy this into a .html file and open it in a browser. Type in the input — the heading changes instantly. This is DOM manipulation in action!

📋 innerHTML Use Case: Dynamic Lists

When you control the template string completely (no user input), innerHTML is perfect for rendering structured content:

HTML
<ul id="todoList"></ul>
<button id="addBtn">
  Add Task
</button>
JS (safe — no user HTML)
let tasks = ["Learn DOM", "Build project"];
const list = document
  .getElementById("todoList");

function render() {
  list.innerHTML = tasks
    .map(t => `<li>` + t + `</li>`)
    .join("");
}
render(); // initial draw
✅ Safe because: we control the template — tasks array values are plain strings from our own code, not from a user text input. Unsafe: list.innerHTML = userInput.

🔄 Changing Multiple Elements

Use querySelectorAll + a loop to update groups of elements at once:

/* HTML */
<p class="price">$10</p>
<p class="price">$25</p>
<p class="price">$15</p>
// Add discount: -20%
const prices = document
  .querySelectorAll(".price");

prices.forEach(p => {
  const val = parseFloat(
    p.textContent.replace("$", "")
  );
  p.textContent = `$${(val * 0.8).toFixed(2)}`;
});
// → $8.00, $20.00, $12.00
💡 Pattern: Select → iterate → update. Same approach works for adding classes, toggling visibility, or changing styles.

📌 Summary & Next Steps

3 Key Takeaways
1. Select first, then manipulate.
Use getElementById (IDs) or querySelector (any CSS selector) to get a reference.
2. textContent for text, innerHTML only for trusted HTML.
When in doubt, textContent. Never put user input into innerHTML.
3. DOM changes are instant.
No refresh, no render call. Set .textContent and the page updates immediately.
