Why a Virtual Environment?

A virtual environment gives each project its own isolated package space — like a dedicated suitcase for one trip instead of sharing one suitcase for every trip.

  • Keeps dependencies separate between projects
  • Avoids "it works on my machine" problems
  • No sudo needed for pip installs
  • Easy to recreate with requirements.txt

🗄️ Database Schema

Design the table that stores your bookmarks

Database Design

Bookmarks Table

Our bookmarks table needs just 4 columns:

bookmarks
idINTEGER PRIMARY KEYauto-increment
titleTEXT NOT NULLdisplay name
urlTEXT NOT NULLthe link
created_atTEXT DEFAULT CURRENT_TIMESTAMPwhen saved

Create the table with this SQL:

CREATE TABLE IF NOT EXISTS bookmarks ( id INTEGER PRIMARY KEY AUTOINCREMENT, title TEXT NOT NULL, url TEXT NOT NULL, created_at TEXT DEFAULT CURRENT_TIMESTAMP );

We use TEXT for dates — SQLite doesn't have a native date type, but its date functions work on text.

⚙️ Python Backend

HTTP server with CRUD operations — no frameworks needed

Step-by-Step

Database Helper Module

Create database.py — a small module that wraps our SQLite operations.

import sqlite3 import os DB_PATH = os.path.join(os.path.dirname(__file__), "bookmarks.db") def get_connection(): return sqlite3.connect(DB_PATH) def init_db(): conn = get_connection() conn.execute("""CREATE TABLE IF NOT EXISTS bookmarks ( id INTEGER PRIMARY KEY AUTOINCREMENT, title TEXT NOT NULL, url TEXT NOT NULL, created_at TEXT DEFAULT CURRENT_TIMESTAMP )""") conn.commit() conn.close() init_db() # runs on import

✅ init_db() runs when the module loads — the database is always ready.

Step-by-Step

Add, List & Delete Functions

Add these functions to database.py:

def add_bookmark(title, url): conn = get_connection() conn.execute( "INSERT INTO bookmarks (title, url) VALUES (?, ?)", (title, url) ) conn.commit() conn.close() def get_all_bookmarks(): conn = get_connection() rows = conn.execute( "SELECT id, title, url, created_at FROM bookmarks ORDER BY id DESC" ).fetchall() conn.close() return rows
def delete_bookmark(bookmark_id): conn = get_connection() conn.execute( "DELETE FROM bookmarks WHERE id = ?", (bookmark_id,) ) conn.commit() conn.close() def search_bookmarks(term): conn = get_connection() rows = conn.execute( """SELECT id, title, url, created_at FROM bookmarks WHERE title LIKE ? OR url LIKE ? ORDER BY id DESC""", (f"%{term}%", f"%{term}%") ).fetchall() conn.close() return rows

⚠️ ? placeholders prevent SQL injection — never use f-strings in SQL queries!

Step-by-Step

HTTP Server with http.server

Create server.py — a simple HTTP server that serves HTML and handles API calls.

from http.server import HTTPServer, BaseHTTPRequestHandler from urllib.parse import urlparse, parse_qs import json from database import add_bookmark, get_all_bookmarks, delete_bookmark class BookmarkHandler(BaseHTTPRequestHandler): def do_GET(self): path = urlparse(self.path).path if path == "/": self.serve_page() elif path == "/api/bookmarks": self.list_bookmarks() else: self.send_error(404)

Step-by-Step

Server — Serving the Page

The serve_page() method reads and returns our HTML file:

def serve_page(self): self.send_response(200) self.send_header("Content-Type", "text/html") self.end_headers() with open("index.html", "r") as f: self.wfile.write(f.read().encode("utf-8"))
def list_bookmarks(self): bookmarks = get_all_bookmarks() data = [ {"id": b[0], "title": b[1], "url": b[2], "created_at": b[3]} for b in bookmarks ] self.send_response(200) self.send_header("Content-Type", "application/json") self.end_headers() self.wfile.write(json.dumps(data).encode("utf-8"))

Step-by-Step

Server — POST (Add) & DELETE

def do_POST(self): if self.path == "/api/bookmarks": length = int(self.headers.get("Content-Length", 0)) body = json.loads(self.rfile.read(length)) add_bookmark(body["title"], body["url"]) self.send_response(201) self.send_header("Content-Type", "application/json") self.end_headers() self.wfile.write(json.dumps({"status": "ok"}).encode())
def do_DELETE(self): if self.path.startswith("/api/bookmarks/"): bm_id = int(self.path.split("/")[3]) delete_bookmark(bm_id) self.send_response(200) self.send_header("Content-Type", "application/json") self.end_headers() self.wfile.write(json.dumps({"status": "deleted"}).encode())

Step-by-Step

Server — Entry Point

Add the main block to start the server:

if __name__ == "__main__": port = 8000 server = HTTPServer(("localhost", port), BookmarkHandler) print(f"Server running at http://localhost:{port}") server.serve_forever()

Run it with:

# Make sure your venv is activated python3 server.py # → Server running at http://localhost:8000

Step-by-Step

JavaScript — Fetch & Display

Create app.js — handles form submission, loads bookmarks, and deletes them.

async function loadBookmarks() { const res = await fetch("/api/bookmarks"); const bookmarks = await res.json(); const list = document.getElementById("bookmarkList"); list.innerHTML = bookmarks.map(b => `<div class="bm"> <a href="${b.url}" target="_blank">${b.title}</a> <span>${b.created_at} <span class="delete" onclick="deleteBookmark(${b.id})">✕</span></span> </div>`).join(""); }

Note: using innerHTML is safe here because the server controls the data — in production, you'd sanitize.

Step-by-Step

JavaScript — Add & Delete

Save Form Handler
document.getElementById("bookmarkForm") .addEventListener("submit", async (e) => { e.preventDefault(); const title = document .getElementById("title").value; const url = document .getElementById("url").value; await fetch("/api/bookmarks", { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ title, url }) }); document.getElementById("bookmarkForm").reset(); loadBookmarks(); });
Delete Handler
async function deleteBookmark(id) { if (!confirm("Delete this bookmark?")) return; await fetch(`/api/bookmarks/${id}`, { method: "DELETE" }); loadBookmarks(); } // Load on page load loadBookmarks();

Architecture Review

How It All Fits Together

Here's the flow when a user saves a bookmark:

Browser ──POST /api/bookmarks──▶ server.py │ HTML Form ◀──JSON response─── server.py │ calls add_bookmark() │ ▼ database.py ──INSERT INTO──▶ SQLite DB │ calls get_all_bookmarks() │ ▼ JS ◀───────JSON list──────── server.py │ └─ renders HTML into #bookmarkList

Exercise 1

Add a "Tags" Feature

Task

Extend the bookmark schema and form to include optional tags (e.g., "work", "tutorial", "reference").

  • Add a tags TEXT column to the database
  • Add a tags input field to the HTML form
  • Display tags next to each bookmark
  • Include tags in the POST body
💡 Hint: Store tags as comma-separated text. Display them as small badges next to each title.

Exercise 2

Add a Search Input

Task

Add a search bar above the bookmark list that filters results as the user types.

  • Add an <input type="text" id="search"> to the HTML
  • On each keystroke, call the search_bookmarks() function from database.py
  • Add a /api/bookmarks/search?q=term endpoint to the server
  • Update the JS to load filtered results
💡 Hint: Use a debounce (300ms delay) on the search input so you don't spam the server on every keystroke.

Exercise 3 (Challenge)

Add a Visit Counter

Task

Add a visits INTEGER DEFAULT 0 column. Each time a bookmark link is clicked, increment the counter.

  • Add a PATCH /api/bookmarks/:id/visit endpoint
  • Call it from the HTML link's click event (before navigating)
  • Display visit count next to each bookmark
💡 Hint: Use e.preventDefault() → fetch(PATCH) → window.open(url) so the visit is counted before navigating away.
