Concept 1/5

Function Declarations

A function declaration defines a reusable block of code with a name. It's hoisted — you can call it before it appears in the file.

function greet(name) {               // ← declaration
  return `Hello, ${name}!`;      // ← template literal
}

console.log(greet("Alice"));        // → "Hello, Alice!"
console.log(greet("Bob"));          // → "Hello, Bob!"

🔑 Key points: function keyword · return sends a value back · parameters go in () · body in {}

Speaker notes

Function declarations are hoisted — JS moves them to the top at runtime, so you can call them before the line they're written on. Mention this is different from function expressions and arrow functions.

Concept 2/5

Arrow Functions ⇒

A shorter syntax using =>. No function keyword. Great for callbacks and one-liners.

Function Declaration

function double(x) {
  return x * 2;
}

Arrow Function

const double = (x) => x * 2;

// With block body:
const double = (x) => {
  return x * 2;
};

💡 Rules of thumb: One param → parens optional: x => x*2. No params → () => .... Implicit return without {}.

Concept 3/5

Arrays · push, pop, length

Arrays store ordered lists. Three essential methods to manage them:

const tasks = ["study", "code"];

tasks.push("review");              // add to end
// tasks → ["study", "code", "review"]

tasks.pop();                       // remove from end
// tasks → ["study", "code"]

console.log(tasks.length);           // → 2
[
"study"0
"code"1
]

.length = 2 (one more than last index)

push adds → pop removes → both at the end (LIFO pattern)

Concept 4/5

The for Loop

Repeat code a known number of times. Three parts: initializer · condition · increment.

// Count 0 → 4
for (let i = 0; i < 5; i++) {
  console.log(i);
}
// → 0  1  2  3  4

// Loop over array
const fruits = ["🍎", "🍊", "🍋"];
for (let i = 0; i < fruits.length; i++) {
  console.log(fruits[i]);
}
// Anatomy
for (
  let i = 0;    // ① start here
  i < 5;       // ② check condition
  i++           // ③ run after each pass
) {
  // ④ body
}

💡 i++ means i = i + 1. Start at 0, stop before 5 → 5 iterations.

Concept 5/5

The while Loop

Run while a condition is true. Perfect when you don't know the exact number of iterations ahead of time.

// Countdown with while
let count = 5;
while (count > 0) {
  console.log(count);
  count--;              // ← don't forget!
}
// → 5  4  3  2  1

// Keep going until a condition
let num = 1;
while (num < 100) {
  num *= 2;
}
console.log(num);       // → 128

⚠️ Infinite Loop Pitfall

// BAD — never updates condition:
let x = 0;
while (x < 5) {
  console.log("forever");
  // x never changes!
}

// FIX — update the variable:
let x = 0;
while (x < 5) {
  console.log(x);
  x++;  // ✓
}
Putting It Together

Functions + Arrays + Loops

Bring it all together — write functions that operate on arrays using loops.

// Function: calculate total from an array of numbers
function sumArray(numbers) {
  let total = 0;
  for (let i = 0; i < numbers.length; i++) {
    total += numbers[i];
  }
  return total;
}

// Arrow function version
const sumArray = (numbers) => {
  let total = 0;
  for (let n of numbers) {
    total += n;
  }
  return total;
};

console.log(sumArray([5, 10, 15]));  // → 30

💡 for...of

Iterates values, not indices. Cleaner when you don't need the index.

📌 Pattern

Function takes array → loop processes each element → return result.

// Ex 1 — Filter Evens
function filterEvens(arr) {
  const evens = [];
  for (let i = 0; i < arr.length; i++) {
    if (arr[i] % 2 === 0) {
      evens.push(arr[i]);
    }
  }
  return evens;
}

// Ex 2 — Countdown
function countdown(n) {
  while (n >= 0) {
    console.log(n);
    n--;
  }
  console.log("Blast off!");
}
// Ex 3 — Array Reverser
function reverse(arr) {
  const reversed = [];
  for (let i = arr.length - 1; i >= 0; i--) {
    reversed.push(arr[i]);
  }
  return reversed;
}

Summary

Key Takeaways 📝

  • Function declarations use function name() {} and are hoisted
  • Arrow functions use () => {} — concise, no function keyword
  • Arrays — push() adds, pop() removes, length tells size
  • for loop — for (let i=0; i<n; i++) for counted iteration
  • while loop — while (cond) {} — always update the condition!
