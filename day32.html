✏️ UPDATE & DELETE

Modifying and removing data

UPDATE — Changing Existing Rows

UPDATE modifies one or more rows that match a WHERE condition.

UPDATE employees
SET salary = 75000
WHERE id = 42;

Always write a WHERE clause — without it,
UPDATE touches every row in the table!

-- Update multiple columns at once
UPDATE employees
SET salary = 80000,
    title = 'Senior Engineer'
WHERE id = 15;

DELETE — Removing Rows

DELETE removes rows that match a condition.

-- Delete a specific employee
DELETE FROM employees
WHERE id = 99;
⚠️ Danger
DELETE FROM table;
deletes every row. There's no undo in production!
✅ Best Practice
Always SELECT first to preview what your WHERE matches, then convert to DELETE.

📊 Aggregate Functions

COUNT, SUM, AVG

COUNT, SUM, AVG — Summarizing Columns

Aggregate functions combine many rows into one result.

SELECT
  COUNT(*)                  -- total rows
  COUNT(department)       -- non-null values
  SUM(salary)              -- total salary
  AVG(salary)              -- average salary
FROM employees;
COUNT(*) vs COUNT(col)
COUNT(*) = all rows
COUNT(col) = non-NULL values in that column
Useful with WHERE
SELECT COUNT(*) FROM ... WHERE ...

GROUP BY — Aggregating by Category

Groups rows by a column, then runs aggregates per group.

SELECT department,
       COUNT(*)     AS emp_count,
       AVG(salary)  AS avg_salary
FROM employees
GROUP BY department;

Rule: Every column in SELECT must either be
in GROUP BY or wrapped in an aggregate function.

HAVING — Filtering Groups

WHERE filters rows before grouping.
HAVING filters groups after aggregation.

SELECT department,
       AVG(salary) AS avg_salary,
       COUNT(*)   AS headcount
FROM employees
WHERE status = 'active'          -- row filter
GROUP BY department
HAVING COUNT(*) >= 5;             -- group filter

SQL order of execution:
FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY

📈 Sample Dataset Analysis

Putting it all together

🗃️ Walkthrough — Sales Dataset

Let's analyze a product sales table. Run these in SQLite:

CREATE TABLE sales (
    id        INTEGER PRIMARY KEY,
    product   TEXT,
    category  TEXT,
    amount    REAL,
    sale_date TEXT
);

INSERT INTO sales VALUES
  (1,'Widget', 'Tools',  29.99, '2026-06-01'),
  (2,'Gadget', 'Electronics', 149.99, '2026-06-01'),
  (3,'Widget', 'Tools',  29.99, '2026-06-02'),
  (4,'Doohickey', 'Tools', 15.00, '2026-06-02'),
  (5,'Gadget', 'Electronics', 199.99, '2026-06-03'),
  (6,'Gizmo', 'Electronics',  89.99, '2026-06-03'),
  (7,'Widget', 'Tools',  34.99, '2026-06-04'),
  (8,'Doohickey', 'Tools', 20.00, '2026-06-04');

🔍 Analysis Queries

Run these one at a time and observe the results:

-- Q1: Total sales revenue
SELECT SUM(amount) AS total_revenue
FROM sales;

-- Q2: Average sale amount by category
SELECT category,
       AVG(amount) AS avg_sale,
       COUNT(*)     AS num_sales
FROM sales
GROUP BY category;
-- Q3: Products with more than 1 sale
SELECT product, COUNT(*) AS times_sold,
       SUM(amount) AS total
FROM sales
GROUP BY product
HAVING COUNT(*) > 1;

✏️ Modifying the Dataset

Practice UPDATE and DELETE on our sales table:

-- Fix: Gizmo amount was actually 99.99 (fix the typo)
UPDATE sales
SET amount = 99.99
WHERE product = 'Gizmo'
  AND sale_date = '2026-06-03';

-- Remove the duplicate Widget sale (id=1 is the same as id=3)
DELETE FROM sales
WHERE id = 1;

SELECT COUNT(*) FROM sales;

🏋️ Practice Exercises

Try these on your own. Don't peek at the next slide until you've attempted each one!

Exercise 1 — Category Revenue

Write a query that shows category, total_revenue (SUM), and avg_price for each category. Sort by total revenue descending.

Use ORDER BY total_revenue DESC at the end.

Exercise 2 — Top Products

Find products with total revenue over $50. Show product and total_revenue.

Use HAVING SUM(amount) > 50.

Exercise 3 — Price Correction

Widget's price changed. UPDATE all Widget sales to $39.99. Count how many rows were affected.

What does changes() return after an UPDATE?

✅ Exercise Solutions

Solution 1 — Category Revenue

SELECT category,
       SUM(amount) AS total_revenue,
       AVG(amount) AS avg_price
FROM sales
GROUP BY category
ORDER BY total_revenue DESC;

Solution 2 — Top Products

SELECT product, SUM(amount) AS total_revenue
FROM sales
GROUP BY product
HAVING SUM(amount) > 50;

Solution 3 — Price Correction

UPDATE sales
SET amount = 39.99
WHERE product = 'Widget';
SELECT changes();  -- returns number of rows updated
