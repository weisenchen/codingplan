Part 1 — UI Polish

🎨 Styling the Dashboard

Card-Based Dashboard Layout

Transform a flat page into a polished card layout. Each data section gets its own card with subtle shadows and consistent spacing.

Weather Icons & Typography

Use OpenWeatherMap's condition codes to display appropriate icons — emoji work great for a simple project, or use weather-icons CSS for production.

// Map condition codes to emoji/icons function getWeatherIcon(code) { if (code >= 200 && code < 300) return '⛈️'; // Thunderstorm if (code >= 300 && code < 400) return '🌦️'; // Drizzle if (code >= 500 && code < 600) return '🌧️'; // Rain if (code >= 600 && code < 700) return '❄️'; // Snow if (code >= 700 && code < 800) return '🌫️'; // Atmosphere if (code === 800) return '☀️'; // Clear if (code > 800) return '☁️'; // Clouds return '🌡️'; // Unknown } // Large icon display document.querySelector('.weather-icon').textContent = getWeatherIcon(data.weather[0].id); document.querySelector('.weather-icon').style.fontSize = '64px';
💡 Pro tip: The OpenWeatherMap icon API (https://openweathermap.org/img/wn/{icon}@2x.png) also provides official icons. Use data.weather[0].icon for the filename.
Part 2 — Robustness

⚠️ Error States & Edge Cases

Handling Three Types of Failure

A production dashboard must handle these gracefully:

❌ Invalid City

User types "xyzabc" — API returns 404. Show a friendly error message, don't crash.

🔌 Network Error

User loses internet. The fetch() throws a TypeError. Show "Check your connection."

⏳ Loading State

While data is fetching, show a spinner or skeleton. Never show yesterday's data while loading new data.

// Unified fetch wrapper async function fetchWeather(city) { showLoading(); try { const res = await fetch( `https://api.openweathermap.org/data/2.5/weather?q=${city}&appid=${API_KEY}&units=metric` ); if (!res.ok) { if (res.status === 404) throw new Error('City not found.'); if (res.status === 401) throw new Error('Invalid API key.'); throw new Error(`Server error (${res.status})`); } const data = await res.json(); displayWeather(data); hideLoading(); } catch (err) { showError(err.message); hideLoading(); } }

Part 3 — Forecast

📅 5-Day Forecast

OpenWeatherMap 5-Day API

The 5 Day / 3 Hour Forecast endpoint returns data at 3-hour intervals. We need to group by day and find the daily high/low.

// API endpoint (same API key) const FORECAST_URL = 'https://api.openweathermap.org/data/2.5/forecast'; // Response structure (list of 40 entries, 5 days × 8 per day) { list: [ { dt: 1688763600, // Unix timestamp main: { temp: 22.3, temp_min: 18.1, temp_max: 24.7 }, weather: [{ id: 802, description: 'scattered clouds', icon: '03d' }] }, // ... 39 more entries ] }
⚠️ Important: The free tier only provides the 5 Day / 3 Hour endpoint. Group results by date (dt_txt.split(' ')[0]) to get daily summaries.

Grouping Forecast Data by Day

function groupByDay(list) { const days = {}; list.forEach(entry => { const date = entry.dt_txt.split(' ')[0]; // "2026-07-08" if (!days[date]) { days[date] = { date, temps: [], icons: [], descriptions: [] }; } days[date].temps.push(entry.main.temp); days[date].icons.push(entry.weather[0].icon); days[date].descriptions.push(entry.weather[0].description); }); return Object.values(days).map(day => ({ date: day.date, high: Math.max(...day.temps), low: Math.min(...day.temps), icon: day.icons[Math.floor(day.icons.length / 2)], // mid-day icon desc: day.descriptions[Math.floor(day.descriptions.length / 2)], })); } // Usage const dailyForecast = groupByDay(data.list); // 5 objects renderForecast(dailyForecast.slice(0, 5)); // next 5 days

Rendering the Forecast Cards

/* CSS for forecast row */ .forecast-row { display: flex; gap: 12px; overflow-x: auto; padding: 8px 0; } .forecast-day { flex: 1; min-width: 100px; text-align: center; padding: 16px 8px; background: #f8fafc; border-radius: 12px; } .forecast-day .day-name { font-weight: 600; font-size: 14px; margin-bottom: 6px; } .forecast-day .day-icon { font-size: 32px; margin: 6px 0; } .forecast-day .day-temps { font-size: 13px; color: #64748b; } .forecast-day .high { font-weight: 600; } .forecast-day .low { color: #94a3b8; }

Part 4 — Ship It

🚀 Deploying on GitHub Pages

GitHub Pages in 3 Steps

Your static dashboard (HTML + CSS + JS) is perfect for GitHub Pages — free hosting with a public URL.

# 1. Make sure your repo has an index.html at the root # (or in a docs/ folder) $ ls index.html style.css app.js README.md # 2. Push your code $ git add . $ git commit -m "Weather Dashboard v2 — styling, errors, forecast" $ git push origin main # 3. Enable GitHub Pages via Settings # Settings → Pages → Source: Deploy from branch # Branch: main, folder: / (root) # → SAVE. Wait ~2 minutes. # Your site is live at: # https://{username}.github.io/{repo-name}/
💡 Alternative: GitHub Actions auto-deploys on push. Or use npm install -g gh-pages + gh-pages -d . for a CLI workflow.

✏️ Practice Exercises

1. Geolocation Support

Add a "Use My Location" button that uses the Geolocation API (navigator.geolocation.getCurrentPosition) to fetch weather at the user's current coordinates.

💡 Hint: OpenWeatherMap supports ?lat={lat}&lon={lon} as an alternative to ?q={city}.

2. Unit Toggle (°C / °F)

Add a toggle button that switches between metric and imperial units. Re-fetch data with units=metric or units=imperial.

💡 Hint: Store the current unit in a variable. On toggle, call fetchWeather(city) again with the new unit.

3. Local Storage for Last City

Save the user's last searched city to localStorage. On page load, auto-fetch weather for that city so the dashboard is never empty.

💡 Hint: localStorage.setItem('lastCity', city) on each search; localStorage.getItem('lastCity') on DOMContentLoaded.

📌 Key Takeaways

01

Styling matters. A card-based layout with hover effects, weather icons, and consistent spacing turns raw data into a professional dashboard.

02

Handle failures early. Invalid city (404), bad API key (401), and network errors all get separate, user-friendly messages — never a blank screen.

03

Group forecast data by day. The API returns 3-hour blocks; compute daily min/max and pick a representative icon for each day.

04

Deploy is one setting away. GitHub Pages hosts your static dashboard at a public URL. Enable it in Settings → Pages, push, and you're live.
