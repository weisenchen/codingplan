💡 Why Review?

Spaced repetition is the most effective study method. Re-typing code from memory builds procedural memory — the same skill you'll use in interviews and on the job.

Emphasize that this is the most productive kind of practice. Looking at code and thinking "I know this" is passive. Typing it from scratch is active — and that's what builds real skill.

📝 Re-Type Every Script

Days 1–5 — from memory, no peeking

This is the core exercise of today. Each script should be written from scratch in a new file.

💻 Script 1: Bash Basics (Day 1)

Write a bash script that prints system info. No peeking — type it from memory:

#!/bin/bash
# System info script (Day 1)
echo "Hello, \$USER!"
echo "Date: \$(date)"
echo "Uptime: \$(uptime -p)"
echo "Disk usage:"
df -h | grep '^/' | awk '{print \$1, \$5}'
🎯 Recall Checklist

✅ Shebang line   ✅ Variables ($USER)   ✅ Command substitution ($(date))   ✅ echo, df, grep, awk

Learners should open a blank file, write the script, save it, chmod +x it, and run it. If they can't get through it without looking, that's a flag they should review Day 1.

📁 Script 2: Filesystem Explorer (Day 2)

Write a script that explores directories and reports their contents:

#!/bin/bash
# Filesystem explorer (Day 2)
target="\${1:-.}"
echo "Exploring: \$target"
echo "--- Directories ---"
ls -d "\$target"/*/
echo "--- Files ---"
ls -l "\$target" | grep '^-' | awk '{print \$NF, \$5}'
echo "--- Sizes ---"
du -sh "\$target"/*/ 2>/dev/null | sort -rh | head -5
🎯 Key Commands to Recall

ls, cd, pwd, mkdir -p, touch, cp -r, mv, rm -rf, du -sh, sort -rh, pipes (|), redirection (>, >>)

Test: can they navigate /tmp, create nested dirs, move files between them, and clean up? If not, that's the gap to fix.

🐍 Script 3: Python Hello & venv (Day 3)

Write a Python script and set up a virtual environment. Recall every step:

Part A: The script

#!/usr/bin/env python3
# Hello Python (Day 3)
print("Hello, Co-op Prep!")
name = input("What's your name? ")
print(f"Nice to meet you, {name}!")

Part B: venv setup

$ python3 -m venv venv
$ source venv/bin/activate
(venv) $ python hello.py
Hello, Co-op Prep!
(venv) $ deactivate
Make sure they also recall: what is a venv? Why use one? Can they create, activate, install a package, and deactivate?

🔢 Script 4: Strings, Numbers & Input (Day 4)

Write a program that takes two numbers and performs calculations:

#!/usr/bin/env python3
# Calculator (Day 4)
a = float(input("First number: "))
b = float(input("Second number: "))
print(f"Sum:      {a + b}")
print(f"Diff:     {a - b}")
print(f"Product:  {a * b}")
print(f"Quotient: {a / b:.2f}")

# Bonus — string methods
text = "  hello, CO-OP prep!  "
print(text.strip().title())     # "Hello, Co-Op Prep!"
print("$".join(text.split()))  # "hello,$CO-OP$prep!"
🎯 Concepts to Confirm

Types: int, float, str, bool · f-strings · .strip(), .split(), .join(), .title() · input() returns a string · type()

The string methods section is often forgotten. The key insight: input() always returns a string, so you need int() or float() to convert.

🔄 Script 5: Conditionals & Loops (Day 5)

Write FizzBuzz from memory — the ultimate Day 5 capstone:

#!/usr/bin/env python3
# FizzBuzz (Day 5)
for i in range(1, 101):
    if i % 15 == 0:
        print("FizzBuzz")
    elif i % 3 == 0:
        print("Fizz")
    elif i % 5 == 0:
        print("Buzz")
    else:
        print(i)
🎯 Keywords

if, elif, else, for, in, range(), ==, !=, % (modulo)

⚠️ Common Mistake

Order matters! Check divisibility by 15 (both 3 and 5) first, before checking 3 or 5 individually.

FizzBuzz from memory is the gold standard for Day 5 recall. If the learner can write it without looking in under 3 minutes, they've internalized conditionals and loops.

📚 How to Use Your Deck Effectively

The Leitner Method

  1. Go through all cards once
  2. Correct → move to box 2 (review every 2 days)
  3. Wrong → keep in box 1 (review daily)
  4. Mastered → box 3 (review weekly)
📆 Today's Goal

Go through all 32 cards. Get at least 20 into box 2 (i.e., you can recall them correctly). For the 12 you miss, write them out 3 times each.

Active Recall Rules

  • Say the answer out loud before flipping
  • Write the command from memory
  • If you hesitate >5s, mark it wrong
  • Mix the deck — don't memorize order
⚠️ Pitfall

Don't just look at the cards. Reading is passive. You must retrieve — cover the answer and force your brain to recall.

The Leitner system is simple but powerful. If the learner has Anki or another digital tool, great — otherwise paper cards with 3 labeled boxes works perfectly.

🏁 Milestone: CLI Greeter

Your first mini-project — combine everything you've learned

This milestone pulls together bash scripting, Python, conditionals, loops, and user input into one cohesive project.

📋 CLI Greeter — Specification

Build a greeting program that:

🏆 Milestone: CLI Greeter
  1. Ask for the user's name
  2. Ask for their age
  3. Print a personalized greeting
  4. Tell them what birth year they were born in
  5. If they're ≥ 18, print a special adult message
  6. Offer to repeat the greeting 3 times (loop)
🎯 Concepts Covered

input() · int() conversion · f-strings · if/else · for loops · range() · script execution with python3

This is everything from Days 3-5 in one script. It should feel like a natural integration exercise, not a test.

💬 CLI Greeter — Solution

Write this from memory first. Then compare:

#!/usr/bin/env python3
# CLI Greeter — Milestone Project (Day 6)

name = input("What's your name? ")
age = int(input("How old are you? "))
birth_year = 2026 - age

print(f"Hello, {name}! You were born in {birth_year}.")

if age >= 18:
    print("You're an adult — time to own your learning!")
else:
    remaining = 18 - age
    print(f"{remaining} more years until adulthood — start building now!")

print("Let me repeat that for you:")
for i in range(3):
    print(f"  {i+1}. Welcome, {name}!")

💡 Run it: python3 greeter.py

The key design decisions: int() wrapping input(), the if/else branching on age, and the for loop with range(3). Each piece was covered in a previous day.

🔭 CLI Greeter — Extend It

Once the basic version works, add one of these challenges:

📈 Easy

Also print the user's zodiac sign based on their birth month (ask for month as a second input). Use if/elif.

⚙️ Medium

Keep asking "Want another greeting? (y/n)" until the user says "n" — use a while True loop with break.

💪 Hard

Write a bash wrapper script that activates a venv, runs the Python greeter, then deactivates. Chain two scripts together.

💡 Hint

For the bash wrapper: create greet.sh that sets up the venv, runs python3 greeter.py, then calls deactivate. This combines Day 2 (bash scripts) + Day 3 (venv) + Days 4-5 (Python).

These extensions give the learner room to grow. The hard version is especially valuable because it connects bash and Python — a real-world skill.

💭 Key Takeaways

1
Active recall beats passive reading. Re-typing scripts from memory builds real skill. Code is a physical act — your fingers need to learn it too.
2
Flashcards make knowledge stick. Linux commands + Python keywords are the vocabulary of programming. Five minutes of flashcards daily compounds into fluency.
3
Milestones turn exercises into projects. The CLI Greeter is your first integrated project — bash + Python + logic. Every week builds toward Project 1 (CLI Todo List).
4
Gaps are data, not failure. Finding what you don't know is the whole point of review day. Fix one gap today and it won't slow you down tomorrow.
The key insight: review days are not a break — they're when learning actually consolidates. The 20 most important minutes of the week are the ones spent reviewing.
