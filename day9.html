Why Functions?

Without functions

# 30 lines of copy-paste
print("Hello, Alice!")
print("Hello, Bob!")
print("Hello, Charlie!")
...
print("Hello, Diana!")

Change the greeting? Edit every line. Tedious, error-prone.

With functions

def greet(name):
    print(f"Hello, {name}!")

greet("Alice")
greet("Bob")
greet("Charlie")
greet("Diana")

Change one line → all calls update. DRY — Don't Repeat Yourself.

Key idea: A function packages a reusable block of logic. You define it once, call it many times.

def — Defining a Function

Syntax, naming, and the body

Function Definition Syntax

def function_name(parameter1, parameter2, ...):
    # body — indented 4 spaces
    return result

The recipe:

  • def keyword tells Python you're defining a function
  • Name — lowercase with underscores (calculate_average, send_email)
  • Parentheses — list parameters (inputs) here; empty () means no inputs
  • Colon — ends the header, body follows indented
  • Body — the logic, indented consistently (4 spaces)
  • return — sends a value back; no return → returns None
# Simple example
def double(x):
    return x * 2

result = double(5)   # result = 10

Parameters vs Arguments

Parameter

The variable in the definition (the placeholder).

def square(n):
    return n ** 2

n is the parameter — a local name that receives whatever value is passed in.

Argument

The actual value you pass when calling.

result = square(7)

7 is the argument — the real value that gets assigned to n inside the function.

Analogy: Parameters are the slots in a vending machine; arguments are the coins you drop in.

return — Getting Data Out

A function without return returns None. Use return to send a result back to the caller.

def add(a, b):
    result = a + b
    return result

# Without return — prints but gives nothing back
def add_and_print(a, b):
    print(a + b)

x = add(3, 4)           # x = 7
y = add_and_print(3, 4)  # prints 7, but y = None!

Key rule: return exits the function immediately. Anything after it doesn't run.

def early_exit(x):
    if x < 0:
        return 0         # early exit for negatives
    return x * 2        # only runs if x >= 0

Scope — What's Inside Stays Inside

# Global scope
message = "Hello"
count = 0

def update(name):
    # Local scope
    greeting = f"{message}, {name}!" # ❓ can read globals
    count += 1                      # ❌ ERROR! can't modify
    return greeting

Rules of thumb:

  • Variables defined inside a function live only there
  • Variables defined outside (global) can be read inside, but not reassigned (without global)
  • Parameters are local variables too
# Fix: pass as parameter
def update(name, count):
    greeting = f"Hello, {name}!"
    return greeting, count + 1

💡 Pro tip: pass data in via parameters, pass results out via return. Avoid touching globals.

Extract FizzBuzz into Functions

Refactoring a script into clean, testable chunks

Before: Flat FizzBuzz

for n in range(1, 16):
    if n % 15 == 0:
        print("FizzBuzz")
    elif n % 3 == 0:
        print("Fizz")
    elif n % 5 == 0:
        print("Buzz")
    else:
        print(n)

Works fine — but the logic is mixed with the output. Can't test individual parts, can't reuse easily.

After: Functional FizzBuzz

def fizzbuzz_value(n):
    """Return the FizzBuzz string for a single number."""
    if n % 15 == 0:
        return "FizzBuzz"
    if n % 3 == 0:
        return "Fizz"
    if n % 5 == 0:
        return "Buzz"
    return str(n)

def fizzbuzz(limit=15):
    """Print FizzBuzz from 1 to limit."""
    for n in range(1, limit + 1):
        print(fizzbuzz_value(n))

✓ fizzbuzz_value is pure — test it in isolation.
✓ fizzbuzz just orchestrates. Change the limit with one argument.
✓ Each function has one responsibility.

Key Takeaways

1

Functions encapsulate logic — define once with def, call many times. DRY principle.

2

Parameters in, return out — pass data via parameters, send results via return. Don't touch globals.

3

Scope matters — variables inside a function are local. Read globals if needed, but prefer passing as parameters.

4

Refactor into small functions — each function should do one thing well. Extract FizzBuzz into fizzbuzz_value + fizzbuzz.
