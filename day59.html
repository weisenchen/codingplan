📚 The Stack

Last In, First Out (LIFO)

What Is a Stack?

A stack is a linear data structure where elements are added and removed from the same end — the top.

Analogy: A stack of plates. You add a plate on top (push). You take a plate from the top (pop). The plate at the bottom is the last one you'll ever touch.

Key operations:
push(x) — O(1) add to top
pop() — O(1) remove from top
peek() — O(1) view top
is_empty() — O(1) check

7
1
9 ← bottom

When we pop: 3 comes out first.

Implementing a Stack in Python

A stack is trivially built on top of a Python list:

class Stack:
    def __init__(self):
        self._items = []

    def push(self, item):
        self._items.append(item)   # O(1) amortized

    def pop(self):
        return self._items.pop()    # O(1)

    def peek(self):
        if self.is_empty():
            return None
        return self._items[-1]

    def is_empty(self):
        return len(self._items) == 0

    def size(self):
        return len(self._items)

Note: Python's list.append() and list.pop() are both O(1) — a natural stack implementation.

📚 The Queue

First In, First Out (FIFO)

What Is a Queue?

A queue is a linear data structure where elements are added at one end (rear) and removed from the other end (front).

Analogy: A line at a coffee shop. The first person in line gets served first. New people join at the back.

Key operations:
enqueue(x) — O(1) add to rear
dequeue() — O(1) remove from front
front() — O(1) view front
is_empty() — O(1) check

4
2
8
← enqueue

Implementing a Queue in Python

Use collections.deque for O(1) operations at both ends:

from collections import deque

class Queue:
    def __init__(self):
        self._items = deque()

    def enqueue(self, item):
        self._items.append(item)    # O(1) — adds to right (rear)

    def dequeue(self):
        return self._items.popleft() # O(1) — removes from left (front)

    def front(self):
        return self._items[0] if not self.is_empty() else None

    def is_empty(self):
        return len(self._items) == 0

    def size(self):
        return len(self._items)

Why deque? Using a plain list for the front would be O(n) — pop(0) shifts all elements. deque.popleft() avoids that.

⚡ LeetCode #20: Valid Parentheses

A classic stack application

Valid Parentheses — Problem

Given a string s containing just the characters '(', ')', '{', '}', '[', ']', determine if the input string is valid.

A string is valid if:

  1. Open brackets must be closed by the same type of bracket.
  2. Open brackets must be closed in the correct order.
  3. Every close bracket has a corresponding open bracket of the same type.

✅ Valid

()
()[]{}
{[()]}

❌ Invalid

(]
([)]
( (unclosed)

Solution — Stack-Based O(n)

Pattern: Push opens, pop-and-match on closes.

def is_valid(s: str) -> bool:
    matching = {')': '(', '}': '{', ']': '['}
    stack = []

    for ch in s:
        if ch in matching:           # Closing bracket
            top = stack.pop() if stack else '#'
            if top != matching[ch]:
                return False
        else:                        # Opening bracket
            stack.append(ch)

    return not stack                 # All opened == closed?

Trace "{[()]}": push { → push [ → push ( → pop ( ✅ → pop [ ✅ → pop { ✅ → empty stack → True.

Key Takeaways

📌 Stacks = LIFO

Items are added and removed from the same end (top). Python lists make excellent stacks via append() / pop().

📌 Queues = FIFO

Items enter at the rear and leave from the front. Use collections.deque for O(1) enqueue/dequeue.

📌 Stack → bracket matching

Valid Parentheses is the canonical stack problem. Push opens, pop-and-validate on closes. Empty stack at the end means valid.

📌 Two stacks = one queue

The two-stack queue is a classic interview twist. Transfer from input to output reverses the order once, giving FIFO.
