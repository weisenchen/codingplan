Prerequisites

Before starting, make sure you're comfortable with:

  • Basic tree terminology: root, node, parent, child, leaf, subtree
  • Recursion — especially recursive calls that return values up a tree
  • Binary tree: each node has at most two children (left & right)
  • Time complexity big-O notation: O(log n) vs O(n)

If these are fuzzy, review Trees 101 first

BST Property

A Binary Search Tree is a binary tree where every node satisfies:

Invariant

For any node with value val:

  • All nodes in the left subtree have values less than val
  • All nodes in the right subtree have values greater than val

No duplicate values (in the classic definition). This invariant enables binary search on a tree — at each step we eliminate half the tree.

Why Use a BST?

Comparison vs. Arrays & Linked Lists

OperationArrayLinked ListBST
SearchO(log n)*O(n)O(log n)
InsertO(n)O(1)**O(log n)
DeleteO(n)O(1)**O(log n)

* Sorted array, binary search
** Given pointer to predecessor

Real-World Uses

  • Database indexes (B-trees are BST-derived)
  • In-memory sorted maps (std::map, TreeMap)
  • Autocomplete & prefix matching
  • Range queries (find all values between X and Y)

⚠ Caveat

BST guarantees O(log n) only when balanced. A degenerate tree (e.g., inserting sorted data) is O(n). That's why self-balancing trees (AVL, Red-Black) exist.

Search in a BST

Given a root and a target value, return the subtree rooted at the matching node, or None.

Algorithm

# Recursive — mirrors the invariant
def searchBST(root, target):
    if not root or root.val == target:
        return root

    if target < root.val:
        return searchBST(root.left, target)
    else:
        return searchBST(root.right, target)

Each recursive call goes left or right — never both. Runtime: O(h) where h = tree height.

Trace: search for 6

8 → left (6<8) → 3 → right (6>3) → found 6

Insert into a BST

Insert a new value while preserving the BST invariant. The new node is always inserted as a leaf.

Algorithm

def insertIntoBST(root, val):
    if not root:
        return TreeNode(val)

    if val < root.val:
        root.left = insertIntoBST(
            root.left, val)
    else:
        root.right = insertIntoBST(
            root.right, val)

    return root

Key insight: we recurse down the correct branch, then rewire the parent to the (possibly new) child. The root changes only when the tree was empty.

Trace: insert 7

8 → left → 3 → right → 6 → right child = 7

Delete from a BST

Deletion is the trickiest BST operation. There are three cases based on how many children the target node has.

CaseChildrenStrategyExample
10 (leaf)Just remove — parent's pointer becomes NoneDelete 1
21 childReplace node with its only child (the child "moves up")Delete 12 (has left=10)
32 childrenFind inorder successor (smallest in right subtree), copy its value, then recursively delete the successorDelete 8
# Case 3: two children — find inorder successor
def getMin(node):
    while node.left:
        node = node.left
    return node

# Copy successor value, delete successor
# (which will be Case 1 or 2 — leaf or 1 child)
succ = getMin(root.right)
root.val = succ.val
root.right = deleteNode(root.right, succ.val)

LeetCode #700 — Search in a BST

Problem

Given the root of a BST and an integer val, return the subtree rooted at the node with that value, or None if it doesn't exist.

Iterative Solution

class Solution:
    def searchBST(self, root, val):
        cur = root
        while cur and cur.val != val:
            if val < cur.val:
                cur = cur.left
            else:
                cur = cur.right
        return cur

O(h) time, O(1) space.
No call stack overhead.

Recursive Approach

class Solution:
    def searchBST(self, root, val):
        if not root or root.val == val:
            return root
        if val < root.val:
            return self.searchBST(
                root.left, val)
        return self.searchBST(
            root.right, val)

O(h) time, O(h) space for recursion stack. Cleaner but uses memory.

LeetCode #701 — Insert into a BST

Problem

Given the root of a BST and a val to insert, return the root. You may assume val doesn't already exist.

Recursive

class Solution:
    def insertIntoBST(self, root, val):
        if not root:
            return TreeNode(val)
        if val < root.val:
            root.left = self.insertIntoBST(
                root.left, val)
        else:
            root.right = self.insertIntoBST(
                root.right, val)
        return root

Iterative

class Solution:
    def insertIntoBST(self, root, val):
        if not root:
            return TreeNode(val)
        cur = root
        while True:
            if val < cur.val:
                if not cur.left:
                    cur.left = TreeNode(val)
                    break
                cur = cur.left
            else:
                if not cur.right:
                    cur.right = TreeNode(val)
                    break
                cur = cur.right
        return root

Both O(h) time. Iterative is O(1) space; recursive is O(h) call stack. The recursive code is more idiomatic — you'll see it in most solutions.

Bonus: Kth Smallest (inorder DFS)

def kthSmallest(root, k):
    stack = []
    cur = root
    while stack or cur:
        while cur:
            stack.append(cur)
            cur = cur.left
        cur = stack.pop()
        k -= 1
        if k == 0:
            return cur.val
        cur = cur.right

BST Verification: Inorder Traversal

Inorder traversal of a BST visits nodes in sorted ascending order. This is your go-to verification tool.

Traversal

# Recursive inorder
def inorder(root):
    if not root:
        return []
    return (inorder(root.left)
          + [root.val]
          + inorder(root.right))

Validation

# Check sorted property
vals = inorder(tree)
is_bst = all(
  vals[i] < vals[i+1]
  for i in range(
    len(vals) - 1)
)

Used in LeetCode #98 Validate BST. A must-know interview pattern.

Common Pitfalls

❌ Off-by-one comparisons

BST invariant is left < node < right. Using <= instead of < breaks the tree for duplicates.

❌ Forgetting to rewire

In recursive insert: root.left = insertIntoBST(root.left, val). Without the assignment, the tree never changes.

❌ Returning early in insert

Don't return root inside the while loop of iterative insert — you'll lose the new node.

❌ Assuming balanced

Inserting 1,2,3,4,5 yields a skewed tree (like a linked list). Always think about worst-case O(n).

Key Takeaways

1
BST invariant is everything — left < node < right. Every algorithm flows from it.
2
Search and insert are O(h) time, just following one path from root to leaf. Iterative saves space; recursive reads cleaner.
3
Delete has three cases. The two-child case uses the inorder successor — practice this, it's a common interview blind spot.
4
Inorder traversal = sorted output. Your best tool for debugging and validation.
5
BST is the foundation for AVL trees, Red-Black trees, and B-trees — all coming later in Phase IV.
