Binary Search — The Idea

Search a sorted array by repeatedly dividing the search range in half.

Invariant: at every step, if the target exists, it's in [left, right].

  • Pick the midpoint
  • If nums[mid] == target → done
  • If nums[mid] < target → search right half
  • If nums[mid] > target → search left half

O(log n) time · O(1) space

# Visual: search for 7
# [1, 3, 5, 7, 9, 11, 13]
#  L        M         R   → 7 < 9, go left
#  L   M    R            → 7 > 3, go right
#      L,M,R             → found at index 3

Each step halves the search space. On 1M elements → ~20 comparisons.

Walkthrough: Standard Binary Search

def binary_search(nums: list[int], target: int) -> int:
    left, right = 0, len(nums) - 1

    while left <= right:
        mid = left + (right - left) // 2  # avoid overflow

        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            left = mid + 1                # search right half
        else:
            right = mid - 1               # search left half

    return -1                              # not found

Key details:

  • left + (right - left) // 2 prevents integer overflow (idiom in all languages)
  • while left <= right — use <= not <, or you miss single-element arrays
  • Return -1 if not found (LC convention)

Edge Cases & Gotchas

Case: Empty array

nums = [] → right = -1 → loop never runs → returns -1 ✔

Case: Single element

nums = [5], target 5 → left=0, right=0 → left <= right is true → mid=0 → found.

Case: Target not present

Loop narrows to empty range, returns -1. But what if we need the insertion point? Enter #35.

Gotcha: Duplicates

Standard binary search returns any matching index, not necessarily the first or last.

Variant: Search Insert Position

#35 Given a sorted array and a target, return the index or where it would be inserted.

def search_insert(nums: list[int], target: int) -> int:
    left, right = 0, len(nums) - 1

    while left <= right:
        mid = left + (right - left) // 2

        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            left = mid + 1
        else:
            right = mid - 1

    return left  # <-- only change from standard BS

When the loop exits, left is the insertion point. Think about why: right has moved past the last element < target.

Quick Sort — The Idea

Divide and conquer sorting algorithm.

Steps:

  1. Pick a pivot (e.g., last element)
  2. Partition: rearrange so elements < pivot are left, > pivot are right
  3. Recursively sort left and right partitions

Average: O(n log n)
Worst: O(n²) — bad pivot choices
Space: O(log n) — recursion stack 

# Visual: pivot = last element
# [3, 6, 8, 10, 1, 2, 1]
#               ↑ pivot = 1
# After partition:
# [1, 1, 2, 10, 8, 6, 3]
#  ← left side →  pivot  ← right →
#  (already sorted)       (needs sorting)

Walkthrough: Partition Step

def partition(arr: list[int], low: int, high: int) -> int:
    pivot = arr[high]                 # pick last element as pivot
    i = low - 1                       # index of smaller element

    for j in range(low, high):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]  # swap

    arr[i + 1], arr[high] = arr[high], arr[i + 1]  # pivot to correct spot
    return i + 1                          # return pivot index

def quick_sort(arr, low, high):
    if low < high:
        pi = partition(arr, low, high)  # partition index
        quick_sort(arr, low, pi - 1)    # sort left of pivot
        quick_sort(arr, pi + 1, high)   # sort right of pivot

i tracks the boundary: everything before i is ≤ pivot. j scans the unsorted section.

Key Takeaways

  • Binary search cuts the search space in half each iteration — O(log n)
  • Use left + (right - left) // 2 for mid — always
  • Loop condition left <= right — handles single-element arrays
  • Return left from failed search to get the insertion point (#35)
  • Quick sort partitions around a pivot, then recurses on each side
  • Binary search patterns extend to rotated arrays, search spaces, and more
