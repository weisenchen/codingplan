🔗 What is a Linked List?

A linked list is a linear data structure where each element (node) points to the next.

Node anatomy:

class Node:
    def __init__(self, value):
        self.value = value
        self.next = None

A node holds a value and a next pointer to the next node (or None if last).

Array vs Linked List:

PropertyArrayLinked List
AccessO(1) randomO(n) sequential
Insert (front)O(n)O(1)
Delete (front)O(n)O(1)
MemoryContiguous blockScattered (overhead)

Linked lists shine when you frequently insert/delete at the head.

🏗 Building the LinkedList Class

class LinkedList:
    def __init__(self):
        self.head = None

    # --- Traversal ---
    def print_list(self):
        curr = self.head
        while curr:
            print(curr.value, end=" → ")
            curr = curr.next
        print("None")

    # --- Append (tail insert) ---
    def append(self, value):
        new_node = Node(value)
        if not self.head:
            self.head = new_node
            return
        curr = self.head
        while curr.next:
            curr = curr.next
        curr.next = new_node
    # --- Prepend (head insert) ---
    def prepend(self, value):
        new_node = Node(value)
        new_node.next = self.head
        self.head = new_node

    # --- Delete by value ---
    def delete(self, value):
        if not self.head:
            return
        if self.head.value == value:
            self.head = self.head.next
            return
        curr = self.head
        while curr.next and curr.next.value != value:
            curr = curr.next
        if curr.next:
            curr.next = curr.next.next

    # --- Usage ---
ll = LinkedList()
ll.append(10)
ll.append(20)
ll.prepend(5)
ll.print_list()  # 5 → 10 → 20 → None

🧠 Pointer Tracing: Append

Walk through ll.append(30) on list 5 → 10 → 20 → None

# Step 1: Create new node
new_node = Node(30)
# new_node: [30 | None]

# Step 2: Traverse to tail
curr = head               # curr → [5 | ●]
while curr.next:         # [5 | ●] → yes
    curr = curr.next      # curr → [10 | ●]
                          # [10 | ●] → yes
    curr = curr.next      # curr → [20 | None]
                          # [20 | None] → no, stop

# Step 3: Wire the new node
curr.next = new_node
# [20 | ●] → [30 | None]

✅ Key insight:

We stop when curr.next is None, not when curr is None. That preserves the previous node's reference so we can update its .next pointer.

⚠️ Common mistake:

Using while curr: instead of while curr.next:. This makes curr land on None, and you can't set None.next — that's an AttributeError.

🧠 Pointer Tracing: Delete

Walk through ll.delete(10) on list 5 → 10 → 20 → None

# Edge case: deleting the head
if self.head.value == value:
    self.head = self.head.next
    return

# General case: find predecessor
self.head: [5 | ●] → [10 | ●] → [20 | None]
                            ↑ delete this

curr = head
while curr.next.value != value:
    curr = curr.next
# curr stays at [5 | ●] (since 5.next.value == 10)

# Bypass the target
curr.next = curr.next.next
# [5 | ●] → [20 | None]  (10 is now orphaned)

✅ Key insight:

To delete a node, you need a pointer to the node before it. That's why the loop checks curr.next.value, not curr.value.

Edge cases always check:

1. Empty list → head is None
2. Delete head → update self.head
3. Delete tail → loop runs to end, sets last .next to None
4. Value not found → no change

🔄 LeetCode #206: Reverse Linked List (Iterative)

Reverse a singly linked list in-place using 3 pointers.

def reverse_iterative(head):
    prev = None
    curr = head

    while curr:
        # 1. Save next before we overwrite
        next_temp = curr.next

        # 2. Reverse the arrow
        curr.next = prev

        # 3. Slide pointers forward
        prev = curr
        curr = next_temp

    return prev  # new head

Tracing on [1 → 2 → 3 → None]:

# Before loop:
prev=None  curr=[1|●]→[2|●]→[3|None]
# Iter 1:
next_temp=[2|●]  curr.next=None
prev=[1|None]     curr=[2|●]
# Iter 2:
next_temp=[3|None]  curr.next=[1|None]
prev=[2|●]→[1|None]  curr=[3|None]
# Iter 3:
next_temp=None  curr.next=[2|●]→[1|None]
prev=[3|●]→[2|●]→[1|None]  curr=None
# Return prev → [3 → 2 → 1 → None] ✅

🔄 LeetCode #206: Recursive Solution

def reverse_recursive(head):
    # Base: empty list or single node
    if not head or not head.next:
        return head

    # Reverse the rest of the list
    new_head = reverse_recursive(head.next)

    # Make the next node point back to head
    head.next.next = head
    head.next = None

    return new_head

🧠 How it works:

reverse_recursive travels to the tail first (base case). On the way back up the call stack, each node reverses one arrow — the node after it now points back to it.

Key line: head.next.next = head

This says: "whoever I'm pointing to, make them point back at me." It's the core of the reversal.

⏱ Space: O(n) call stack vs O(1) iterative. Prefer iterative in interviews unless asked.

🔍 Exercise Solutions

1. Find Middle

def find_middle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    return slow  # middle (2nd if even)

3. Nth from End

def get_nth_from_end(head, n):
    fast = slow = head
    for _ in range(n):
        if not fast: return None
        fast = fast.next
    while fast:
        slow = slow.next
        fast = fast.next
    return slow.value

2. Has Cycle

def has_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow is fast:
            return True
    return False

4. Merge Two Sorted Lists

def merge_two_lists(l1, l2):
    dummy = Node(0)
    curr = dummy
    while l1 and l2:
        if l1.value < l2.value:
            curr.next = l1; l1 = l1.next
        else:
            curr.next = l2; l2 = l2.next
        curr = curr.next
    curr.next = l1 or l2
    return dummy.next

📌 Key Takeaways

1. Nodes have two parts: value + next (pointer). None marks the end.

2. Append = walk to tail (while curr.next:), then wire. Prepend = point new node to head, update head.

3. Delete needs the predecessor. That's why we check curr.next.value, not curr.value.

4. Reverse = three pointers (prev, curr, next_temp). Iterative is O(n) time, O(1) space.

5. Two-pointer patterns (slow/fast) solve many linked list problems: middle, cycle, nth-from-end.
