Why Python + SQLite?

✅ Zero setup

SQLite ships with Python — no install, no server, no config.

✅ Self-contained

A single file (.db) is the whole database. Email it, commit it, delete it.

✅ Portable

Your script works on any machine with Python — laptops, CI, servers.

Real-world use cases

  • CLI tools that remember state (Todo list, bookmarker)
  • Local analytics — query CSVs dumped into SQLite
  • Prototyping before migrating to Postgres
  • App config / user preferences

The sqlite3 Module

Python's built-in module follows the DB-API 2.0 standard — same pattern used by psycopg2, mysql-connector, and others.

import sqlite3

# Connect (creates file if missing)
conn = sqlite3.connect("books.db")

# Get a cursor
cur = conn.cursor()

# Execute SQL
cur.execute("CREATE TABLE ...")

# Save changes
conn.commit()

# Clean up
cur.close()
conn.close()

The 4-step pattern

  1. 1 connect() — get a connection
  2. 2 cursor() — create a cursor
  3. 3 execute() — run SQL
  4. 4 commit() — persist changes

💡 :memory: instead of a filename creates an in-memory DB.

Walkthrough: Create & Query

Let's build a script that creates a books table, inserts rows, and queries them.

1
Create the DB and table
import sqlite3

conn = sqlite3.connect("library.db")
cur = conn.cursor()

cur.execute("""CREATE TABLE IF NOT EXISTS books (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    author TEXT NOT NULL,
    year INTEGER,
    rating REAL
)""")
2
Insert data
cur.execute("INSERT INTO books (title, author, year, rating) VALUES (?, ?, ?, ?)",
           ("Dune", "Frank Herbert", 1965, 4.7))
cur.execute("INSERT INTO books (title, author, year, rating) VALUES (?, ?, ?, ?)",
           ("Neuromancer", "William Gibson", 1984, 4.5))
conn.commit()
3
Query and fetch
# Fetch all rows
cur.execute("SELECT * FROM books")
rows = cur.fetchall()
for row in rows:
    print(row)

# Fetch one row
cur.execute("SELECT title, rating FROM books WHERE rating > ?",
           (4.5,))
top = cur.fetchone()
print(top)  # ('Dune', 4.7)

Expected output:

(1, 'Dune', 'Frank Herbert', 1965, 4.7)
(2, 'Neuromancer', 'William Gibson', 1984, 4.5)
('Dune', 4.7)

Safety: Parameterized Queries

Never build SQL by concatenating strings — use ? placeholders instead.

❌ Dangerous — SQL injection

name = "Dune"
# Never do this!
cur.execute(f"SELECT * FROM books WHERE title = '{name}'")

A malicious value like '; DROP TABLE books; -- would delete your table.

✅ Safe — parameterized

name = "Dune"
# Always use ? placeholders
cur.execute(
    "SELECT * FROM books WHERE title = ?",
    (name,)
)

The sqlite3 module escapes the value automatically. Tuple with one element needs a trailing comma: (name,).

Robust Patterns

Using with (context manager)

import sqlite3

with sqlite3.connect("library.db") as conn:
    cur = conn.cursor()
    cur.execute("SELECT * FROM books")
    rows = cur.fetchall()
    for row in rows:
        print(row)
# conn closed automatically

The context manager commits on exit unless an exception occurs (then it rolls back).

Error handling

import sqlite3
from sqlite3 import OperationalError

try:
    conn = sqlite3.connect("library.db")
    cur = conn.cursor()
    cur.execute("SELECT * FROM nonexistent")
except OperationalError as e:
    print(f"DB error: {e}")
finally:
    conn.close()

Common exceptions: OperationalError (bad SQL, missing table), IntegrityError (UNIQUE constraint, NOT NULL), ProgrammingError (wrong number of params).

Fetching Results

Three ways to retrieve data after execute().

fetchone()

Returns the next row as a tuple, or None if no more rows.

cur.execute("SELECT * FROM books")
row = cur.fetchone()
print(row)
# (1, 'Dune', 'Frank Herbert', 1965, 4.7)
fetchmany(n)

Returns up to n rows as a list of tuples. Useful for pagination.

cur.execute("SELECT * FROM books")
rows = cur.fetchmany(2)
for row in rows:
    print(row)
# (1, 'Dune', ...)
# (2, 'Neuromancer', ...)
fetchall()

Returns all remaining rows as a list of tuples. Most common.

cur.execute("SELECT * FROM books")
rows = cur.fetchall()
print(f"Found {len(rows)} books")
for row in rows:
    print(row[1])  # just titles

🏗️ Project 2 Preview: Bookmark Saver

After Week 6, you'll build a CLI bookmark manager. Here's a sneak peek at how SQLite powers it:

📌 Key features

  • Add a bookmark (URL + title + tags)
  • List all bookmarks
  • Search by tag or keyword
  • Delete stale bookmarks
  • Data persists across runs in bookmarks.db

🧠 Skills from today you'll use

  • sqlite3.connect() — create/open the DB
  • Parameterized INSERT — safe user input
  • SELECT ... WHERE tag = ? — search
  • DELETE — remove old entries
import sqlite3

def init_db():
    with sqlite3.connect("bookmarks.db") as conn:
        conn.execute("""CREATE TABLE IF NOT EXISTS bookmarks (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            url TEXT NOT NULL UNIQUE,
            title TEXT,
            tags TEXT,
            created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
        )""")

def add_bookmark(url, title, tags):
    with sqlite3.connect("bookmarks.db") as conn:
        conn.execute(
            "INSERT INTO bookmarks (url, title, tags) VALUES (?, ?, ?)",
            (url, title, tags)
        )

def search_by_tag(tag):
    with sqlite3.connect("bookmarks.db") as conn:
        rows = conn.execute(
            "SELECT * FROM bookmarks WHERE tags LIKE ?",
            (f"%{tag}%",)
        ).fetchall()
    return rows

✏️ Exercise: Book Tracker CLI

Build a Python script that manages a reading list using SQLite.

Requirements

  • Create a DB reading.db with a books table: id, title, author, status (unread/reading/done), pages
  • Insert at least 3 books
  • Print all unread books
  • Update one book's status to "reading"
  • Print the count of books per status

💡 Hints

  • Use WHERE status = 'unread' to filter
  • UPDATE books SET status = ? WHERE title = ?
  • Count with SELECT status, COUNT(*) FROM books GROUP BY status
  • Use conn.row_factory = sqlite3.Row for named columns
import sqlite3

conn = sqlite3.connect("reading.db")
conn.row_factory = sqlite3.Row
cur = conn.cursor()

cur.execute("""CREATE TABLE IF NOT EXISTS books (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    author TEXT,
    status TEXT DEFAULT 'unread',
    pages INTEGER
)""")

books = [
    ("The Pragmatic Programmer", "Hunt & Thomas", "unread", 352),
    ("Clean Code", "Robert C. Martin", "reading", 464),
    ("Designing Data-Intensive Applications", "M. Kleppmann", "unread", 616),
]
cur.executemany(
    "INSERT INTO books (title, author, status, pages) VALUES (?, ?, ?, ?)",
    books
)
conn.commit()

print("--- Unread books ---")
cur.execute("SELECT title, author FROM books WHERE status = 'unread'")
for row in cur.fetchall():
    print(f"  - {row['title']} by {row['author']}")

cur.execute(
    "UPDATE books SET status = 'reading' WHERE title = ?",
    ("The Pragmatic Programmer",)
)
conn.commit()

print("\n--- Books by status ---")
cur.execute("SELECT status, COUNT(*) as cnt FROM books GROUP BY status")
for row in cur.fetchall():
    print(f"  {row['status']}: {row['cnt']}")

conn.close()

🎯 Key Takeaways

Three things to write down:

1. The 4-step pattern

connect() → cursor() → execute() → commit(). Every SQLite interaction follows this rhythm.

2. Always parameterize

Use ? placeholders, never f-strings. It's the difference between working code and a dropped table.

3. Context managers are your friend

with sqlite3.connect("db") as conn: auto-commits on success, rolls back on error, closes the connection.
