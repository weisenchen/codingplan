POST — Creating Resources

POST sends data to the server to create a new resource. Unlike GET, it has a body.

📤 Client → Server

POST /api/items
Content-Type: application/json
Body: { "name": "Buy milk" }

📥 Server → Client

Status: 201 Created
Body: { "id": 3, "name": "Buy milk" }

The server parses the JSON body, creates the resource, and returns it with a 201 status.

⏱ 8 min

POST Endpoint — Express

// server.js — POST endpoint
import express from 'express';
const app = express();
const PORT = 3000;

app.use(express.json());  // ← REQUIRED to parse JSON body

let items = [
  { id: 1, name: "Learn REST APIs" },
  { id: 2, name: "Build a project" }
];
let nextId = 3;

app.post('/api/items', (req, res) => {
  const { name } = req.body;
  if (!name) {
    return res.status(400).json(
      { error: "name is required" }
    );
  }
  const item = { id: nextId++, name };
  items.push(item);
  res.status(201).json(item);
});
Key point: express.json() middleware is required before any route that reads req.body.
⏱ 8 min

DELETE — Removing Resources

DELETE removes a resource. The resource ID goes in the URL, and the server responds with confirmation (or 404 if not found).

📤 Client → Server

DELETE /api/items/2
No body needed

📥 Server → Client

200 OK — deleted
or 404 Not Found

// DELETE /api/items/:id
app.delete('/api/items/:id', (req, res) => {
  const id = Number(req.params.id);
  const idx = items.findIndex(i => i.id === id);

  if (idx === -1) {
    return res.status(404).json(
      { error: "Item not found" }
    );
  }

  items.splice(idx, 1);
  res.json({ message: "Deleted" });
});
⏱ 5 min

HTTP Status Codes

Every response includes a 3-digit code. Know these:

200 OK Success

Standard success for GET, PUT, DELETE

201 Created

Resource was created (POST responses)

400 Bad Request

Missing or invalid data in the request

404 Not Found

Resource doesn't exist

409 Conflict

Duplicate or conflict (e.g. duplicate email)

500 Server Error

Something broke on the server (your bug!)

POST from an HTML Form

A simple form that sends JSON to your API:

<!-- index.html -->
<form id="itemForm">
  <input type="text" id="nameInput"
         placeholder="New item name" required>
  <button type="submit">Add Item</button>
</form>
<ul id="itemList"></ul>

<script>
  const form = document.getElementById('itemForm');

  form.addEventListener('submit', async (e) => {
    e.preventDefault();

    const name = document.getElementById('nameInput').value;

    const res = await fetch('/api/items', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ name })
    });

    if (res.ok) {
      const item = await res.json();
      console.log("Created:", item);
      // → append item to the list
    }
  });
</script>

DELETE from the Frontend

// Delete button click handler
async function deleteItem(id) {
  const res = await fetch(`/api/items/${id}`, {
    method: 'DELETE'
  });

  if (res.ok) {
    // Remove the item's DOM element
    document.getElementById(`item-${id}`).remove();
  } else {
    const err = await res.json();
    alert(err.error);
  }
}

// Example: render list items with delete buttons
function renderItems(items) {
  const list = document.getElementById('itemList');
  list.innerHTML = '';
  for (const item of items) {
    const li = document.createElement('li');
    li.id = `item-${item.id}`;
    li.textContent = item.name;
    const btn = document.createElement('button');
    btn.textContent = '✕';
    btn.onclick = () => deleteItem(item.id);
    li.appendChild(btn);
    list.appendChild(li);
  }
}

JSON Request / Response

Three things to always check when debugging:

1️⃣ Headers

Content-Type: application/json
must be set on every request with a body

2️⃣ Stringify

JSON.stringify() converts JS objects to JSON string for the wire

3️⃣ Parse

Server: express.json()
Client: res.json() to parse the response

Pro tip: Open DevTools → Network tab → click a request to see the raw JSON. This is your best debugging tool!
⏱ 8 min

Walkthrough: Full CRUD Flow

  1. Start your server: node server.js → listening on port 3000
  2. GET all items — fetch('/api/items') to see the initial array
  3. POST a new item — send { "name": "Practice daily" }
  4. Verify — GET /api/items again, confirm it's now in the list
  5. DELETE an item — send DELETE with its ID
  6. Verify again — GET the list, confirm it's removed
  7. Try errors — DELETE a non-existent ID → get 404
Expected output: Each step returns a JSON response. Use console.log() to see every response in DevTools.

Common Pitfalls

❌ req.body is undefined

Forgot express.json() middleware. Add it before routes!

❌ Wrong Content-Type

Client must set 'Content-Type': 'application/json' in fetch headers.

❌ Using GET for mutations

Use POST for create, DELETE for delete. GET should be read-only.

❌ Not checking HTTP status

Always check res.ok or res.status — don't assume success!
