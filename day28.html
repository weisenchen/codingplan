What Are Events?

The browser tells your code when something happens.

⚡ The Event-Driven Model

JavaScript in the browser sits idle until the user does something — then it reacts.

Common Events

clickmouse click
inputtext field changes
submitform submitted
keydownkey pressed
mouseoverhover over element
scrollpage/element scrolled

The Analogy

A restaurant: the kitchen doesn't cook until the waiter rings a bell. The bell is the event. The chef is the handler.

// "When the button rings, run this"
button.addEventListener('click', () => {
  console.log('Order up! 🍳');
});

🔌 addEventListener

The standard way to register an event handler.

element.addEventListener('event-type', handlerFunction);

Parts

  • element — the DOM node to listen on
  • 'event-type' — string like 'click'
  • handlerFunction — runs when the event fires

Two ways to write the handler

// Inline arrow function
btn.addEventListener('click', () => {
  alert('Clicked!');
});
// Named function (reusable)
function handleClick() {
  alert('Clicked!');
}
btn.addEventListener('click', handleClick);

🖱️ Click Events

The most common event — fires when a user clicks an element.

// Get the button
const btn = document.querySelector('#myButton');

// Listen for clicks
btn.addEventListener('click', () => {
  console.log('Button was clicked!');
});

⚠️ Common Pitfall

Don't call the function in the listener — that runs it immediately:

// ❌ Wrong — runs handleClick right now
btn.addEventListener('click', handleClick());

// ✅ Correct — passes the function reference
btn.addEventListener('click', handleClick);

⌨️ Input Events

Fires every time the value of an <input>, <textarea>, or <select> changes.

'input' event

Fires on every keystroke — real-time.

const input = document.querySelector('#name');

input.addEventListener('input', (event) => {
  console.log(event.target.value);
  // logs after each character typed
});

'change' event

Fires when the user commits the change (blur or Enter).

input.addEventListener('change', (event) => {
  console.log('Final value:', event.target.value);
  // fires after user clicks away
});

💡 Tip: Use 'input' for live preview (search-as-you-type). Use 'change' when you only care about the final value (form validation after blur).

Walkthrough

Build a button that counts clicks

🛠️ Click Counter — HTML & JS

Create counter.html — follow along:

1. HTML Structure

<!-- counter.html -->
<h1>Click Counter</h1>
<p>You've clicked <span id="count">0</span> times</p>
<button id="clickBtn">Click me</button>

2. Select Elements

const countDisplay = document.querySelector('#count');
const button = document.querySelector('#clickBtn');
let clickCount = 0;

3. Add Event Listener

button.addEventListener('click', () => {
  clickCount++;
  countDisplay.textContent = clickCount;
});

4. Expected Behavior

Every click increments the counter. The <span> updates immediately — no page reload needed.

📄 Click Counter — Full Solution

<!DOCTYPE html>
<html>
<head><title>Click Counter</title></head>
<body>
  <h1>Click Counter</h1>
  <p>Clicked <span id="count">0</span> times</p>
  <button id="clickBtn">Click me</button>

  <script>
    const countDisplay = document.querySelector('#count');
    const button = document.querySelector('#clickBtn');
    let clickCount = 0;

    button.addEventListener('click', () => {
      clickCount++;
      countDisplay.textContent = clickCount;
    });
  </script>
</body></html>
▶

Open in browser — the counter updates live on every click.

🔁 Live Input Preview

Combine 'input' events with click events for a richer interaction.

const nameInput = document.querySelector('#nameInput');
const greeting = document.querySelector('#greeting');

// Update greeting as user types
nameInput.addEventListener('input', (e) => {
  greeting.textContent = `Hello, ${e.target.value}!`;
});

// Bonus: clear button
document.querySelector('#clearBtn').addEventListener('click', () => {
  nameInput.value = '';
  greeting.textContent = 'Hello!';
  nameInput.focus();
});

💡 Key insight: The event object (first parameter to the handler) carries event.target — the element that fired the event. Use it to read .value from inputs.

🎯 Key Takeaways

✓
Events drive interactivity — the browser fires events (click, input, keydown, etc.) and your code responds via handlers.
✓
addEventListener is the standard way to register handlers — pass the event type as a string and a callback function.
✓
The event object carries data — event.target tells you which element fired the event; event.target.value reads input content.
✓
'input' vs 'change' — use 'input' for real-time updates (live preview) and 'change' for final values (form commit).
✓
Never call the handler — pass the function reference, not the result of calling it (btn.addEventListener('click', fn) not fn()).
