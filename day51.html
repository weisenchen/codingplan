📊 Character Frequency Counting

Why it matters

Frequency maps are the bread and butter of string problems. Every time you need to answer "how many of each character?" — anagrams, duplicates, ransom notes, first unique char — reach for a frequency counter.

Two approaches

Manual dict: freq[c] = freq.get(c, 0) + 1

collections.Counter: Counter(s) — one line, same O(n)

Code

# Manual frequency count
def char_freq(s: str) -> dict:
    freq = {}
    # c.isalpha() to skip spaces/punctuation
    for c in s.lower():
        freq[c] = freq.get(c, 0) + 1
    return freq

# One-liner with Counter
from collections import Counter

freq = Counter(s.lower())

💡 Counter is great for interviews — it signals "I know the stdlib" and reduces bugs from manual increment logic.

Walk through the dict.get pattern — this trips up beginners. Show how Counter handles missing keys automatically. Mention that in an interview, starting with Counter is fine, but be ready to implement the dict version if asked.

🔄 Valid Anagram — LeetCode #242

Problem

Given two strings s and t, return True if t is an anagram of s, and False otherwise.

Example 1:

s = "anagram", t = "nagaram" → True

Example 2:

s = "rat", t = "car" → False

Key insight

Two strings are anagrams iff they have the same character counts. Order doesn't matter — only frequency.

Solution

from collections import Counter

def is_anagram(s: str, t: str) -> bool:
    # Early exit: different lengths =
    # can't be anagrams
    if len(s) != len(t):
        return False
    # Compare frequency maps
    return Counter(s) == Counter(t)

# Time: O(n) — one pass each
# Space: O(k) — k = unique chars
#          (at most 26 for lowercase)

Alternative: use a single dict — increment for s, decrement for t, then check all zeros.

Emphasize the early-length check — it's an easy optimization many miss. Show the single-dict variant manually: freq = {}; for c in s: freq[c]=freq.get(c,0)+1; for c in t: freq[c]=freq.get(c,0)-1; then check all values are 0. This is more efficient for interviews than comparing two Counters.

✍️ Walkthrough: #242 Step by Step

Let's trace through is_anagram("anagram", "nagaram")

1

Length check: len("anagram") == len("nagaram") → True

2

Build Counter for s: {'a': 3, 'n': 1, 'g': 1, 'r': 1, 'm': 1}

3

Build Counter for t: {'n': 1, 'a': 3, 'g': 1, 'r': 1, 'm': 1}

4

Compare: Both Counters have same items → True 🎯

Easy O(n) time O(1) space (26 letters)
Walk through each character being added to the counter. Point out that Counter equality checks both keys and values — {'a': 3} != {'a': 2}. For the sorted alternative (return sorted(s) == sorted(t)), mention it's O(n log n) and still passes but interviewers prefer O(n).

🔄 Valid Palindrome — LeetCode #125

Problem

A phrase is a palindrome if, after converting all uppercase letters to lowercase and removing all non-alphanumeric characters, it reads the same forward and backward.

Example 1:

"A man, a plan, a canal: Panama" → True

Example 2:

"race a car" → False

Two-pointer approach

def is_palindrome(s: str) -> bool:
    left, right = 0, len(s) - 1
    while left < right:
        # Skip non-alphanumeric
        while (left < right and
               not s[left].isalnum()):
            left += 1
        while (left < right and
               not s[right].isalnum()):
            right -= 1
        # Compare (case-insensitive)
        if s[left].lower() != s[right].lower():
            return False
        left += 1
        right -= 1
    return True

# Time: O(n) — each char visited once
# Space: O(1) — in-place
The tricky part is skipping non-alphanumeric characters. Many candidates forget the inner while loops entirely. Emphasize isalnum() — it's the key method here. The .lower() on comparison handles case insensitivity. This is a pure two-pointer problem with a filtering twist.

✍️ Walkthrough: #125 Step by Step

Tracing "A man, a plan, a canal: Panama"

1

left = 0 ('A'), right = 29 ('a') — both alphanumeric

2

'A'.lower() == 'a'.lower() → match, move: left=1, right=28

3

left=1 (' ') not alnum → skip to left=2 ('m')
right=28 ('m') alnum → match, move: left=3, right=27

4

Continue skipping commas, spaces... each pair matches. Pointers meet in the middle → True ✅

⚠️ Common pitfall: Forgetting the inner boundary check left < right in skip loops can cause index out of bounds on strings with only non-alnum characters!

Walk through the pointer movements slowly. Use your hands — left hand on 'A', right hand on 'a'. Show how ' ' and ',' get skipped. The boundary check in the inner while loops is the most common bug — without it, "!!!" would crash.

🔁 Palindrome: Clean-Then-Compare Approach

When to use this

If the interviewer says "space doesn't matter" or you want the simplest readable solution:

Pro: Dead simple to read and debug

Con: O(n) extra space for the cleaned string

Verdict: Fine for interviews if you mention the tradeoff

Comparison

Two-pointerClean+Reverse
TimeO(n)O(n)
SpaceO(1)O(n)
ReadabilityMediumHigh

Code

def is_palindrome_clean(s: str) -> bool:
    # Build a cleaned string: lowercase
    # alphanumeric only
    cleaned = [c.lower() for c in s
               if c.isalnum()]
    # Compare with its reverse
    return cleaned == cleaned[::-1]

# Equivalent one-liner (less readable):
# return (c:=[c.lower() for c in s
#         if c.isalnum()]) == c[::-1]

# Both work — pick what you can
# explain clearly in an interview
This is worth showing because many candidates reach for the clean-then-reverse approach first. Validate it — it's correct — but then show the two-pointer optimization. Interviewers love when you say "We could do this with O(n) space, but let's optimize to O(1) with two pointers."

Exercise 1: First Unique Character

Given a string s, find the first non-repeating character and return its index. If none exists, return -1.

Example: s = "loveleetcode" → 2 (character 'v')

💡 Hint: Two passes — first pass builds frequency, second pass finds first char with count 1.

from collections import Counter

def first_uniq_char(s: str) -> int:
    freq = Counter(s)
    for i, c in enumerate(s):
        if freq[c] == 1:
            return i
    return -1

# Time: O(n), Space: O(1) — 26 letters

Exercise 2: Palindrome Permutation

Given a string s, return True if a permutation of the string could form a palindrome.

Example: "carerac" → True (can form "racecar")

💡 Hint: A palindrome can have at most one character with odd frequency.

from collections import Counter

def can_permute_palindrome(s: str) -> bool:
    freq = Counter(s)
    odd_count = sum(
        count % 2
        for count in freq.values()
    )
    return odd_count <= 1

# Time: O(n), Space: O(1)
Give learners 5-7 minutes to try each exercise. Encourage them to verbalize their approach before coding. For exercise 2, the insight is that palindromes need paired characters — only one character can be left unpaired (the middle character in odd-length strings).

📌 Key Takeaways

1. Frequency maps are your Swiss Army knife — master Counter and the manual dict.get pattern. They solve anagrams, duplicates, first unique char, and more.

2. Anagrams = equal character counts — length check first, then compare frequencies. O(n) time beats O(n log n) sorting.

3. Two-pointer for palindrome — O(1) space, O(n) time. Remember .isalnum() and .lower(). Don't forget inner boundary checks.

4. Know your tradeoffs — clean-then-reverse is simpler but costs O(n) space. Two-pointer is trickier but optimal. Pick based on what the interviewer asks.

These four takeaways are the core. Ask the audience to write them down. Mention that frequency maps, two-pointer, and string sanitization are patterns that appear in Medium problems too — they're foundational.
