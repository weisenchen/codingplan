What is useEffect?

Synchronising your component with the outside world

useEffect Basics

useEffect lets you run side effects in function components — things that happen outside React's render cycle.

Common use cases

  • 🌐 Data fetching (API calls)
  • 📋 Subscriptions (WebSocket, events)
  • ⏱ Timers (setTimeout, setInterval)
  • 🖱 DOM manipulations

Signature

useEffect(() => {
  // side effect code
}, [dependencies]);

Takes a callback and an optional dependency array.

When Does It Run?

🟢 No array

useEffect(() => {
  // runs on EVERY render
});

After every mount and update. Rarely what you want.

🔵 Empty array

useEffect(() => {
  // runs ONCE on mount
}, []);

Runs after first render only. Perfect for initial data fetch.

🟡 With deps

useEffect(() => {
  // runs when 'id' changes
}, [id]);

Runs on mount AND whenever a dependency changes value.

Data Fetching with useEffect

The standard pattern: three states for any async operation.

✅ Loading

Show a spinner or skeleton while data loads.

✅ Data

Render the fetched content when it arrives.

❌ Error

Catch errors and show a friendly message.

const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);

useEffect(() => {
  fetch('https://api.example.com/items')
    .then(res => {
      if (!res.ok) throw new Error('Fetch failed');
      return res.json();
    })
    .then(data => setData(data))
    .catch(err => setError(err.message))
    .finally(() => setLoading(false));
}, []);

async/await Inside useEffect

You can't make the effect callback itself async — but you can define an async function inside.

useEffect(() => {
  const fetchData = async () => {
    try {
      setLoading(true);
      const res = await fetch(`https://api.example.com/users`);
      if (!res.ok) throw new Error(`HTTP ${res.status}`);
      const json = await res.json();
      setData(json);
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  };
  fetchData();
}, []);

// Render based on state
if (loading) return <Spinner />;
if (error) return <p>Error: {error}</p>;
return <UserList users={data} />;

Cleanup: Avoid Memory Leaks

When a component unmounts before a fetch completes, calling setState causes a warning. Return a cleanup function to abort.

useEffect(() => {
  let cancelled = false;

  const fetchData = async () => {
    const res = await fetch(url);
    const json = await res.json();
    if (!cancelled) setData(json);
  };
  fetchData();

  return () => { cancelled = true; };
}, [url]);

Why clean up?

  • User navigates away → component unmounts
  • Fetch completes → tries setState on unmounted component
  • React warns: "Can't perform a React state update on an unmounted component"
  • Cleanup flag prevents the update

Common Pitfalls

🚫 Missing deps

useEffect(() => {
  fetch(`/api/${userId}`);
}, []); // userId changed? Ignored!

The effect never re-runs when userId changes. Add it to the array.

🚫 Infinite loop

useEffect(() => {
  setCount(count + 1);
}, [count]); // re-render → re-run → re-render...

Don't set state that's also a dependency. Causes a render loop.

🚫 Stale closure

useEffect(() => {
  const timer = setInterval(() => {
    console.log(count); // always 0
  }, 1000);
}, []);

The effect "captures" the initial count. Use refs or include in deps.

Walkthrough: Weather Dashboard

Let's fetch weather data and display loading, error, and results.

function Weather({ city }) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    let cancelled = false;
    setLoading(true);
    setError(null);

    fetch(`https://api.weather.gov/...`)
      .then(r => r.json())
      .then(d => { if (!cancelled) setData(d); })
      .catch(e => { if (!cancelled) setError(e.message); })
      .finally(() => { if (!cancelled) setLoading(false); });

    return () => { cancelled = true; };
  }, [city]);

  if (loading) return <p>Loading...</p>;
  if (error) return <p>⚠ {error}</p>;
  return <p>{data.temp}°C, {data.condition}</p>;
}
  1. States: data, loading, error
  2. Effect: fetches when city changes
  3. Cleanup: cancelled flag prevents stale updates
  4. Reset: set loading/error before fetching
  5. Render: check loading → error → data

⏱ This pattern is the foundation for every data-fetching component you'll write in React.
