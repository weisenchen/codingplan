1 · Lists

Ordered & Unordered Lists

<ul> Unordered (bullets)

<ul>
  <li>Apples</li>
  <li>Bananas</li>
  <li>Cherries</li>
</ul>
• Apples
• Bananas
• Cherries

<ol> Ordered (numbered)

<ol>
  <li>Preheat oven</li>
  <li>Mix ingredients</li>
  <li>Bake 30 min</li>
</ol>
1. Preheat oven
2. Mix ingredients
3. Bake 30 min

💡 Nest lists by putting a <ul> or <ol> inside an <li>.

1 · Lists (cont.)

Nested Lists & Attributes

<ol type="A">                        <!-- A, B, C... -->
  <li>Section One
    <ul style="list-style: square;">  <!-- nested -->
      <li>Sub-item A</li>
      <li>Sub-item B</li>
    </ul>
  </li>
  <li>Section Two</li>
</ol>

Useful <ol> attributes:

AttributeExampleResult
typetype="I"I, II, III
startstart="5"5, 6, 7
reversedreversed3, 2, 1

2 · Tables

Structuring Tabular Data

<table>
  <thead>
    <tr>
      <th>Name</th>
      <th>Role</th>
      <th>Exp</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Alice</td>
      <td>Dev</td>
      <td>4y</td>
    </tr>
    <tr>
      <td>Bob</td>
      <td>PM</td>
      <td>6y</td>
    </tr>
  </tbody>
</table>

Renders as:

NameRoleExp
AliceDev4y
BobPM6y

🔑 <thead> = header, <tbody> = body, <tfoot> = footer. Use colspan to merge cells.

2 · Tables (cont.)

Spanning Cells

<table>
  <tr>
    <th colspan="2">Name</th>
    <th>Score</th>
  </tr>
  <tr>
    <td>First</td>
    <td>Last</td>
    <td>95</td>
  </tr>
</table>
NameScore
FirstLast95

colspan="2" makes the header span 2 columns.
rowspan="N" spans N rows.

3 · Forms

Inputs, Labels & Buttons

<form action="/submit">
  <label for="name">Name:</label>
  <input type="text"
         id="name"
         name="name"
         required>

  <label for="email">Email:</label>
  <input type="email"
         id="email"
         name="email">

  <button type="submit">
    Sign Up
  </button>
</form>

Key points:

  • Always pair <label> with for → input id
  • type="text", type="email", type="password", type="number"
  • required validates client-side
  • placeholder shows hint text
  • <button type="submit"> submits the form

⚠ Always set type="submit" on submit buttons — default is type="submit" anyway, but being explicit avoids confusion with type="reset" or type="button".

3 · Forms (cont.)

Select, Textarea & Fieldset

<label for="bio">Bio:</label>
<textarea id="bio" rows="4"></textarea>

<label for="country">Country:</label>
<select id="country">
  <option>Canada</option>
  <option>USA</option>
</select>

<fieldset>
  <legend>Preferences</legend>
  <label>
    <input type="checkbox">
    Subscribe
  </label>
</fieldset>
  • <textarea> = multi-line text input
  • <select> + <option> = dropdown
  • <fieldset> groups related fields
  • <legend> labels the group
  • <input type="checkbox"> / type="radio"

💡 Wrap an input inside a <label> to make the label clickable without needing for/id.

4 · Semantic HTML

Structure That Means Something

<body>
  <header>
    <nav>
      <a href="/">Home</a>
    </nav>
  </header>

  <main>
    <article>
      <h1>Post Title</h1>
      <p>Content...</p>
    </article>
    <aside>Related links</aside>
  </main>

  <footer>
    <p>© 2026</p>
  </footer>
</body>

Why semantics matter:

  • ✅ Screen readers navigate by landmark
  • ✅ SEO — search engines understand structure
  • ✅ Cleaner, more maintainable code
  • ✅ Future-proof — CSS/JS targets elements naturally
TagPurpose
<header>Intro / nav bar
<nav>Navigation links
<main>Primary content (1 per page)
<article>Self-contained content
<section>Thematic grouping
<aside>Sidebar / tangential info
<footer>Footer / copyright

  </footer>
</body>
