📂 File Basics

open() + with = the foundation of all file work

The open() Function

open(filename, mode) returns a file object. The mode tells Python what you intend to do.

Common Modes

'r'read (default)
'w'write (overwrites!)
'a'append
'r+'read + write
'x'create, fail if exists

Without with (old way)

f = open('data.txt', 'r')
content = f.read()
f.close()  # easy to forget!

⚠️ Forgetting close() leaks file handles.

The with Block — Context Manager

Python's with statement automatically closes the file, even if an error occurs.

# The modern, safe way
with open('data.txt', 'r') as f:
    content = f.read()
# File is closed here — no close() needed!
💡 Mental Model

with open(...) as f: = "open this file, call it f inside this block, and close it when we're done — no matter what."

✅ Good: auto-close, readable, Pythonic

❌ Bad: manual close(), easy to forget, risk of corruption

📖 Walkthrough: Reading Files

Three common ways to read. Create a file sample.txt with a few lines of text first.

# 1. Read entire file at once
with open('data.txt', 'r') as f:
    text = f.read()
# 2. Read into a list of lines
with open('data.txt', 'r') as f:
    lines = f.readlines()  # each line + \n as a list element
# 3. Iterate line by line (memory efficient for big files)
with open('data.txt', 'r') as f:
    for line in f:
        print(line.strip())  # strip removes trailing \n
🔑 Key Insight

For small files, .read() is fine. For large files (GB of logs), iterate with for line in f — it reads one line at a time.

✍️ Walkthrough: Writing Files

Use 'w' (overwrite) or 'a' (append).

# Write mode — creates or overwrites
with open('output.txt', 'w') as f:
    f.write('Hello, World!\n')
    f.write('Line 2\n')
# Append mode — adds to end
with open('output.txt', 'a') as f:
    f.write('This goes at the end\n')
# Write multiple lines from a list
lines = ['alpha\n', 'beta\n', 'gamma\n']
with open('list.txt', 'w') as f:
    f.writelines(lines)

⚠️ 'w' mode deletes existing content — no warning! Be careful. Use 'x' mode to create-but-never-overwrite.

🔨 Build It: Line & Word Counter

This script reads a file and reports line count, word count, and character count.

# counter.py
def analyze_file(path):
    try:
        with open(path, 'r') as f:
            text = f.read()
    except FileNotFoundError:
        return "File not found."

    lines = text.count('\n')
    words = len(text.split())
    chars = len(text)
    return {'lines': lines, 'words': words, 'chars': chars}

# Usage
stats = analyze_file('sample.txt')
print(f"Lines: {stats['lines']}, Words: {stats['words']}, Chars: {stats['chars']}")

Save this as counter.py and test it on a text file you create. str.split() splits on whitespace — perfect for word counting.

✅ Checkpoint

Quick question — which of these correctly reads a file safely?

A) f = open('data.txt'); text = f.read()

B) with open('data.txt') as f: text = f.read() ← Correct!

C) read('data.txt')

Why B? with auto-closes the file, even if an exception occurs. A works but risks forgetting close(). C isn't a real function.

📝 Key Takeaways

1

Always use with open(...) as f: — it guarantees the file is closed, no exceptions.

2

Choose the right mode: 'r' (read), 'w' (write — overwrites!), 'a' (append), 'x' (create-exclusive).

3

Iterate with for line in f for large files; use .read() or .readlines() for small files.
