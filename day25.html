Variables  ·  let & const

Declaring Variables: let vs const

JavaScript has two main ways to declare variables. Pick the right one:

let — can be reassigned

Use when the value changes over time: counters, accumulators, user input.

let score = 0;
let playerName = "Alice";

score = 10;  // ✅ allowed — let can be reassigned

const — cannot be reassigned

Use when the value stays the same: configs, fixed references, most things.

const pi = 3.14159;
const siteName = "Co-op Prep";

pi = 3;  // ❌ TypeError — const can't be reassigned

Rule of thumb: start with const. Only switch to let if you know the value will change. (var is legacy — don't use it.)

Primitive Types  ·  Strings, Numbers, Booleans

Strings — text data

Strings hold text. Use single quotes ' ', double quotes " ", or backticks ` ` (template literals).

const greeting = "Hello, world!";
const name = 'Alice';
const message = `Hi, ${name}!`;  // template literal — injects variables

// Common string operations:
console.log(greeting.length);       // 13 — number of characters
console.log(greeting.toUpperCase()); // "HELLO, WORLD!"
console.log(message);               // "Hi, Alice!"
💡 Template literals (backticks)

`Hello ${name}` — use ${ } to embed any expression. Much cleaner than concatenation with +. Prefer backticks whenever you mix text and variables.

Numbers — integers & floats

JS has one number type for all numeric values. Basic math works as expected.

const age = 25;           // integer
const price = 19.99;      // floating-point
const negative = -5;

// Arithmetic:
const sum = 10 + 5;      // 15
const diff = 10 - 5;     // 5
const product = 10 * 5;  // 50
const quotient = 10 / 3; // 3.3333...  (true division, not integer)
const remainder = 10 % 3; // 1  (modulo — "what's left over")

Watch out: 0.1 + 0.2 is 0.30000000000000004 — floating-point precision is imperfect in every language.

Booleans — true / false

Booleans are the simplest type: either true or false. They power conditionals and comparisons.

Declaring booleans

const isLoggedIn = true;
const isComplete = false;

let score = 85;
const passed = score >= 60;  // true

Comparison operators

// These produce booleans:
10 > 5    // true
10 === 5  // false  (strict equality)
10 !== 5  // true   (not equal)
"abc" < "z" // true  (alphabetical)
⚠️ Always use ===, never ==

== does type coercion (5 == "5" is true!). === checks type and value. Always use triple equals to avoid subtle bugs.

Console  ·  console.log()

Printing with console.log()

console.log() prints values to the browser's Developer Tools console — the JS equivalent of print() in Python.

const name = "Alice";
const age = 22;

console.log(name);         // "Alice"
console.log(age);          // 22
console.log("Hello", name); // "Hello Alice"

// Template literals work great:
console.log(`${name} is ${age} years old`);
// → "Alice is 22 years old"
// Multiple values:
console.log("Score:", 95, "Name:", name);

// Objects and arrays display interactively:
console.log({ name, age });

// Type-checking helper:
console.log(typeof name);   // "string"
console.log(typeof age);    // "number"
console.log(typeof true); // "boolean"

Tip: Open DevTools with F12 or Ctrl+Shift+I (Windows/Linux) / Cmd+Opt+I (Mac) to see your output.

Linking JS to HTML

Walkthrough: Your First JS Project

Create a folder, an HTML file, and a JS file. Step by step.

1. Project structure

my-first-js/
├── index.html
└── script.js

One folder, two files. The HTML loads the JS.

2. Terminal commands

$ mkdir my-first-js
$ cd my-first-js
$ touch index.html script.js

Create the directory and both files.

3. Write index.html

Standard HTML boilerplate — the <script> tag links the JS file.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>My First JS</title>
</head>
<body>
  <h1>Open the Console!</h1>
  <p>Press F12 to see JavaScript output.</p>

  <!-- This one line links script.js -->
  <script src="script.js"></script>
</body>
</html>
⚠️ Put <script> at the end of <body>

Scripts in <head> run before the page renders — they can't access HTML elements that don't exist yet. Placing it just before </body> ensures the DOM is ready. (We'll learn defer and DOMContentLoaded later.)

4. Write script.js

Your first JavaScript file — declare variables and log them.

// script.js — my first JS code

// Strings
const greeting = "Hello, co-op prep!";
const name = 'Alice';
const intro = `${greeting} I'm ${name}`;

// Numbers
let score = 0;
const maxScore = 100;
score = 85;  // reassigned — score changes

// Booleans
const passed = score >= 60;
let isComplete = false;

// Log everything to the console
console.log(intro);
console.log("Score:", score, "/", maxScore);
console.log("Passed?", passed);
console.log("Type of score:", typeof score);

Now open index.html in your browser and check the DevTools console (F12 → Console tab).

5. Verify — what you should see

If everything is set up correctly, the DevTools Console will show:

▼ Console Output
Hello, co-op prep! I'm Alice
Score: 85 / 100
Passed? true
Type of score: number

If you see nothing:

  • Did you save script.js?
  • Is the <script src="script.js"> tag at the end of <body>?
  • Refresh the page — some browsers cache old files

Exercise Solutions

Don't peek until you've tried!

Ex.1 — Budget

Ex.2 — Grade

let studentName = "Alice";
let grade = 85;
const passing = grade >= 60;
const honours = grade >= 80;

console.log("Passing:", passing, "Honours:", honours);
Ex.3 — Type explorer

console.log(42, typeof 42) → 42 "number". Try null (typeof returns "object" — a famous JS quirk!), and a variable you declare without assigning anything (let x; → typeof x is "undefined").
