🤔 What is React?

React is a JavaScript library for building user interfaces from reusable pieces called components.

Key Ideas

  • Declarative — describe what you want, not how
  • Component-based — encapsulate HTML, CSS, and logic together
  • Reactive — when data changes, the UI updates automatically

Analogy: 🧱 LEGO

Each component is a LEGO brick. Bricks are reusable, composable (stack on each other), and self-contained. A page is built from many bricks, not one giant piece of plastic.

React vs. Plain JS

Plain JS: manually create elements, set attributes, append to DOM

// Create a button with vanilla JS
const btn = document.createElement('button');
btn.textContent = 'Click me';
btn.className = 'btn-primary';
btn.onclick = () => alert('Hi!');
document.body.appendChild(btn);

React: describe what you want

// Same button in React
function MyButton() {
  return (
    <button className="btn-primary"
            onClick={() => alert('Hi!')}>
      Click me
    </button>
  );
}
2 / 12
React removes the manual DOM manipulation. You describe the UI you want and React figures out how to update the real DOM efficiently.

📝 JSX: JavaScript + XML

JSX looks like HTML but runs inside JavaScript. Every JSX expression produces a React element.

📏 JSX Rules

  • One root element — wrap siblings in a <div> or <>...</> (Fragment)
  • Close every tag — <br />, <img />
  • className instead of class (JS reserved word)
  • Curly braces { } embed JavaScript expressions
  • Comments: {/* comment */}
// Simple JSX — one root, embedded expression
function Greeting() {
  const name = "Alice";
  return (
    <div className="greeting">
      <h1>Hello, {name}!</h1>
      <p>2 + 2 = {2 + 2}</p>
      <img
        src={user.avatar}
        alt="Avatar"
        className="avatar"
      />
    </div>
  );
}

Anything inside { } is evaluated as JavaScript — variables, arithmetic, function calls, ternary expressions.

5 / 12
JSX is the syntax that makes React feel like writing HTML. Emphasize the curly-brace escape hatch into JS — it's the most common 'aha' moment.

⚠️ JSX Gotchas to Know

❌ HTML attribute names

JSX uses camelCase for most attributes:

// HTML:
<input onclick="submit()" maxlength="10" tabindex="0">

// JSX:
<input
  onClick={submit}
  maxLength={10}
  tabIndex={0}
/>

❌ Inline styles

Inline styles use a camelCase object, not a string:

// HTML:
<div style="background-color: blue; font-size: 20px">

// JSX:
<div style={{
  backgroundColor: 'blue',
  fontSize: 20
}}>

❌ Conditional rendering

// Ternary works inside JSX:
{isLoggedIn
  ? <Profile />
  : <LoginButton />
}

// && for "show if true":
{unreadCount > 0 &&
  <span>{unreadCount} new</span>
}
6 / 12
These three gotchas (camelCase attributes, style objects, conditional rendering) are where beginners trip up most. Cover them clearly.

🎁 Props: Passing Data to Components

Props (short for "properties") let you pass data from a parent component to a child component — like function arguments for your UI.

How it works

  1. Parent passes props as HTML-like attributes
  2. Child receives them as a single object (usually destructured)
  3. Props are read-only — the child never modifies them

Analogy: 📦 Shipping label

A parent component is like a sender putting a package together. The props are the shipping label — address, contents, handling instructions. The child component receives the package and uses the label info as-is.

// Parent passes props:
function App() {
  return (
    <UserCard
      name="Alice"
      age={25}
      isOnline={true}
    />
  );
}

// Child receives props (destructured):
function UserCard({ name, age, isOnline }) {
  return (
    <div className="card">
      <h3>{name}</h3>
      <p>Age: {age}</p>
      <span>
        {isOnline ? '🟢 Online' : '🔴 Offline'}
      </span>
    </div>
  );
}
7 / 12
Props are the fundamental communication channel in React. Emphasize that props are immutable — the child should never assign to them.

🛠️ Props in Practice

Let's build a reusable ProductCard component that renders product info from props.

// ProductCard.jsx
function ProductCard({ title, price, inStock }) {
  return (
    <div className="product">
      <h3>{title}</h3>
      <p>${price.toFixed(2)}</p>
      <p>
        {inStock
          ? '✅ In stock'
          : '❌ Out of stock'}
      </p>
    </div>
  );
}

// App.jsx — using it:
function App() {
  return (
    <div>
      <ProductCard
        title="Wireless Mouse"
        price={29.99}
        inStock={true}
      />
      <ProductCard
        title="Mechanical Keyboard"
        price={89.99}
        inStock={false}
      />
    </div>
  );
}

Same component, different data = two different cards.

💡 Default props pattern

  return <h1>Hi, {name}!</h1>;

✏️ Exercise 2: Product Listing

Task

You have a product array. Render a list of ProductCard components using Array.map() inside JSX.

// Data (put this in App.jsx):
const products = [
  { id: 1, title: "USB-C Hub",
    price: 34.99, inStock: true },
  { id: 2, title: "Webcam HD",
    price: 79.99, inStock: true },
  { id: 3, title: "Monitor Arm",
    price: 129.99, inStock: false },
];

// Use .map() inside JSX:
function App() {
  return (
    <div className="product-list">
      {products.map(product =>
        <ProductCard
          key={product.id}
          title={product.title}
          price={product.price}
          inStock={product.inStock}
        />
      )}
    </div>
  );
}

⚠️ The key Prop

When rendering lists with .map(), each element needs a unique key prop. React uses it to track which items change. Use the item's id if available — never the array index.

💡 Hints

Click to reveal →

  • Use the ProductCard component from the code slide
  • The key prop goes on the mapped component, not a wrapper
  • Your output should show 3 cards — 2 in stock, 1 out of stock

🎯 Key Takeaways

1️⃣ React is a component library

Write reusable functions that return UI descriptions (JSX). Compose them like LEGO bricks.

2️⃣ JSX blends HTML with JavaScript

Use { } for expressions, className for CSS, camelCase for attributes. One root element per component.

3️⃣ Props make components reusable

Pass data from parent to child via attributes. Props are read-only — a component never modifies its own props.
