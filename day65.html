What is Dynamic Programming?

Solving problems by breaking them into overlapping subproblems
and storing results to avoid redundant work

DP sounds intimidating, but it's really just "don't compute the same thing twice."
3 / 13

Overlapping Subproblems

In recursive problems like Fibonacci, the same subproblems get solved over and over — that's the overlapping subproblems property.

❌ Naive recursion

def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)

# fib(5) calls fib(4) and fib(3)
# fib(4) calls fib(3) AGAIN → duplicated work!

Time: O(2n) — exponential!

✅ With memoization

def fib(n, memo={}):
    if n in memo:
        return memo[n]
    if n <= 1:
        return n
    memo[n] = fib(n-1, memo) + \
               fib(n-2, memo)
    return memo[n]

# Each n computed once → O(n) time

Time: O(n) — linear!

Draw the recursion tree on a whiteboard to show how fib(3) is called twice in the naive version. The memoization cache eliminates those duplicates.

Fibonacci — Step by Step

Follow along. Create fib.py and type each version.

Version 1: Naive (test it)

def fib_naive(n):
    if n <= 1:
        return n
    return fib_naive(n-1) + \
           fib_naive(n-2)

# Try: fib_naive(10) → 55
# Try: fib_naive(40) → wait...

Version 2: Memoized

def fib_memo(n, memo={}):
    if n in memo:
        return memo[n]
    if n <= 1:
        return n
    memo[n] = fib_memo(n-1, memo) + \
               fib_memo(n-2, memo)
    return memo[n]

# Try: fib_memo(100) → instant!

Version 3: Bottom-up (for comparison)

def fib_bottom_up(n):
    if n <= 1:
        return n
    prev2, prev1 = 0, 1
    for i in range(2, n + 1):
        cur = prev1 + prev2
        prev2, prev1 = prev1, cur
    return prev1
# O(1) space — only 2 variables!
Run fib_naive(40) first — it'll take ~30 seconds. Then run fib_memo(100) — instant. The contrast is the whole point of DP.

Classic "this is Fibonacci in disguise" problem. Most common DP question asked by interviewers.
6 / 13

Climbing Stairs — Walkthrough

Problem: You're climbing a staircase with n steps. Each move is +1 or +2. How many distinct ways to reach the top?

Think about it:

  • To reach step i, you come from i-1 (took 1 step) or i-2 (took 2 steps)
  • So: ways[i] = ways[i-1] + ways[i-2]
  • Base: ways[1] = 1, ways[2] = 2

It's Fibonacci! ways[n] = fib(n+1)

Solution:

def climbStairs(n):
    if n <= 2:
        return n
    prev2, prev1 = 1, 2
    for i in range(3, n + 1):
        cur = prev1 + prev2
        prev2, prev1 = prev1, cur
    return prev1

Time: O(n) · Space: O(1)

Walk through n=3 manually: (1+1+1), (1+2), (2+1) = 3 ways. Then show how the recurrence emerges naturally from the problem statement.

LeetCode 198: House Robber

Rob houses on a street — can't rob two adjacent houses.
Maximize the total loot.

Still DP but the recurrence is slightly different — shows DP isn't always Fibonacci. The key is deciding whether to rob the current house or skip it.
9 / 13

House Robber — Walkthrough

Problem: Array nums = money in each house. Can't rob adjacent houses. Maximize total.

Recurrence:

At each house i, two choices:

  • Rob it: take nums[i] + dp[i-2]
  • Skip it: keep dp[i-1]
dp[i] = max(
    nums[i] + dp[i-2],   # rob
    dp[i-1]               # skip
)

Example:

# nums = [1, 2, 3, 1]
# dp[0] = 1
# dp[1] = max(2, 1) = 2
# dp[2] = max(3+1, 2) = 4
# dp[3] = max(1+2, 4) = 4
# → max = 4 (rob houses 1 + 3 != adjacent)

def rob(nums):
    rob, skip = 0, 0
    for n in nums:
        new_rob = skip + n
        skip = max(rob, skip)
        rob = new_rob
    return max(rob, skip)

Time: O(n) · Space: O(1)

Walk through the example manually on a whiteboard. Show that the optimal decision at each house depends on the best up to i-2 and i-1 — that's the DP subproblem structure.

Practice Exercises

✏️ Exercise 1: Fibonacci with Cache Limit

Modify fib_memo to accept a max_size parameter. If the memo cache exceeds max_size, fall back to the naive O(2n) approach. Test with fib_limit(100, 5) — how slow is it?

✏️ Exercise 2: Min Cost Climbing Stairs

LeetCode 746. You're given an array cost where cost[i] is the cost to step on stair i. You can start at step 0 or 1 and climb 1 or 2 steps at a time. Find the minimum cost to reach the top.

Hint: dp[i] = cost[i] + min(dp[i-1], dp[i-2])

💡 General DP approach: 1) Identify the recurrence relation. 2) Solve base cases. 3) Iterate from bottom-up with O(1) space when possible. 4) Test with small examples first.

# Exercise 1: Fib with cache limit
def fib_limit(n, max_size, memo=None):
    if memo is None:
        memo = {}
    if n in memo:
        return memo[n]
    if n <= 1:
        return n
    if len(memo) >= max_size:
        return fib_naive(n)  # fall back
    memo[n] = fib_limit(n-1, max_size, memo) + \
               fib_limit(n-2, max_size, memo)
    return memo[n]

# Exercise 2: Min Cost Climbing Stairs
def minCostClimbingStairs(cost):
    n = len(cost)
    prev2, prev1 = cost[0], cost[1]
    for i in range(2, n):
        cur = cost[i] + min(prev1, prev2)
        prev2, prev1 = prev1, cur
    return min(prev1, prev2)
These exercises extend the concepts just enough: cache limits test understanding of memoization tradeoffs, and Min Cost Stairs is a small variant on Climbing Stairs that changes the recurrence.

Key Takeaways

1

Overlapping subproblems are the key signal to use DP — if the recursion tree re-computes the same inputs, memoize them.

2

Top-down (memoization) is easiest to write. Bottom-up (tabulation) saves space and avoids recursion limits.

3

Many DP problems follow Fibonacci-style recurrence (Climbing Stairs) or max-of-two-choices (House Robber). Recognize the pattern.

The three takeaways fit on one slide. Ask learners to write them down.
