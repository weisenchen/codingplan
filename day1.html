1 What Is a Computer?

A computer is a machine that takes input, processes it, stores results, and produces output. Every computer — laptop, phone, server — has the same four core parts:

Long-term memory

Input / Output

you ↔ computer

📦 The Desk Analogy

Storage (disk) = your filing cabinet — all files are stored here permanently, but getting them out takes time.

RAM = your desk surface — only what you're actively working on sits here. You can grab things from the cabinet and put them on your desk (load into RAM).

CPU = your hands and brain — reading, writing, calculating. Whatever is on your desk (RAM), you can work with.

This is why a computer with more RAM and a faster CPU feels "snappy" — more desk space, faster hands.

2 Files, Folders & Programs

📄 Files

A file is a named bundle of data on your disk. Every file has a type determined by its extension (the part after the dot):

ExampleTypeWhat it contains
report.docxWord documentText, formatting, images — readable by Microsoft Word
photo.jpgImageCompressed picture data — readable by image viewers
script.pyPython programCode — readable by the Python interpreter
index.htmlWeb pageHTML markup — readable by a web browser
appExecutable binaryMachine code (not human-readable) — run directly by the CPU
💡 Key insight: A file's extension doesn't change what's inside — it tells the OS which program to open it with. Rename photo.jpg to photo.txt and it'll still contain image data (your text editor will show gibberish).

📁 Folders (Directories)

A folder (or directory) is a container that holds files and other folders. This creates a tree structure — one folder inside another, branching out from the root.

/ ← root (the top-level folder)
├── home/
│ └── wei/ ← your home directory (~)
│ ├── bash-examples.sh
│ ├── Desktop/
│ ├── Documents/
│ └── projects/
│ └── todo.py
├── tmp/ ← temporary files
├── etc/ ← system configuration
└── usr/ ← installed programs

Every file has a path — its address in this tree. /home/wei/bash-examples.sh means: start at root /, enter home, enter wei, find the file bash-examples.sh.

⚙️ Programs

A program is a file that the CPU can execute. When you run a program, the OS:

  1. Reads the program file from disk
  2. Loads it into RAM
  3. Tells the CPU to start executing its instructions
  4. Keeps track of what's running (memory, files it opened, etc.)

Examples of programs: your browser (Chrome/Firefox), a text editor (VS Code), the terminal, the ls command you'll learn today.

3 How the Operating System Works

🏢
Operating System
The manager

The Operating System (OS) — Linux, Windows, macOS — is the master program that runs the whole computer. Think of it as a building manager:

JobWhat the OS doesWhy it matters
📋 Process management Decides which programs get CPU time and when Lets you browser the web while playing music, each thinking it has the CPU
🧠 Memory management Assigns RAM to each program, prevents them from stepping on each other One program can't read another's private data by accident
📁 File system Organizes files on disk into the tree you saw above You access files by name/path, not by raw disk sector numbers
🔒 Permissions Decides which users and programs can read/write/run each file Prevents a random app from deleting your documents
🔌 Drivers Talks to hardware (keyboard, screen, Wi-Fi, GPU) You don't need to know the exact model of your network card to use the internet

🏢 The Building Manager

The OS is a building manager. The CPU is a meeting room. Programs (tenants) request the room, the manager schedules them, gives them desk space (RAM), a filing cabinet drawer (disk space), and makes sure they don't steal from each other (security). When you hit Ctrl+C, the manager kicks that tenant out and reclaims the room.

File System Mental Model

The file system is the most important concept for Day 1. Memorize this:

Three facts about the file system:
1. It's a tree — one root (/), everything branches from there
2. Every file/folder has one parent (except root)
3. You are always "in" one folder called the current working directory

When you open a terminal, you start in your home directory — usually /home/yourname (shortcut: ~). From there you can navigate down into subfolders (cd Desktop) or up to the parent (cd ..).

4 What Is a Terminal?

📟 Terminal vs Shell

Two things you'll hear constantly — they're different but always used together:

TermWhat it isAnalogy
Terminal The window app — just draws text and accepts keystrokes The physical monitor + keyboard you're looking at
Shell The program running inside — interprets commands, runs them, shows results The person sitting at the desk, reading your request and doing it
Bash The most common shell on Linux (Bourne Again SHell) Alex — a very experienced assistant who knows a lot of shortcuts

When you open a terminal app, it starts a shell (bash). You see a prompt like:

wei@computer:~$ # this is the prompt — it's waiting for you

The prompt tells you: who you are (wei), where you are (~ = home), and that it's ready for a command ($).

⚡ Why Use a Terminal?

Why not just use the mouse and click folders?

  • Speed — one command can do what takes 10 clicks. A single grep -r "error" /var/log/ searches thousands of files instantly.
  • Automation — write commands in a script and run them all at once, on a schedule, or on 100 servers.
  • Remote work — no mouse on a remote server (SSH). Terminal is the only way.
  • Power — many development tools (git, docker, compilers, package managers) are CLI-only or faster in CLI.
  • Transparency — you see exactly what happens, no hidden clicks, no GUIs lying to you.
🔥 Your first rule: Every time you find yourself clicking through five menus to do something, ask yourself: "Can I type this instead?" The answer is almost always yes, and it'll be faster.

6 Putting It All Together — Real Example

Let's trace what happens when you type this in the terminal:

wei@computer:~$ grep "error" /var/log/syslog | tail -5
# What happens step by step:
# 1. bash reads your line, splits it by |
# 2. bash starts grep "error" /var/log/syslog
# - The OS loads /usr/bin/grep into RAM
# - grep asks the OS to open /var/log/syslog
# - The OS checks: can wei read this file? (permissions)
# - Yes → OS reads the file from disk, hands data to grep
# - grep filters lines containing "error", sends them to stdout
# 3. The | pipe connects grep's stdout to tail's stdin
# 4. tail -5 reads the first 5 lines from stdin, prints them
# 5. bash shows the output on your screen
# 6. bash prints a new prompt — ready for your next command

Every command you run follows this same cycle. The OS, the shell, file permissions, pipes — they all work together.

7 🏆 Your 5-Minute Challenge

Type these commands one by one in your terminal. Do not copy-paste — type them. Your fingers need to learn the keys.
