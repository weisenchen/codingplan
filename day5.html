💡 Conditionals

Making decisions in code

Section break. Transition from setup to if/else.

Boolean Expressions

Every conditional starts with a boolean expression — something that's either True or False.

Comparison Operators

==equals
!=not equal
<less than
>greater than
<=less or equal
>=greater or equal

Examples

print(5 == 5)      # True
print(5 != 3)      # True
print(3 < 5)       # True
print(10 > 20)     # False
print(7 <= 7)      # True

age = 18
print(age >= 18)   # True
⚠ Watch Out

= is assignment (set a variable). == is comparison (check equality). Mixing them up is the #1 beginner bug.

One of the most important slides. Learners need to internalize that == compares values and = assigns them. Show the difference with a live demo if time allows.

if / elif / else

The backbone of decision-making in Python:

score = 85

if score >= 90:
    print("A")
elif score >= 80:
    print("B")
elif score >= 70:
    print("C")
else:
    print("F")
# Prints: B
Rules
  • if is required first
  • Zero or more elif
  • else at most once, at the end
  • Colon : after each condition
  • Body is indented (4 spaces)
  • First true branch runs; rest skipped
Truthiness

Values like 0, "", None, [] are falsy — they evaluate to False in an if condition. Everything else is truthy.

The grade example is familiar to everyone. First-only-match is critical — learners often think all matching branches run. Contrast with a chain of if-statements (no elif) to show the difference.

Combining Conditions

Use and, or, and not to build complex logic:

# Both must be True
temp = 75
sunny = True
if temp > 70 and sunny:
    print("Let's go outside!")

# At least one True
if has_key or knows_code:
    print("Door opens")

# Negation
if not logged_in:
    print("Please log in")
Operator Precedence

When in doubt, use parentheses to make order explicit:

if (age >= 18 and has_id) or parent_ok:
    print("Allowed")
Short-Circuit

Python stops evaluating and/or as soon as it knows the result. This can prevent errors:

if items and items[0] > 0:
    print("First item positive")
Short-circuit evaluation is a superpower — it protects you from IndexError on empty lists. Demonstrate what happens without the 'items and' guard.

🔁 for Loops

Doing things repeatedly

Section break. Transition from conditionals to loops.

for Loops with range()

Repeat an action a fixed number of times:

# range(stop): 0 to stop-1
for i in range(5):
    print(i)
# Prints: 0 1 2 3 4

# range(start, stop)
for i in range(2, 6):
    print(i)
# Prints: 2 3 4 5

# range(start, stop, step)
for i in range(0, 10, 2):
    print(i)
# Prints: 0 2 4 6 8
range() Signatures
range(stop)0 → stop-1
range(start, stop)start → stop-1
range(start, stop, step)start → stop-1, by step
⚠ Off-by-One

range(n) produces n values, 0 through n-1.

for i in range(3):
    print(i)  # 0, 1, 2 — NOT 3!
Three forms of range(). The off-by-one error is THE most common loop bug — emphasize it hard.

Looping Over Lists & Strings

for loops work with any iterable object:

# Over a list
fruits = ["apple", "banana", "cherry"]
for fruit in fruits:
    print(fruit)

# Over a string (characters)
for ch in "hello":
    print(ch)
# Prints: h e l l o

# Using enumerate for index
for i, fruit in enumerate(fruits):
    print(f"{i}: {fruit}")
enumerate()

Need the index and the element? Use enumerate(). It yields (index, value) tuples.

# Manual way (awkward):
for i in range(len(items)):
    print(items[i])

# Pythonic way:
for i, item in enumerate(items):
    print(item)
⚠ Don't Modify While Looping

Don't add/remove items from a list while looping over it. Build a new list instead.

enumerate is a key Python idiom. Show both the manual range(len()) way and the Pythonic enumerate way so they see why enumerate is preferred.

Nested Loops & The Inner Loop Pattern

Put a loop inside another loop for grids, tables, and multiplication tables:

# Multiplication table (1 to 3)
for row in range(1, 4):
    line = ""
    for col in range(1, 4):
        line += f"{row*col:3}"
    print(line)
# Output:
#   1  2  3
#   2  4  6
#   3  6  9
Outer Loop

Runs once per row. The inner loop runs completely for each outer iteration.

Inner Loop

Completes all its iterations before the outer loop advances.

Complexity

N iterations × M iterations = N × M total steps.

Nested loops are tough for beginners. The multiplication table is the most intuitive example. Walk through it step by step on the whiteboard.

FizzBuzz Walkthrough

The classic coding interview warm-up:

# Print 1 to 100. Multiples of 3: "Fizz"
# Multiples of 5: "Buzz". Both: "FizzBuzz"

for n in range(1, 101):
    if n % 3 == 0 and n % 5 == 0:
        print("FizzBuzz")
    elif n % 3 == 0:
        print("Fizz")
    elif n % 5 == 0:
        print("Buzz")
    else:
        print(n)
Why FizzBuzz?

Tests if you can combine if/else, loops, and the modulo operator (%) into a working program. Surprisingly many interviewees can't!

Key Insight

Check the most specific condition first (both 3 AND 5). If you check 3 first, n=15 prints "Fizz" instead of "FizzBuzz".

Walk through n=1..20 manually on the whiteboard. Show what happens if you swap the order — that's where most bugs come from. The modulo operator % gives remainder — n % 3 == 0 means divisible by 3.

Exercise 1: Even or Odd

Hint

Exercise 2: Sum of 1 to N

Hint

Exercise 3: FizzBuzz (your turn!)

Write FizzBuzz from scratch, but this time only go from 1 to 30. Make sure "FizzBuzz" appears on 15 and 30.

Hint

Use range(1, 31) and check the 15 case first. Test with a few values manually to confirm.

Give learners 5-7 minutes per exercise. The exercises progressively build — Even/Odd introduces the pattern, Sum adds loop accumulation, FizzBuzz combines everything.

Solutions (Try First!)

# Ex 1: Even or Odd
num = int(input("Enter a number: "))
if num % 2 == 0:
    print(f"{num} is even")
else:
    print(f"{num} is odd")

# Ex 2: Sum of 1 to N
n = int(input("Enter N: "))
total = 0
for i in range(1, n + 1):
    total += i
print(f"Sum = {total}")
# Ex 3: FizzBuzz (1 to 30)
for n in range(1, 31):
    if n % 15 == 0:
        print("FizzBuzz")
    elif n % 3 == 0:
        print("Fizz")
    elif n % 5 == 0:
        print("Buzz")
    else:
        print(n)

# Alternative using % 15:
# n % 15 == 0 is the same as
# n % 3 == 0 and n % 5 == 0

Only check after you've tried each exercise for at least 5 minutes!

Solutions slide. Emphasize that the n % 15 trick is a faster way to check both conditions.

Key Takeaways

1
if/elif/else lets code make decisions. First true branch wins — order matters! Use == for comparison, never =.
2
for loops iterate over sequences. range(n) gives 0 to n-1. Combine with enumerate() to get indices.
3
FizzBuzz combines everything: loops + conditionals + modulo. Check the most specific case first (both conditions) before individual ones.
