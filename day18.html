🔧 Polish & Error Handling

Make it robust — real programs crash gracefully

Error Handling — The Mindset

A user should never see a raw Python traceback. Wrap everything in friendly messages.

❌ Before

def load_todos():
    with open(FILE) as f:
        return json.load(f)
# Crash if FILE missing or corrupt
# User sees a terrifying traceback

✅ After

def load_todos():
    try:
        with open(FILE) as f:
            return json.load(f)
    except FileNotFoundError:
        return []
    except json.JSONDecodeError:
        print("⚠️ todos.json is corrupt. Starting fresh.")
        return []

Same pattern for add_task, complete_task, and delete_task — wrap every file operation.

Input Validation — Assume Nothing

Never trust user input. Validate everything before acting.

def complete_task(todo_id):
    # Validate the argument exists
    if not todo_id:
        print("❌ Usage: todo complete <id>")
        return 1
    try:
        idx = int(todo_id)
    except ValueError:
        print(f"❌ '{todo_id}' is not a valid number")
        return 1

    todos = load_todos()
    # Validate the ID is in range
    if idx < 1 or idx > len(todos):
        print(f"❌ No task with id {idx}")
        return 1
    # Validate it's not already done
    if todos[idx-1]["done"]:
        print(f"⚠️ Task {idx} is already completed")
        return 0

    todos[idx-1]["done"] = True
    save_todos(todos)
    print(f"✅ Task {idx} marked complete!")
    return 0

Return 0 for success, 1 for errors — standard Unix convention.

Exit Codes & Script Entry Point

Every command returns a proper exit code. The entry point dispatches cleanly.

Main dispatch

import sys
import argparse

def main():
    parser = argparse.ArgumentParser(
        description="📋 CLI Todo List"
    )
    sub = parser.add_subparsers(dest="command")
    # ... add subcommand parsers ...
    args = parser.parse_args()

    if args.command == "add":
        return add_task(args.text)
    # ... etc ...
    return 0

if __name__ == "__main__":
    sys.exit(main())

Why sys.exit(main())?

Every function returns an int. sys.exit(0) = success, sys.exit(1) = error. This means:

  • Shell scripts can check $?
  • CI pipelines know if it failed
  • Other tools can chain it with &&

📦 Git & GitHub

From local code to a shipped project

Initializing the Repo

Start fresh with git init. Keep your commits small and atomic.

First, .gitignore

# .gitignore
venv/
__pycache__/
*.pyc
.env
.DS_Store

Then commit

git init
git add .
git status           # verify before commit
git commit -m "feat: CLI todo list with add/list/complete/delete"

Commit message convention: type: description — e.g. feat:, fix:, docs:, refactor:. This is Conventional Commits, used by most teams.

Pushing to GitHub

Create the repo on GitHub first (empty, no README or .gitignore since we already have them locally).

# After creating the repo on github.com:
git remote add origin git@github.com:<your-username>/todo-list.git
git branch -M main
git push -u origin main

Use SSH (the git@github.com:... URL) over HTTPS — it's more secure and you won't need to re-authenticate. GitHub SSH setup guide.

⚠️ Common pitfall: If you get a remote origin already exists error, run git remote set-url origin <url> instead.
