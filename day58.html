🔍 BFS: Level-Order Traversal

How it works: Visit nodes one level at a time, from left to right, top to bottom.

3 → 9 20 → 15 7

BFS order: 3 → 9 → 20 → 15 → 7

Key insight: BFS uses a queue (FIFO). Process the front, enqueue its children. This ensures nodes at depth d are always visited before any node at depth d+1.

Algorithm:

1 queue = deque([root]) 2 while queue: 3 node = queue.popleft() 4 process(node) 5 if node.left: 6 queue.append(node.left) 7 if node.right: 8 queue.append(node.right)

Time: O(n) · Space: O(w) where w = max width

🧩 BFS: Collecting by Level

The standard BFS visits all nodes — but many problems (like LeetCode #102) ask for values grouped by level.

def level_order(root): if not root: return [] result = [] queue = deque([root]) while queue: level_size = len(queue) # nodes at current level level_vals = [] for _ in range(level_size): # process entire level node = queue.popleft() level_vals.append(node.val) if node.left: queue.append(node.left) if node.right: queue.append(node.right) result.append(level_vals) # [[3], [9,20], [15,7]] return result
Pattern: Snapshot len(queue) at the start of each level — that tells you how many nodes belong to the current level. Then iterate exactly that many times.

🧭 DFS: Three Orders

DFS visits go deep first — explore a full branch before backtracking. Three canonical orders:

OrderSequenceUse case
Pre-orderroot → left → rightTree copy, serialize
In-orderleft → root → rightSorted order (BST)
Post-orderleft → right → rootTree deletion
Pattern: DFS uses a stack (LIFO). Push children, pop the top. The order you push determines the traversal direction.

On the tree [3,9,20,null,null,15,7]:

# Pre-order: 3 → 9 → 20 → 15 → 7 # In-order: 9 → 3 → 15 → 20 → 7 # Post-order: 9 → 15 → 7 → 20 → 3 # Iterative DFS with stack: stack = [root] while stack: node = stack.pop() # LIFO process(node) if node.right: # push right first so stack.append(node.right) # left is processed first if node.left: stack.append(node.left)

Time: O(n) · Space: O(h) where h = tree height

💻 DFS: Recursive vs Iterative

🔁 Recursive (simpler)

def preorder(root, res): if not root: return res.append(root.val) preorder(root.left, res) preorder(root.right, res) def inorder(root, res): if not root: return inorder(root.left, res) res.append(root.val) inorder(root.right, res) def postorder(root, res): if not root: return postorder(root.left, res) postorder(root.right, res) res.append(root.val)

📦 Iterative (stack-based)

# Pre-order iterative stack = [root] while stack: node = stack.pop() process(node) if node.right: stack.append(node.right) if node.left: stack.append(node.left) # In-order iterative stack, curr = [], root while stack or curr: while curr: stack.append(curr) curr = curr.left curr = stack.pop() process(curr) curr = curr.right # Post-order (reverse of pre-order) stack, out = [root], [] while stack: node = stack.pop() out.append(node.val) if node.left: stack.append(node.left) if node.right: stack.append(node.right) # out is reverse post-order -> reverse it

🧪 Walkthrough: LeetCode #102

Problem: Given the root of a binary tree, return the level order traversal of its nodes' values (left to right, level by level).

# Input: [3, 9, 20, null, null, 15, 7] # Output: [[3], [9, 20], [15, 7]] from collections import deque class Solution: def levelOrder(self, root: Optional[TreeNode]) -> List[List[int]]: if not root: return [] result = [] q = deque([root]) while q: level_vals = [] for _ in range(len(q)): # freeze level size node = q.popleft() level_vals.append(node.val) if node.left: q.append(node.left) if node.right: q.append(node.right) result.append(level_vals) return result
Edge cases to consider: empty tree (return []), single node (return [[val]]), skewed tree (each level has one node).

⚖️ BFS vs DFS — When to Use What

Use BFS when:
  • You need the shortest path in an unweighted graph
  • You need level-by-level processing
  • Finding nodes close to the root
  • Tree is wide but shallow

BFS space = O(w) — worst-case for a full binary tree, w ≈ n/2

Use DFS when:
  • You need to explore all paths to leaves
  • Checking path existence or path sum
  • Tree is deep but narrow
  • You want in-order (sorted) traversal of a BST

DFS space = O(h) — worst-case for a skewed tree, h = n

⚠️ Common Pitfalls

❌ Using list.pop(0) as a queue — O(n) per operation! Always use deque.popleft() for O(1).
❌ Forgetting the empty tree check — Always guard if not root: return [] before starting traversal.
❌ Mutating the level size — Snapshot len(queue) at the start of each level; don't use a dynamic len(queue) in the loop condition.
❌ Push order in DFS — Push right then left if you want left-first traversal (stack is LIFO).

📝 Key Takeaways

📊

BFS = Queue

FIFO order. Process level-by-level. Use deque.popleft(). Snapshot len(queue) for level grouping.

🌲

DFS = Stack

LIFO order. Three variants: pre/in/post-order. Recursive is simpler; iterative uses an explicit stack. Push right then left for left-first.

⚡

Both are O(n)

BFS space = O(w) (width). DFS space = O(h) (height). Pick BFS for shortest-path / level problems; DFS for path-exploration / BST.
