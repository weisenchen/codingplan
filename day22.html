Flexbox is the cornerstone of modern CSS layout. This session gives them a tool they'll use on every single project going forward. Emphasize that Flexbox replaces float-based layouts and makes centering something you can achieve in one line.

🤔 Why Flexbox?

Before Flexbox, centering content or creating equal-height columns required CSS hacks, negative margins, and frustration.

❌ The Old Way

/* Floats + clearfix — nightmare */
.box { float: left; width: 33%; }
.clearfix::after {
  content: "";
  display: table;
  clear: both;
}

✅ Flexbox Way

display: flex;
gap: 16px;

Three words. That's it. Flexbox handles distribution, alignment, wrapping, and reordering — all without touching float.

Key insight: Flexbox is one-dimensional — great for rows OR columns, not both at once (that's Grid).

3 / 16

Show them the pain of float-based layouts first so they appreciate how much simpler Flexbox is. Demo the clearfix hack briefly so they recognize it when they see it in legacy codebases.

🚀 Activating Flexbox

One declaration on the container turns all direct children into flex items:

/* Container becomes a flex container */
.container {
  display: flex;
}

/* HTML */
<div class="container">
  <div>Item 1</div>
  <div>Item 2</div>
  <div>Item 3</div>
</div>

Item 2
Item 3

Mental model: The container manages the layout; items just flow into it.

justify-content is the secret to navbar layouts — space-between for logo + nav links is the most common pattern on the web.

/* The golden combo */

The 'perfect centering' combo (justify-content + align-items both center) is a Flexbox superpower. Before Flexbox, this required display:table or position:absolute + transform hacks.

📦 flex-wrap & gap

flex-wrap

By default, items shrink to fit one row. Add flex-wrap: wrap to let items overflow onto new lines — essential for responsive layouts.

display: flex;
flex-wrap: wrap;   /* allow wrapping */
gap: 16px;         /* modern spacing */

gap

Sets space between items without margin hacks. Works on both rows and columns.

2
3
4
5
Items wrap to new lines as space shrinks

⚠ Note: gap works only between flex items — no extra space before/after.

8 / 16

gap was added to flexbox in 2021 and is now broadly supported. Always prefer gap over margin hacks. flex-wrap: wrap is what makes a card grid responsive without media queries.

🏗️ Walkthrough: Build a Navbar

Let's build a real navbar using everything we've learned.

/* Step 1: HTML structure */
<nav class="navbar">
  <div class="logo">MyApp</div>
  <ul class="nav-links">
    <li><a href="#">Home</a></li>
    <li><a href="#">Features</a></li>
    <li><a href="#">Pricing</a></li>
  </ul>
</nav>

/* Step 2: Flexbox layout */
.navbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: 16px 32px;
  background: #1a1c27;
}

.nav-links {
  display: flex;
  gap: 24px;          /* space links apart */
  list-style: none;
}
MyApp
Home Features Pricing
← Result: Logo left, links right →

✓ justify-content: space-between pushes logo and links to opposite ends.
✓ align-items: center keeps everything vertically aligned.

The flex shorthand (flex-grow, flex-shrink, flex-basis) is what makes Flexbox responsive without media queries. flex: 1 1 250px is the 'golden' responsive card pattern.

🔧 Flex Item Properties

Properties that go on the children (the flex items), not the container.

Property What It Does Common Value
flex-grow How much an item can grow (ratio) 1 (all equal)
flex-shrink How much an item can shrink 0 (don't shrink)
1 (allow shrink)
flex-basis Starting size before grow/shrink 250px or auto
align-self Override cross-axis alignment for one item flex-end, center 
/* Shorthand: grow shrink basis */
.card { flex: 1 1 250px; }
.logo { flex: 0 0 auto; }   /* fixed, don't grow/shrink */
.spacer { flex: 1; }           /* shorthand: flex: 1 1 0 */

Pro tip: flex: 1 on a single item makes it fill remaining space — great for search bars and spacers in a navbar.

11 / 16

Most people just use the flex shorthand. The individual properties are there for edge cases. align-self is less common but very useful for navbar layouts where one item needs to be at the bottom.

🧩 Common Flexbox Patterns

Three patterns you'll use daily as a junior dev:

1. Centered Hero

.hero {
  display: flex;
  justify-content: center;
  align-items: center;
  min-height: 80vh;
}

One-line centering on both axes.

2. Holy Grail

.page {
  display: flex;
  flex-direction: column;
  min-height: 100vh;
}
.main {
  display: flex;
  flex: 1;          /* fills space */
}

Header + footer with content filling the middle.

3. Sidebar + Content

.layout {
  display: flex;
  gap: 24px;
}
.sidebar { flex: 0 0 250px; }   /* fixed 250px */
.content { flex: 1; }           /* takes rest */

Fixed sidebar, fluid content area.

12 / 16

These three patterns form the backbone of most web layouts. If a student can wireframe a page using one of these three patterns in Flexbox, they're ready for Grid.

✏️ Practice Exercise

Build a product listing section with:

  • A container with display: flex and flex-wrap: wrap
  • Product cards with flex: 1 1 200px
  • Each card has: title, price, and a "Add to Cart" button
  • justify-content: center on the container
  • gap: 24px between cards
  • Cards should be align-self: flex-end on the last row if rows are uneven

💡 Hints:

  • Start with a <section> container and 6 <div> cards
  • Give each card a fixed height so uneven rows are visible
  • Try changing align-items and see how card heights change

This exercise tests wrapping, sizing, and alignment. The stretch behavior is important — students often don't realize align-items: stretch (the default) makes all items the same height. Give them 10-12 minutes for this.

✅ Exercise Solution

<!-- HTML -->
<section class="product-grid">
  <div class="product-card">
    <h3>Widget</h3>
    <p class="price">$19.99</p>
    <button>Add to Cart</button>
  </div>
  <!-- Repeat for 5 more cards -->
</section>

/* CSS */
.product-grid {
  display: flex;
  flex-wrap: wrap;
  justify-content: center;
  gap: 24px;
}
.product-card {
  flex: 1 1 200px;
  max-width: 280px;
  padding: 20px;
  border: 1px solid #e0e0e0;
  border-radius: 10px;
  text-align: center;
}
.price { color: #34d399; font-weight: bold; }

✓ Cards wrap responsively. ✓ Equal sizing. ✓ Centered layout without media queries.

14 / 16

Walk through the solution. Emphasize that max-width prevents cards from growing too wide on large screens — a common issue with flex-basis set to a percentage.

⚠️ Common Pitfalls

Forgot display: flex — Applying justify-content or align-items to a non-flex container does nothing. Always start with display: flex.
Mixing px and % in gap — gap only accepts absolute units (px, em, rem) or % (which is relative to container, not individual items).
flex-direction: column + height — For vertical layouts, the container needs an explicit height or min-height for justify-content to distribute space.
Children not flexing — flex: 1 means "take equal share of remaining space." If there's no remaining space (because content already fills the row), items won't grow visibly.
15 / 16

Go through each pitfall slowly. The forgotten display:flex is the #1 mistake beginners make — they write justify-content on a div and wonder why nothing changes.

🎯 Key Takeaways

1. display: flex turns any container into a flex container. Its children become flex items that you can arrange in rows or columns.

2. justify-content controls main-axis spacing; align-items controls cross-axis alignment. Together they replace every centering hack.

3. flex-wrap: wrap + gap + flex: 1 1 <base> creates responsive grids without media queries.

4. Flexbox powers three universal patterns: navbar, card grid, and centered hero. Master these and you're ready for real UI work.
