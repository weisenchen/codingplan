The Meta-Pattern

All three problems share the same structure:

pattern
1. TRANSFORM  the input into a canonical form
2. HASH       store transformed values in a dict / set
3. SOLVE      look up complements / matches in the hash

3Sum

Transform: sort array
Hash: two-pointer scan (implicit hash by sorted order)
Solve: find triplets summing to 0

Group Anagrams

Transform: sort each string → canonical key
Hash: map key → list of anagrams
Solve: return grouped lists

Product Except Self

Transform: build prefix products
Hash: prefix array (memoized computation)
Solve: combine prefix × suffix per index

Two Sum (Easy)

Transform: none
Hash: value → index
Solve: look up target - n

Medium 3Sum

Problem: Given nums = [-1,0,1,2,-1,-4], return all unique triplets [nums[i], nums[j], nums[k]] where i ≠ j ≠ k and nums[i] + nums[j] + nums[k] == 0.

Approach

  1. Sort the array — O(n log n)
  2. For each i, use two pointers (l = i+1, r = n-1)
  3. If sum < 0 → move l up; if sum > 0 → move r down; if == 0 → record
  4. Skip duplicates at each level to avoid repeated triplets

Time: O(n²) · Space: O(1) or O(n) for sorting

3Sum is the quintessential two-pointer Medium. The key insight: sorting makes duplicate-handling trivial and enables the two-pointer sweep. Emphasize that skipping duplicates is what makes this "Medium" — without that, it's just nested loops.

3Sum — Code Walkthrough

pythondef threeSum(nums):
    nums.sort()                            # O(n log n)
    res = []

    for i in range(len(nums) - 2):
        if i > 0 and nums[i] == nums[i-1]:
            continue                       # skip duplicate i

        l, r = i + 1, len(nums) - 1
        while l < r:
            s = nums[i] + nums[l] + nums[r]
            if s < 0:
                l += 1
            elif s > 0:
                r -= 1
            else:
                res.append([nums[i], nums[l], nums[r]])
                l += 1
                r -= 1
                while l < r and nums[l] == nums[l-1]:
                    l += 1                 # skip duplicate l
                while l < r and nums[r] == nums[r+1]:
                    r -= 1                 # skip duplicate r
    return res

🔑 The duplicate-skipping lines are what trip most people up.

Walk through the code line by line. Trace with [-1,0,1,2,-1,-4] on a whiteboard. Show how sorting makes [-1, -1, 0, 1, 2, 4] and the pointers converge. Point out each duplicate-skip guard.

Medium Group Anagrams

Problem: Given strs = ["eat","tea","tan","ate","nat","bat"], group the anagrams together.

Approach

  1. Canonical form: sort each string alphabetically — "eat" → "aet"
  2. Use a hash map: canonical → list of original strings
  3. Return all values from the map

Why sorting works

Anagrams have the same letters → same sorted form. "eat" and "tea" both become "aet".

Optimization

Use a 26-char count tuple as key instead of sorting: (1,0,0,...,1,0,...). Avoids O(k log k) per string.

Time: O(n · k log k) sorting · O(n · k) counting · Space: O(n · k)

Group Anagrams is the classic "hash map with transformed key" problem. The sorted-string trick is elegant and simple. Mention the counting-array optimization for extra credit in interviews — it demonstrates algorithmic maturity.

Group Anagrams — Code

pythonfrom collections import defaultdict

def groupAnagrams(strs):
    groups = defaultdict(list)

    for s in strs:
        # Canonical form: sorted string
        key = ''.join(sorted(s))
        groups[key].append(s)

    return list(groups.values())

# Example run:
# "eat" -> "aet":  ["eat"]
# "tea" -> "aet":  ["eat", "tea"]
# "tan" -> "ant":  ["tan"]
# "ate" -> "aet":  ["eat", "tea", "ate"]
# "nat" -> "ant":  ["tan", "nat"]
# "bat" -> "abt":  ["bat"]
# Result: [["eat","tea","ate"], ["tan","nat"], ["bat"]]

🔑 8 lines. defaultdict(list) keeps the code clean.

Highlight how short and clean the solution is. defaultdict(list) is a Python interview trick worth knowing. Walk through the trace.

Medium Product of Array Except Self

Problem: Given nums = [1,2,3,4], return an array output where output[i] is the product of all elements except nums[i]. No division allowed.

Approach

  1. Forward pass: build prefix products — prefix[i] = product of nums[0..i-1]
  2. Backward pass: multiply by suffix products — track running suffix product
  3. Combine in-place for O(1) extra space (excluding output)
Why no division? The twist is that the array can contain zeros. Division would fail on zeros; the prefix/suffix approach handles zeros naturally.

Time: O(n) · Space: O(1) (excluding output array)

This is the most elegant of the three. The no-division constraint forces you to think differently. Walk through the two-pass intuition with a small example on the board.

Product Except Self — Code

pythondef productExceptSelf(nums):
    n = len(nums)
    res = [1] * n

    # Forward: prefix products
    prefix = 1
    for i in range(n):
        res[i] = prefix
        prefix *= nums[i]

    # Backward: multiply by suffix products
    suffix = 1
    for i in range(n - 1, -1, -1):
        res[i] *= suffix
        suffix *= nums[i]

    return res

# nums = [1, 2, 3, 4]
# After forward:  [1, 1, 2, 6]
# After suffix:   [24, 12, 8, 6]  ✓

🔑 The res[i] = prefix assignment before prefix *= nums[i] is the key ordering.

Trace the forward pass: res starts [1,1,1,1], after loop becomes [1, 1, 2, 6]. Then trace backward: suffix starts 1, multiply res[3]=6*1=6, suffix=4; res[2]=2*4=8, suffix=12; res[1]=1*12=12, suffix=24; res[0]=1*24=24, suffix=24. Final: [24,12,8,6].

The Unified Insight

All three rely on preprocessing the input (sorting, canonicalizing, prefix products) so that a simple scan or hash lookup finishes the job. The "Medium" part is knowing what transformation unlocks the solution.

This is the synthesis slide. The takeaway: when stuck on a Medium, ask yourself "what transformation would make this problem trivial?" The answer is usually sorting, hashing, or prefix computation.

3 Key Takeaways

1. Transform the input first

Sorting, canonical keys, and prefix arrays turn impossible problems into trivial loops. Ask: "What form would make this easy?"

2. Hash maps are your best friend

Every problem today used a hash map or implicit hash (two-pointer scan). If you need to look up something fast, dict is the answer.

3. Medium = Easy + one more step

Don't be intimidated by the Medium tag. Break it down: what 2-3 Easy patterns combine to solve it? Usually it's just Two Sum + sorting, or hash map + grouping.
