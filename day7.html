What is a List?

A list is an ordered, mutable collection of items. Think of it like a row of lockers — each locker has a number (index) and holds a value.

fruits = ["apple", "banana", "cherry"]
       # index:   0        1         2
mixed = [42, "hello", 3.14, True]
       # Lists hold ANY type — even mixed

💡 Key idea: Lists keep things in order. The first item stays at position 0 unless you change it. Unlike a set or dict, order is preserved.

The locker analogy matters — many learners confuse list indices with "the Nth item" (1-based). Emphasize that computer scientists start counting at 0.
5

Indexing — 0-Based Access

Grab any item by its position number (index). Python uses 0-based indexing.

colors = ["red", "green", "blue", "yellow"]
       # index:   0       1        2        3

print(colors[0])  # → red
print(colors[2])  # → blue

Negative Indexing

Count from the end with negative numbers:

print(colors[-1])  # → yellow  (last item)
print(colors[-2])  # → blue    (second from last)

⚠️ Gotcha: colors[4] raises IndexError — index out of range!

Negative indexing is a Python superpower. Most languages don't have it. Show that -1 is always the last item, regardless of list length.
6

Slicing — [start:end]

Extract a sublist (a "slice") using the colon syntax. Think: "start at A, go up to (but not including) B."

nums = [10, 20, 30, 40, 50]

print(nums[1:4])  # → [20, 30, 40]   (index 1,2,3 — NOT 4)
print(nums[:3])   # → [10, 20, 30]   (start defaults to 0)
print(nums[2:])    # → [30, 40, 50]   (end defaults to end)
print(nums[::-1])  # → [50, 40, 30, 20, 10]  (reverse!)

📐 Visual: [start:end] is like a fence — start is the first post included, end is the first post excluded. Slice [1:4] includes posts 1, 2, 3.

The "up to but not including" rule is the #1 confusion point. Draw the fencepost analogy on a whiteboard if you have one. Show that slicing never raises IndexError — out-of-bounds slices just return what's available.
7

Growing & Shrinking Lists

.append()

Add one item to the end:

tasks = []
tasks.append("study")
tasks.append("code")
print(tasks)  # → ['study', 'code']

.pop()

Remove & return the last item (or at an index):

tasks = ["a", "b", "c"]
last = tasks.pop()  # → 'c'
print(tasks)        # → ['a', 'b']

len() — How many items?

print(len(tasks))  # → 2

🧠 Memory trick: len() works on anything that holds items — lists, strings, tuples, dictionaries. It's Python's universal size tool.

append and pop are the two most common list-mutating methods. They form a stack (LIFO) when used together. Mention that pop(index) also works but is less common.

🛒 Walkthrough: Shopping List App

Let's build a program that lets us add, view, remove, and search items —
all using lists.

Type each step. Expected output shown after each block.

This is the hands-on part. Encourage everyone to open their editor and type along. Pause after each step.

Step 1: Create the List & Show Menu

# shopping.py — start here
shopping_list = []

while True:
    print("\n=== SHOPPING LIST ===")
    print("1. Add item")
    print("2. Show list")
    print("3. Remove last item")
    print("4. Quit")

    choice = input("Choose (1-4): ")

✅ Expected: Menu prints, waits for input. No error.

We start with an empty list and a while True loop — the same pattern from Day 6's number guesser. The menu gives users a clear interface.
10

Step 2: Add & Show Items

    if choice == "1":
        item = input("Item to add: ")
        shopping_list.append(item)
        print(f"'{item}' added!")

    elif choice == "2":
        if len(shopping_list) == 0:
            print("List is empty.")
        else:
            for i, item in enumerate(shopping_list):
                print(f"{i+1}. {item}")

✅ Expected: Add "milk" → prints "'milk' added!". Show list → "1. milk".

enumerate gives us both index and value. We use i+1 so the display is 1-based for users, even though the list is 0-based internally. This is a common pattern.
11

Step 3: Remove, Quit & Search

    elif choice == "3":        # Remove last
        if shopping_list:
            removed = shopping_list.pop()
            print(f"Removed '{removed}'")
        else:
            print("Nothing to remove!")

    elif choice == "4":
        print("Goodbye!")
        break

    else:
        print("Invalid choice, try again.")

✅ Expected: Full program runs. Add items, view them, pop the last one, quit.

Note the guard: pop() on an empty list raises IndexError. We check 'if shopping_list:' first — empty lists are falsy in Python. The break keyword exits the while loop.

🎯 Bonus: Search with Slicing

After choice handling, add a search feature using in and slicing:

    elif choice == "5":        # Search (bonus option)
        query = input("Search for: ").lower()
        results = [item for item in shopping_list
                   if query in item.lower()]
        if results:
            print("Found:", results)
        else:
            print("No matches.")

This uses a list comprehension — we cover that formally in Week 3, but see if you can read it now!

🧪 Quick Check

What does this code print?

nums = [5, 10, 15, 20, 25]
print(nums[-2])
print(nums[1:3])
print(len(nums))
20
[10, 15]
5

Explanation: [-2] = second from last (20). [1:3] = items at index 1 and 2 (10, 15). len() = 5 items total.

Quick multiple-choice style check. If anyone got it wrong, re-explain the fencepost rule for slicing.

📝 Summary — 5 Key Takeaways

  • Create: my_list = [item1, item2] or [] for empty
  • Index: Zero-based list[0] for first; list[-1] for last
  • Slice: list[start:end] — includes start, excludes end. Misses last item!
  • Mutate: .append(x) adds at end, .pop() removes from end
  • Measure: len(list) tells you how many items you have

📌 For your toolbox: Lists are Python's most versatile data structure. Everything you learn from here — loops, functions, file I/O — builds on lists.
