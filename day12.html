✍️ Exercise 1 — List Utilities (8 min)

Re-type these three functions from memory. No IDE help. Expected output shown inline.

Functions to re-type

# 1. Remove duplicates (preserve order)
def unique(items: list) -> list:
    seen = []
    for item in items:
        if item not in seen:
            seen.append(item)
    return seen

# 2. Flatten a nested list
def flatten(nested: list) -> list:
    result = []
    for sub in nested:
        result.extend(sub)
    return result

# 3. Find most frequent element
def most_common(items: list):
    return max(set(items), key=items.count)

Expected output

print(unique([3,1,2,1,3,3,2]))
# → [3, 1, 2]

print(flatten([[1,2],[3,4,5],[6]]))
# → [1, 2, 3, 4, 5, 6]

print(most_common([1,3,2,3,4,3,2]))
# → 3
Walk through the room. Common pitfalls: using list(set()) which doesn't preserve order, forgetting .extend vs .append, using .count in a loop (O(n²)) -- fine for small lists but mention it.

✍️ Exercise 2 — Dictionary Workouts (8 min)

Re-type these dict-focused functions from scratch.

# 4. Word frequency counter
def word_counts(words: list) -> dict:
    counts = {}
    for w in words:
        counts[w] = counts.get(w, 0) + 1
    return counts

# 5. Invert a dictionary (values → keys)
def invert_dict(d: dict) -> dict:
    return {v: k for k, v in d.items()}

# 6. Merge two dicts (later values win)
def merge_dicts(a: dict, b: dict) -> dict:
    result = a.copy()
    result.update(b)
    return result

🎯 Check your output

word_counts(["a","b","a","c","b","a"]) → {'a': 3, 'b': 2, 'c': 1}
invert_dict({"x": 1, "y": 2}) → {1: 'x', 2: 'y'}
merge_dicts({"a": 1}, {"b": 2, "a": 3}) → {'a': 3, 'b': 2}

If invert_dict with duplicate values trips them up, note it — that's an undergrad interview classic. Also mention dict unpacking ( {**a, **b} ) as Python 3.9+ shorthand.

📇 List Methods — Flashcards

list.append(x)
Adds x to the end of the list. a=[1]; a.append(2) → [1,2]
list.extend(iterable)
Adds each element of iterable. a=[1]; a.extend([2,3]) → [1,2,3]
list.insert(i, x)
Inserts x at index i. a=[1,3]; a.insert(1,2) → [1,2,3]
list.pop(i=-1)
Removes & returns item at index i (default: last).
list.remove(x)
Removes the first occurrence of x. Raises ValueError if absent.
list.sort(key=None, reverse=False)
Sorts the list in-place. Returns None. Use sorted() for a new list.
list.reverse()
Reverses in-place. Returns None. Use reversed() for an iterator.
list.index(x, start, end)
Returns index of first x. [5,7,9].index(7) → 1. Raises ValueError.
list.count(x)
Returns number of occurrences. [1,2,2,3].count(2) → 2
list.copy()
Shallow copy. Equivalent to a[:]. b = a.copy()
list.clear()
Removes all elements. a.clear() → []. Same as a[:] = [].
len(list), del list[i]
len() returns item count. del removes by index (no return).
Ask: which of these have you never used in practice? .insert and .copy are common blind spots. Mention that O(n) removal in the middle of large lists is a red flag — consider deque or list comprehension filter instead.

📇 Dict Methods — Flashcards

d.get(k, default=None)
Returns d[k] if key exists, else default. No KeyError. Essential for counters.
d.setdefault(k, default)
Sets d[k] = default if key missing, returns d[k] either way.
d.keys(), d.values(), d.items()
Return view objects (dynamic). Wrap in list() if you need copies.
d.pop(k, default)
Removes key & returns value. Returns default if key missing.
d.update(other_dict)
Merges keys from other_dict into d (overwrites existing).
d.copy()
Shallow copy. Use deepcopy from copy module for nested dicts.
dict.fromkeys(iterable, value)
Class method. dict.fromkeys(['a','b'], 0) → {'a':0,'b':0}
d \|= other_dict
Python 3.9+ merge operator. d \|= {"x": 1} — in-place.
key in d
O(1) membership check. "name" in person — much faster than list.
len(d)
Number of key-value pairs. len({"a":1,"b":2}) → 2
Dict views are a common interview topic. Mention that they reflect changes to the dict in real-time. Also mention defaultdict from collections as a cleaner alternative to setdefault/get patterns.

🔎 Solution — Group by Length

Read through only after you've tried writing it yourself.

from collections import defaultdict

def group_by_length(words: list) -> dict:
    groups = defaultdict(list)
    for w in words:
        groups[len(w)].append(w)
    for length in groups:
        groups[length].sort()
    return dict(groups)  # optional: convert back to plain dict

# One-liner alt (same logic):
def group_by_length_v2(words):
    return {
        k: sorted(v)
        for k, v in defaultdict(list,
            ((len(w), w) for w in words)
        ).items()
    }
Key insight: The defaultdict(list) pattern appears in ~60% of interview questions involving grouping. Memorise it as "auto-vivifying list factory".
Discuss the trade-off between readability and conciseness. For coding interviews, the first version is safer — it's obvious what's happening. The one-liner is a nice flex but risky if you stumble.
