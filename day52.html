The Hash Map Superpower

A hash map (Python dict) maps keys → values with O(1) average-time lookups, inserts, and deletes.

The key insight: trade memory for speed. Instead of scanning a list (O(n)), remember what you've seen (O(1)).

❌ Without hash map

# Check if array has duplicates def has_duplicates(nums): for i in range(len(nums)): # O(n²) for j in range(i+1, len(nums)): if nums[i] == nums[j]: return True return False

← scans the whole list for each element

✅ With hash map

# Check if array has duplicates def has_duplicates(nums): seen = {} # O(1) lookups for n in nums: if n in seen: # O(1) ✔ return True seen[n] = True return False # O(n) total

← each lookup is instant

Rule of thumb: need to find something in a list repeatedly? Build a hash map first.

Frequency Counter Pattern

The most common hash-map pattern: count occurrences of elements in a collection.

Three ways to build one

# 1. Plain dict freq = {} for c in s: freq[c] = freq.get(c, 0) + 1 # 2. collections.defaultdict from collections import defaultdict freq = defaultdict(int) # missing keys default to 0 for c in s: freq[c] += 1 # 3. collections.Counter (most concise) from collections import Counter freq = Counter(s) # one-liner!

Interview tip: start with Counter for frequency problems — it's idiomatic, readable, and built-in. Fall back to defaultdict(int) for more complex logic.

[n] Notes
Walk through all three patterns. Counter is the Pythonic choice for "count everything" problems. defaultdict(int) is better when you need to conditionally update counts. Plain dict is fine but verbose — use get() if going this route.

Valid Anagram LeetCode #242 · Easy

Problem: Given two strings s and t, return True if t is an anagram of s.

from collections import Counter def is_anagram(s: str, t: str) -> bool: # Shortcut: different lengths can't be anagrams if len(s) != len(t): return False # Count characters in both strings freq_s = Counter(s) freq_t = Counter(t) # If same characters with same counts → anagram return freq_s == freq_t # Counter supports == directly!

Time: O(n) · Space: O(k) where k = distinct characters (≤ 26 for lowercase)

💡

Alternative: sort both strings and compare — O(n log n). The hash-map version is faster and more general (works for any character set).

The Two-Sum Pattern

Idea: For each element x, we need to find target - x somewhere in the array. A hash map stores numbers we've seen → their indices.

def two_sum(nums: list[int], target: int) -> list[int]: seen = {} # value → index for i, n in enumerate(nums): complement = target - n # what we need to find if complement in seen: # O(1) lookup ✨ return [seen[complement], i] seen[n] = i # remember this value return [] # no solution found

Time: O(n) — single pass · Space: O(n) — store up to n entries

🔑 The complement pattern is LeetCode #1 and the most-asked coding interview question on the planet. Nail this.

6 / ?
[n] Notes
Two-sum is the "hello world" of hash-map problems. The critical insight: we check the complement BEFORE storing the current number. Why? Because we can't use the same element twice. Also note: we return indices, not values. Interviewers sometimes ask "return values instead" or "return all pairs" — be ready to adapt.

Two-Sum Variations

The same complement-trick generalizes to many problems:

Two-sum (sorted)

# Two-pointer (no extra space) left, right = 0, len(nums)-1 while left < right: total = nums[left] + nums[right] if total == target: return [left, right] elif total < target: left += 1 else: right -= 1

O(n) time, O(1) space

Three-sum

# Fix one, two-sum the rest nums.sort() for i in range(len(nums)): if i > 0 and nums[i]==nums[i-1]: continue # skip dupes left, right = i+1, len(nums)-1 while left < right: s = nums[i] + nums[left] + nums[right] ...

O(n²) time, O(1) or O(n) space

For now, master two-sum. Three-sum is a Week 10+ problem.

First Non-Repeating Character LeetCode #387 · Easy

Problem: Find the first character in a string that doesn't repeat. Return its index, or -1 if none.

from collections import Counter def first_uniq_char(s: str) -> int: # Pass 1: count every character freq = Counter(s) # Pass 2: find first with count == 1 for i, ch in enumerate(s): if freq[ch] == 1: return i return -1

Time: O(n) · Space: O(1) — at most 26 lowercase letters

💡

A two-pass algorithm: first build the frequency map, then scan again. This is a recurring DSA pattern — you can't know what's non-repeating until you've seen everything.

📌 Three Takeaways

1
Hash maps give O(1) lookups. Any time you need to repeatedly check membership, build a dict/set. It's the difference between O(n²) and O(n).
2
Frequency counters = counting things. Use Counter or defaultdict(int). Valid Anagram and First Non-Repeating Char are the classic examples.
3
The complement pattern powers Two-Sum. For each n, check if target - n has been seen. This trick generalizes to many hash-map problems.
