What Is an API?

API = Application Programming Interface. Think of it as a waiter in a restaurant.

You (client)
curl / browser / app
You place an order (request)
Waiter (API)
FastAPI server
Takes your order to the kitchen
Kitchen (AI)
Google Gemini
Prepares the response
Your table
JSON response
Food arrives!

When you send a message to our chatbot:

You: "What is co-op?"
  → POST to http://localhost:8000/chat
  → FastAPI forwards to Gemini AI
  → Gemini replies
  → FastAPI sends back: {"reply": "Co-op is..."}
You read the reply in your terminal/browser

HTTP Methods: GET vs POST

APIs speak HTTP. Two methods matter for this project.

Method What it does Like
GETFetch data, no side effectsLooking at a menu
POSTSend data, get something backOrdering food

Our chatbot will use POST — you send a message, the AI replies.

# GET request — asking without sending data
curl http://localhost:8000/

# POST request — sending a message, getting a reply
curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "Hello!"}'

What Is a Virtual Environment?

A venv is a self-contained folder that holds Python + all the packages for one specific project.

Without venv
pip install flask
Installs into your global Python. Project A and B share the same packages. Upgrade for A? Might break B.
With venv
pip install flask
Installs inside the project folder. Each project has its own packages. No conflicts, ever.
Think of it like a suitcase:

You pack only the clothes (packages) you need for this trip (project). When you switch projects, you pack a different suitcase.

Step 2 — Activate it

After activation, you'll see (venv) at the beginning of your terminal prompt. That means you're inside the environment.
Step 3 — Install packages (while activated)

Step 4 — When you're done

Next time: cd project-folder && source venv/bin/activate

Your First FastAPI App (5 Lines!)

Create a file called main.py and write this:

# main.py
from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def home():
    return {"status": "alive", "project": "Gemini Chatbot"}

Now run it:

You'll see:

INFO:     Press CTRL+C to stop

Open your browser and go to http://localhost:8000 — you should see JSON!
Also try http://localhost:8000/docs — FastAPI gives you a free Swagger UI.

Understanding the Code

Let's read your first FastAPI script line by line.

# main.py
from fastapi import FastAPI   # 1. Import the framework

app = FastAPI()                 # 2. Create an app instance

@app.get("/")                   # 3. Decorator: "when someone visits /, run this"
def home():                    # 4. Define the function
    return {"status": "alive"}  # 5. Return JSON (Python dict = JSON)
🔑 Key concepts:
  • @app.get("/") — this is a route. It listens for GET requests at /
  • def home() — this is a path operation function. Run when the route is hit
  • return {"key": "value"} — FastAPI automatically converts Python dict → JSON
  • --reload flag — auto-restarts the server when you change the code (development only)

💡 Stop the server with Ctrl+C. Restart it whenever you change main.py (or keep --reload on).

Adding a POST Route

For the chatbot, the client sends a message. That's a POST request with a JSON body.

Add this to main.py:

from pydantic import BaseModel

# Define the shape of incoming data
class ChatRequest(BaseModel):
    message: str

@app.post("/chat")
def chat(request: ChatRequest):
    return {"reply": f"You said: {request.message}"}
🔑 What's new:
  • BaseModel — a Pydantic model. It validates that the client sent a message field with text
  • request: ChatRequest — FastAPI reads the JSON body and converts it into a Python object
  • request.message — access the message field the client sent

Test it (in a new terminal window while the server is running):

curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "Hello chatbot!"}'

Expected response: {"reply": "You said: Hello chatbot!"}

Write the Full Chatbot Code

Now replace your entire main.py with this — it connects FastAPI to Gemini.

from fastapi import FastAPI
from pydantic import BaseModel
from google import genai
from dotenv import load_dotenv
import os

load_dotenv()                                  # load .env file

# Configure Gemini
genai.configure(api_key=os.getenv("GEMINI_API_KEY"))
model = genai.GenerativeModel("gemini-2.5-flash")

app = FastAPI()

class ChatRequest(BaseModel):
    message: str

@app.post("/chat")
def chat(request: ChatRequest):
    response = model.generate_content(request.message)
    return {"reply": response.text}

That's it! 17 lines — a fully functional AI chatbot API.

🔑 What each line does:
  • load_dotenv() — reads your API key from .env
  • genai.configure(...) — tells the SDK your key
  • GenerativeModel("gemini-2.5-flash") — picks the AI model
  • model.generate_content(msg) — sends message to Gemini, gets reply
  • response.text — extracts the AI's response text

One More Thing: CORS

If you want to call your chatbot from a web page (not just curl), you need CORS.

What is CORS?

Browsers block JavaScript from one website talking to another. CORS is a header that says "it's OK, let them through."

Add this to main.py (put it after app = FastAPI()):

from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],       # allow any website (for development)
    allow_methods=["*"],       # allow any HTTP method
    allow_headers=["*"],       # allow any header
)

💡 For production, replace "*" with your actual frontend URL.

Troubleshooting

"pip: command not found"

Try python3 -m pip install ... instead of pip install ...

"ModuleNotFoundError: No module named 'google'"

Run pip install google-genai. Make sure you installed it (check pip list).

"API key not found" or 401 error

Check your .env file. Is GEMINI_API_KEY spelled correctly? Is the file in the same folder as main.py?

"Address already in use"

Port 8000 is busy. Either stop the other process (Ctrl+C) or use a different port: uvicorn main:app --reload --port 8001

curl returns "Connection refused"

Is Uvicorn still running? Check the terminal. Did you stop it with Ctrl+C? Restart it.

Still stuck?

Copy the exact error message into Google. Stack Overflow has the answer for 99% of errors.
