Foundation

What Is Big O?

The Core Idea

Big O answers one question: how does the runtime grow as the input gets larger?

We don't care about exact milliseconds or CPU cycles. Those change with hardware. We care about the rate of growth — the shape of the curve when you plot input size vs. runtime.

Flat lines (O(1)) are ideal. Steep curves (O(n²)) are expensive.

The Rules of Big O

Big O describes the worst-case scenario and focuses on the dominant term:

1. Drop Constants

O(2n) → O(n)
O(n/2) → O(n)

Constant factors don't change the growth rate.

2. Keep the Biggest Term

O(n² + n) → O(n²)
O(n + log n) → O(n)

For large n, the smaller terms don't matter.

⚠️ Interview tip: Always state the simplified form. Saying "O(2n)" marks you as a beginner.
The Big Four

O(1) · O(n) · O(n²) · O(log n)

O(1) — Constant Time

The fastest category. Runtime does not depend on input size. Whether the array has 10 items or 10 million — same speed.

Examples

def get_first(arr): return arr[0] def lookup(dict, key): return dict[key] def is_even(n): return n % 2 == 0

Key Insight

O(1)
Constant Time
array[0], dict[key], arithmetic

One operation, always. No loops, no recursion — just a direct computation or lookup.

O(n) — Linear Time

Runtime grows proportionally with input size. Double the input → double the time. A single loop that visits every element.

Examples

def find_max(arr): max_val = arr[0] for n in arr: if n > max_val: max_val = n return max_val def sum_list(arr): total = 0 for n in arr: total += n return total

Pattern

O(n)
Linear Time
Single loop over n elements

One non-nested loop that iterates over the input = O(n).

O(n²) — Quadratic Time

Runtime grows with the square of input size. This happens when you have nested loops — for every element, you loop over every element again.

Examples

def has_duplicates(arr): for i in range(len(arr)): for j in range(len(arr)): if i != j and \ arr[i] == arr[j]: return True return False # Also O(n²): def print_pairs(arr): for x in arr: for y in arr: print(x, y)

Pattern

O(n²)
Quadratic Time
Nested loops over n elements

Warning: For n=10,000, O(n²) does 100 million operations. Avoid for large inputs.

O(log n) — Logarithmic Time

Runtime grows slowly. Each step cuts the problem size in half. Even for huge inputs, the number of steps stays tiny.

Binary Search

def binary_search(arr, target): left, right = 0, len(arr) - 1 while left <= right: mid = (left + right) // 2 if arr[mid] == target: return mid elif arr[mid] < target: left = mid + 1 else: right = mid - 1 return -1

How Fast?

O(log n)
Logarithmic Time
n=1M → ~20 steps

Searching 1 million items takes only ~20 checks! Key requirement: input must be sorted.

Complexity at a Glance

How the four complexities compare for real-world input sizes:

O(1)
Constant
n=10 → 1 op · n=1M → 1 op
O(log n)
Logarithmic
n=10 → ~3 ops · n=1M → ~20 ops
O(n)
Linear
n=10 → 10 ops · n=1M → 1M ops
O(n²)
Quadratic
n=10 → 100 ops · n=1M → 1T ops

O(n²) at 1 million inputs is 1 trillion operations. That's the difference between 1 second and 31,000 years.

Hands-On

Counting Operations in Loops

Loop Patterns — Walkthrough

Let's trace through common loop patterns and identify their Big O:

Single loop

for i in range(n): print(i)

→ O(n)

Two separate loops

for i in range(n): print(i) for j in range(n): print(j)

→ O(2n) → O(n)

Nested loops

for i in range(n): for j in range(n): print(i, j)

→ O(n²)

Halving each step

n = len(arr) while n > 0: print(n) n //= 2

→ O(log n)

Common Traps

Nested with different bounds

for i in range(n): for j in range(m): print(i, j)

→ O(n × m)

If n ≈ m, that's O(n²). If n is a constant (like 26 letters), O(m).

Loop slicing the input

for i in range(n): for j in range(i, n): print(i, j)

→ O(n²/2) → O(n²)

Half the iterations, but n²/2 still simplifies to O(n²).

⚠️ Off-by-one trap: for j in range(min(1000, n)) — if n is small, this looks O(1). But the intent matters. Always ask: "What dominates for large n?"

🎯 Key Takeaways

1

Big O measures growth rate, not speed. It tells you how runtime scales as input grows — O(1) is flat, O(n) is proportional, O(n²) explodes, O(log n) barely climbs.

2

Simplify ruthlessly. Drop constants (O(2n) → O(n)) and keep only the largest term (O(n² + n) → O(n²)).

3

Patterns map to complexities. Single loop → O(n). Nested loops → O(n²). Halving each step → O(log n). Direct access → O(1). Learn the pattern and you can analyze any function on sight.
