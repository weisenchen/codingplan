🏗️ CREATE TABLE — Defining Your Schema

A table has a name and columns. Each column has a name, a type, and optional constraints.

Basic syntax:

CREATE TABLE table_name (
    column1 TYPE constraints,
    column2 TYPE constraints,
    ...
);

Example — a projects table:

CREATE TABLE projects (
    id          INTEGER PRIMARY KEY,
    name        TEXT    NOT NULL,
    description TEXT,
    deadline    TEXT,
    completed   INTEGER DEFAULT 0
);

📥 INSERT — Adding Data

Add rows one at a time (or many at once).

INSERT a single row:

INSERT INTO projects (name, description, deadline, completed)
VALUES ('CLI Todo List', 'A command-line task manager',
        '2026-06-10', 1);

INSERT multiple rows:

INSERT INTO projects (name, description, deadline)
VALUES
    ('Bookmark Saver', 'Save and tag bookmarks from CLI', '2026-07-05'),
    ('Weather Dashboard', 'API-driven weather app',      '2026-07-19'),
    ('Full-Stack App', 'Complete web app with auth',      '2026-09-06');

Tip: Column order in INSERT INTO must match the VALUES order. Omitted columns get NULL or their DEFAULT.

🔍 SELECT — Asking Questions

The most-used SQL command. You SELECT columns you want FROM a table.

All columns:

SELECT * FROM projects;

Specific columns:

SELECT name, deadline FROM projects;

With WHERE filter:

SELECT name FROM projects
WHERE completed = 1;

ORDER BY:

SELECT name, deadline
FROM projects
ORDER BY deadline DESC;

LIMIT:

SELECT * FROM projects
LIMIT 2;

Combined:

SELECT name, deadline
FROM projects
WHERE completed = 0
ORDER BY deadline
LIMIT 2;

💻 Walkthrough: Your First SQL Session

Type each step in your terminal:

1

Launch SQLite with a new database file:

$ sqlite3 coop_prep.db

You're now in the SQLite shell (sqlite> prompt).

2

Create a table:

sqlite> CREATE TABLE projects (
   ...>     id          INTEGER PRIMARY KEY,
   ...>     name        TEXT    NOT NULL,
   ...>     description TEXT,
   ...>     deadline    TEXT,
   ...>     completed   INTEGER DEFAULT 0
   ...> );
3

Verify with .tables and .schema:

sqlite> .tables
projects
sqlite> .schema projects
-- CREATE TABLE projects (
--     id INTEGER PRIMARY KEY,

--     deadline TEXT,
--     completed INTEGER DEFAULT 0
-- );

✏️ Practice Exercises

Try these on your own. Spend ~20 minutes.

Exercise 1: Create a applications table

Columns: id (INTEGER PK), company (TEXT NOT NULL), role (TEXT), date_applied (TEXT), status (TEXT DEFAULT 'applied'). Insert 3 rows: one each for a company you'd apply to.

Hint: CREATE TABLE applications (...) then INSERT INTO applications (company, role) VALUES ('Google', 'SWE Intern');

Exercise 2: Query practice

Write a query that shows all applications sorted by date_applied newest-first. Then write one that shows only the 2 oldest applications.

Hint: ORDER BY date_applied DESC and LIMIT 2

Exercise 3 (Challenge): Combined query

Show all applications where status = 'applied', ordered by date_applied ascending, limited to 1 result.

💡 Exit SQLite with .quit

✅ Exercise Solutions

Exercise 1 — Create & Insert

CREATE TABLE applications (
    id          INTEGER PRIMARY KEY,
    company     TEXT    NOT NULL,
    role        TEXT,
    date_applied TEXT,
    status      TEXT    DEFAULT 'applied'
);
INSERT INTO applications (company, role, date_applied)
VALUES
    ('Google',       'SWE Intern',     '2026-06-15'),
    ('Stripe',       'Backend Intern', '2026-06-16'),
    ('Shopify',      'Data Intern',    '2026-06-17');

Exercise 2 — Sort & Limit

-- Newest first
SELECT * FROM applications ORDER BY date_applied DESC;
-- Two oldest
SELECT * FROM applications ORDER BY date_applied LIMIT 2;

Exercise 3 — Filtered + Sorted + Limited

SELECT company, role, date_applied
FROM applications
WHERE status = 'applied'
ORDER BY date_applied
LIMIT 1;

📌 Key Takeaways

1

CREATE TABLE defines your schema — columns, types, constraints. Get this right and everything else follows.

2

INSERT INTO adds rows. Always name the columns; omit for auto-increment PK and defaults.

3

SELECT with WHERE + ORDER BY + LIMIT is your core query pattern. Practice combining them until it's automatic.

4

SQLite is your learning sandbox. It's everywhere, zero-setup, and SQL you write here transfers to PostgreSQL, MySQL, etc.
