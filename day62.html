🔗 Problem 1: Merge Two Sorted Lists

Given two sorted linked lists, merge them into one sorted list

Approach: Dummy Node + Two Pointers

  • Create a dummy head node — avoids edge cases
  • Compare l1.val vs l2.val, append the smaller
  • Advance the pointer of the chosen list
  • When one list ends, attach the remainder

Time: O(n + m) Space: O(1)

💻 Merge Two Sorted Lists — Code

def mergeTwoLists(l1: ListNode, l2: ListNode) -> ListNode:
    dummy = ListNode()          # sentinel
    curr = dummy

    while l1 and l2:
        if l1.val < l2.val:
            curr.next = l1
            l1 = l1.next
        else:
            curr.next = l2
            l2 = l2.next
        curr = curr.next

    # Attach remaining nodes
    curr.next = l1 or l2
    return dummy.next

Key insight: The dummy node eliminates the need to handle head is None as a special case. curr.next = l1 or l2 handles both remainder cases in one line.

🔢 Problem 2: Contains Duplicate

Return true if any value appears at least twice in the array

Hash Set Pattern

The brute-force O(n²) checks every pair. The smart approach:

  • Iterate once, track seen numbers in a hash set
  • If a number is already in the set → duplicate found
  • Otherwise add it and continue

Time: O(n) Space: O(n)

Code

def containsDuplicate(nums: List[int]) -> bool:
    seen = set()
    for n in nums:
        if n in seen:
            return True
        seen.add(n)
    return False

🔤 Problem 3: First Unique Character

Find the first non-repeating character in a string; return its index

Two-Pass Frequency Map

  • Pass 1: Build a frequency dict (char → count)
  • Pass 2: Scan left to right, return first char with count == 1
  • Return -1 if none found

Time: O(n) Space: O(1) (26 letters)

Code

def firstUniqChar(s: str) -> int:
    freq = {}
    for ch in s:
        freq[ch] = freq.get(ch, 0) + 1
    for i, ch in enumerate(s):
        if freq[ch] == 1:
            return i
    return -1

Interview tip: For lowercase-only strings, use an array of size 26 instead of a dict — slightly faster, same logic.

⚡ First Unique Char — Array Variant

Same logic, constant-space array for lowercase Latin letters

def firstUniqChar(s: str) -> int:
    freq = [0] * 26
    for ch in s:
        freq[ord(ch) - 97] += 1
    for i, ch in enumerate(s):
        if freq[ord(ch) - 97] == 1:
            return i
    return -1

Pattern: When the input domain is small and bounded (26 letters, 128 ASCII), an array beats a hash map on constant factors — less hashing overhead, better cache locality.

🧗 Problem 4: Climbing Stairs

You climb n steps. Each time you can take 1 or 2 steps. How many distinct ways?

It's Fibonacci in Disguise

To reach step n, you must have come from either:

  • Step n-1 (1 step), or
  • Step n-2 (2 steps)

So ways[n] = ways[n-1] + ways[n-2]

Base: ways[1] = 1, ways[2] = 2

Time: O(n) Space: O(1)

Bottom-Up DP (O(1) space)

def climbStairs(n: int) -> int:
    if n <= 2:
        return n
    one, two = 1, 2
    for i in range(3, n + 1):
        curr = one + two
        one, two = two, curr
    return two

🧠 Climbing Stairs — Why DP Works

Memoization vs Tabulation

  • Top-down (memo): recursive + cache — easier to reason about, uses O(n) call stack
  • Bottom-up (tabulation): iterative, O(1) space — always preferred in interviews

The problem is a classic intro to DP because the recurrence is obvious from the problem statement itself.

Walkthrough: n = 5

n=1 → 1
n=2 → 2
n=3 → 1+2 = 3
n=4 → 2+3 = 5
n=5 → 3+5 = 8  ✅

The sequence: 1, 2, 3, 5, 8, 13, 21...

📈 Problem 5: Best Time to Buy and Sell Stock

Given daily prices, find max profit from one buy + one sell

Greedy Single Pass

  • Track min price seen so far
  • For each price, compute profit = price - min_price
  • Keep max profit seen
  • Update min_price when a lower price appears

Time: O(n) Space: O(1)

Code

def maxProfit(prices: List[int]) -> int:
    min_price = float('inf')
    max_profit = 0
    for p in prices:
        if p < min_price:
            min_price = p
        elif p - min_price > max_profit:
            max_profit = p - min_price
    return max_profit

Modify Climbing Stairs so you can take 1, 2, or 3 steps at a time. What's the recurrence?

def climbStairs3(n: int) -> int:
    if n <= 2: return n
    if n == 3: return 4
    a, b, c = 1, 2, 4
    for i in range(4, n + 1):
        a, b, c = b, c, a + b + c
    return c

Recurrence: ways[n] = ways[n-1] + ways[n-2] + ways[n-3]

3. Merge Multiple Sorted Lists

Given k sorted linked lists, merge them into one. Hint: use a min-heap (priority queue).

Show hint 
import heapq

def mergeKLists(lists: List[ListNode]) -> ListNode:
    heap = []
    for i, l in enumerate(lists):
        if l:
            heapq.heappush(heap, (l.val, i, l))
    dummy = ListNode()
    curr = dummy
    while heap:
        _, i, node = heapq.heappop(heap)
        curr.next = node
        curr = curr.next
        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))
    return dummy.next

🎯 Key Takeaways

Write these down — they'll save you in interviews

Patterns

  • Dummy node for linked list construction — eliminates edge cases
  • Hash set/ map for duplicates, frequency, and lookup problems
  • Constant-space tracking (min_price, two variables) for greedy single-pass
  • Fibonacci recurrence for combinatorial DP (stairs, robot paths, etc.)

Watch Out For

  • Forgetting to attach remaining nodes in merge
  • Using O(n²) when O(n) with a hash set works
  • Writing recursive Fibonacci (stack overflow for large n)
  • Buy/sell: selling before buying (must be chronological)
