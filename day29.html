Concept · Objects

JavaScript Objects — Dot Notation

Objects store key-value pairs. Keys are strings (called properties), values can be any type.

// Create an object literal
const book = {
  title:  "The Pragmatic Programmer",
  author: "Hunt & Thomas",
  pages:  352,
  isRead: false
};

// Access with dot notation
console.log(book.title);     // "The Pragmatic Programmer"
console.log(book.pages);    // 352

// Modify a property
book.isRead = true;

// Add a new property
book.rating = 4.8;

Dot notation is clean and preferred — when you know the property name at write time.

Concept · Objects

Bracket Notation — Dynamic Keys

Use bracket notation when the property name is dynamic, computed, or has special characters.

Dot notation

const user = { name: "Alice" };
user.name;   // "Alice"

// ❌ Fails with dynamic key
let key = "name";
user.key;    // undefined

Bracket notation

const user = { name: "Alice" };
user["name"]; // "Alice"

// ✅ Dynamic key from variable
let key = "name";
user[key];   // "Alice"

Also required for keys with spaces / hyphens: user["first name"]

Concept · JSON

JSON — JavaScript Object Notation

JSON is a lightweight text format for storing and transmitting data. It looks like a JS object but all keys must be in double quotes.

JSON.stringify()

Convert JS object → JSON string

const obj = { name: "Alice", age: 25 };
const json = JSON.stringify(obj);
// '{"name":"Alice","age":25}'

console.log(typeof json);
// "string" ← it's text now!

JSON.parse()

Convert JSON string → JS object

const json = '{"name":"Bob","age":30}';
const obj = JSON.parse(json);

console.log(obj.name);
// "Bob" ← it's an object again!

💡 This is how data travels across the internet — APIs send JSON strings, your code parses them into objects.

Walkthrough

Fetching Data with fetch()

fetch(url) returns a Promise. We use await (inside async) to wait for the response.

// Get a random cat fact from a free API
async function getCatFact() {
  const res = await fetch(
    "https://catfact.ninja/fact"
  );
  const data = await res.json();
  console.log(data.fact);
}

getCatFact();
// → "A cat's tail contains nearly 10% of all..."

fetch(url) — sends the HTTP request

response.json() — parses the JSON body into a JS object

data — now you can access any property like data.fact

Walkthrough

Displaying API Data in the DOM

Bring fetch results into your page:

<!-- HTML -->
<button id="loadBtn">Get a Fact!</button>
<p id="factDisplay"></p>

<script>
  const btn = document.getElementById("loadBtn");
  const display = document.getElementById("factDisplay");

  btn.addEventListener("click", async () => {
    const res = await fetch("https://catfact.ninja/fact");
    const data = await res.json();
    display.textContent = data.fact;
  });
</script>

Click the button → fetch → parse → display. This is the core pattern of almost every web app.

Concept

Handling Fetch Errors

Network requests can fail. Always handle errors with try/catch:

async function getData(url) {
  try {
    const res = await fetch(url);
    if (!res.ok) {
      throw new Error(
        `HTTP ${res.status}: ${res.statusText}`
      );
    }
    const data = await res.json();
    return data;
  } catch (err) {
    console.error("Fetch failed:", err.message);
    display.textContent = "⚠️ Could not load data. Try again later.";
  }
}

Always check res.ok — fetch() only rejects on network failure, not on 404 or 500!
