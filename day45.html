📦 Rendering Lists with .map()

The bread-and-butter of dynamic UIs: transform an array of data into an array of JSX elements.

function FruitList() {
  const fruits = ['Apple', 'Banana', 'Cherry'];

  return (
    <ul>
      {fruits.map(fruit =>
        <li>{fruit}</li>
      )}
    </ul>
  );
}

How it works:

['Apple','Banana','Cherry'].map(fn)
→ [<li>Apple</li>, ...]

⚠️ You'll see a React warning — we need key. Next slide!

🔑 The key Prop — Why It Matters

React uses key to track which items changed, moved, or were removed. Without it, re-renders are buggy and slow.

✅ Correct

{items.map(item =>
  <li key={item.id}>
    {item.name}
  </li>
)}

❌ Bad

{items.map((item, i) =>
  <li key={i}>  // index!
    {item.name}
  </li>
)}

Rule: Use a stable, unique ID from your data. Never use array index unless the list never changes (no add/remove/reorder).

🔀 Conditional Rendering — Ternary &&

Show or hide parts of the UI based on state. Two essential patterns:

Ternary (? :)

{isLoggedIn ?
  'Welcome back!' :
  'Please sign in'
}

Use for either/or choices.

Logical AND (&&)

{hasError &&
  <div className="error">
    {errorMessage}
  </div>
}

Use for show/hide without an else.

⚠️ Gotcha: Don't use && with 0 or empty array — 0 && <Thing/> renders 0! Use !! or compare explicitly.

🛠️ Step 4–6: Filtering + Render

Step 4 — Filter logic:

const filtered = PRODUCTS.filter(p => {
  const matchSearch = p.name
    .toLowerCase()
    .includes(search.toLowerCase());

    true : p.category === category;

Steps 5–6 — Render:

<input value={search}
  onChange={e => setSearch(e.target.value)}
  placeholder="Search products..." />

<ul>
  {filtered.map(p =>
    <li key={p.id}>
      {p.name} — ${p.price}
    </li>
  )}
</ul>

💡 .filter() + .map() chained together — filter first, then render. Both return new arrays; original data is untouched.

🛠️ Step 7: Category Buttons & Empty State

Category selector:

{['All', 'Electronics', 'Clothing', 'Home'].map(c =>
  <button
    key={c}
    onClick={() => setCategory(c)}
    style={{ fontWeight:
      category === c ? 'bold' : 'normal' }}
  >
    {c}
  </button>
)}

Empty state (conditional):

{filtered.length === 0 ?
  <p>No products found.</p> :
  <ul>
    {filtered.map(p =>
      <li key={p.id}>{p.name}</li>
    )}
  </ul>
}

🔁 Categories are also rendered with .map() + key. The active category gets bold text via ternary styling.

📄 The Complete Component

function FilterableList() {
  const [search, setSearch] = useState('');
  const [category, setCategory] = useState('All');

  const filtered = PRODUCTS.filter(p => {
    const textMatch = p.name
      .toLowerCase()
      .includes(search.toLowerCase());
    const catMatch = category === 'All' ?
      true : p.category === category;
    return textMatch && catMatch;
  });

  return (
    <div>
      <input value={search} onChange={e=>setSearch(e.target.value)}
        placeholder="Search" />
      <div>
        {['All','Electronics','Clothing','Home'].map(c =>
          <button key={c} onClick={()=>setCategory(c)}
            style={{fontWeight: category===c?'bold':'normal'}}>{c}</button>
        )}
      </div>
      {filtered.length === 0 ?
        <p>No products found.</p> :
        <ul>{filtered.map(p =>
          <li key={p.id}>{p.name} — ${p.price}</li>
        )}</ul>
      }
    </div>
  );
}

🎯 Key Takeaways

1
Use .map() to convert data → JSX

Every dynamic list in React starts with array.map(fn) inside JSX.

2
Always provide a stable key

Use unique IDs from data, never array indices. React depends on keys for correct re-renders.

3
Ternary for either/or, && for show/hide

Chain filter + map for dynamic, filterable UIs. Empty states are conditional rendering too.
