Problem 1: Two Sum Easy

Prompt: Given an array of integers nums and an integer target, return indices of the two numbers that add up to target. You may assume exactly one solution exists.

⏱ Time yourself: 8 minutes max. No looking at the solution below until you're done (or stuck).

🟢 Brute Force

def two_sum_brute(nums, target):
    for i in range(len(nums)):
        for j in range(i + 1, len(nums)):
            if nums[i] + nums[j] == target:
                return [i, j]
# O(n²) time, O(1) space

⭐ Optimal (One-Pass Hash)

def two_sum(nums, target):
    seen = {}  # val → index
    for i, n in enumerate(nums):
        complement = target - n
        if complement in seen:
            return [seen[complement], i]
        seen[n] = i
# O(n) time, O(n) space

Pattern: "Need a pair" → hash map for O(1) lookups. Check complement before storing current.

Problem 2: Longest Substring W/O Repeating Medium

Prompt: Given a string s, find the length of the longest substring without repeating characters.

⏱ 8 minutes. Write it from memory, then check below.

⭐ Sliding Window + Char Set

def length_of_longest_substring(s: str) -> int:
    char_set = set()
    left = max_len = 0
    for right in range(len(s)):
        while s[right] in char_set:
            char_set.remove(s[left])
            left += 1
        char_set.add(s[right])
        max_len = max(max_len, right - left + 1)
    return max_len
# O(n) time, O(min(n, |alphabet|)) space

Pattern: "Contiguous subarray with constraint" → sliding window. Expand right, shrink left when constraint violated.

Edge case to test: "bbbbb" → 1  ·  "pwwkew" → 3

Problem 3: Reverse Linked List Easy

Prompt: Given the head of a singly linked list, reverse it and return the new head.

⏱ 8 minutes. First write iterative (must-know), then recursive (bonus).

⭐ Iterative (3 pointers)

def reverse_list(head):
    prev = None
    curr = head
    while curr:
        nxt = curr.next    # save
        curr.next = prev   # reverse
        prev = curr        # move prev
        curr = nxt         # move curr
    return prev            # new head
# O(n) time, O(1) space

Recursive

def reverse_list_rec(head):
    if not head or not head.next:
        return head
    new_head = reverse_list_rec(head.next)
    head.next.next = head
    head.next = None
    return new_head
# O(n) time, O(n) call stack space

Mnemonics: "Prev, curr, nxt — save, reverse, advance." Draw the arrow-flipping on paper.

Problem 4: Max Depth of Binary Tree Easy

Prompt: Given the root of a binary tree, return its maximum depth (number of nodes on the longest root-to-leaf path).

⏱ 8 minutes. Write BFS and DFS versions.

⭐ DFS (Recursive)

def max_depth(root):
    if not root:
        return 0
    return 1 + max(
        max_depth(root.left),
        max_depth(root.right)
    )
# O(n) time, O(h) space (h = height)

BFS (Level-order)

from collections import deque

def max_depth_bfs(root):
    if not root:
        return 0
    q = deque([root])
    depth = 0
    while q:
        for _ in range(len(q)):
            node = q.popleft()
            if node.left: q.append(node.left)
            if node.right: q.append(node.right)
        depth += 1
    return depth
# O(n) time, O(w) space (w = max width)

Pattern: Tree height → recursive DFS is simplest. BFS useful when you also need level info.

Problem 5: Climbing Stairs Easy

Prompt: You're climbing a staircase with n steps. Each time you can climb 1 or 2 steps. In how many distinct ways can you reach the top?

⏱ 8 minutes. Write the DP bottom-up solution, then the O(1) space optimization.

Bottom-Up DP

def climb_stairs(n: int) -> int:
    if n <= 2:
        return n
    dp = [0] * (n + 1)
    dp[1], dp[2] = 1, 2
    for i in range(3, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]
# O(n) time, O(n) space

⭐ Optimized (Fibonacci)

def climb_stairs_opt(n: int) -> int:
    if n <= 2:
        return n
    a, b = 1, 2
    for _ in range(3, n + 1):
        a, b = b, a + b
    return b
# O(n) time, O(1) space

Pattern: "Count ways with limited moves" → DP. Recurrence: dp[i] = dp[i-1] + dp[i-2] (Fibonacci).

🎯 Summary: Key Takeaways

Write these down. They're the three things to remember from today.

1. Hash Map is Your Swiss Army Knife

Two Sum, contains duplicate, anagram check, two-sum-variants — the O(1) lookup makes hash maps the most versatile tool for "remember what you've seen" problems.

2. Sliding Window Solves Subarray Problems

Learn the pattern: expand right, shrink left while constraint violated, update answer. Same structure works for longest substring, min window subarray, and anagrams.

3. Know Your Complexity Tables Cold

Interviewers expect you to justify every data structure choice with Big O. If you can't recite the access/search/insert/delete for arrays, linked lists, hash maps, and trees, make flashcards today.
