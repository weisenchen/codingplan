🧠 What Is a List Comprehension?

A concise way to create lists by applying an expression to every item in an iterable — without writing an explicit loop.

❌ Traditional loop

squares = []
for x in range(10):
    squares.append(x ** 2)

✅ Comprehension

squares = [x ** 2 for x in range(10)]
Why it matters

Comprehensions are 25-40% faster than manual for loops in CPython because they avoid the overhead of repeated .append() calls and use a dedicated opcode (LIST_COMP).

📐 Basic Syntax

Template

[expression
 for item
 in iterable]

Three parts wrapped in []:

  1. Expression — what to compute
  2. Item variable — the loop variable
  3. Iterable — where to loop over

Examples

# Double each number
doubled = [x * 2 for x in [1, 2, 3, 4]]
# → [2, 4, 6, 8]

# Uppercase strings
names = ["alice", "bob", "charlie"]
upper = [n.upper() for n in names]
# → ["ALICE", "BOB", "CHARLIE"]

# Convert types
nums = [int(s) for s in ["1", "2", "3"]]
# → [1, 2, 3]

⌨️ Walkthrough: Build It Yourself

Open a Python REPL and type along:

# Step 1: Basic squaring
squares = [x**2 for x in range(1, 6)]
squares  # → [1, 4, 9, 16, 25]
# Step 2: String transformations
words = ["hello", "world", "python"]
caps = [w.capitalize() for w in words]
caps  # → ['Hello', 'World', 'Python']
# Step 3: Data extraction from dicts
users = [{"name": "Alice", "age": 25},
         {"name": "Bob", "age": 30}]
ages = [u["age"] for u in users]
ages  # → [25, 30]

🔍 Conditional Comprehensions

Add an if clause after the for to filter items:

[expression for item in iterable if condition]

Even numbers only

evens = [x for x
         in range(20)
         if x % 2 == 0]
# → [0, 2, 4, ... 18]

Filter + transform

long = [w.upper()
        for w in words
        if len(w) > 4]
# words = ["cat","elephant","dog","giraffe"]
# → ["ELEPHANT", "GIRAFFE"]
Pitfall

Don't confuse filter (after for) with ternary (inside expression). [x if x>0 else 0 for x in ...] is a ternary — it transforms every item. [x for x in ... if x>0] is a filter — it drops non-matching items.

🔄 Multiple for Clauses

Use multiple for clauses (same order as nested loops):

Nested loop

pairs = []
for x in [1, 2]:
    for y in ["a", "b"]:
        pairs.append((x, y))
# → [(1,"a"), (1,"b"),
#     (2,"a"), (2,"b")]

Comprehension

pairs = [(x, y)
         for x in [1, 2]
         for y in ["a", "b"]]
# → [(1,"a"), (1,"b"),
#     (2,"a"), (2,"b")]

Rule of thumb: The leftmost for is the outermost loop. for clauses execute left to right, just like nested loops top to bottom.

⚡ Comprehensions vs. map / filter

Three ways to do the same thing — comprehension is almost always the right choice:

nums = [1, 2, 3, 4, 5, 6]

# ❶ Comprehension (✅ idiomatic)
result = [n * 2 for n in nums if n % 2 == 0]
# → [4, 8, 12]

# ❷ map + filter (❌ less readable)
result = list(map(lambda n: n * 2,
                filter(lambda n: n % 2 == 0, nums)))
# → [4, 8, 12]

# ❸ Manual loop (❌ verbose)
result = []
for n in nums:
    if n % 2 == 0:
        result.append(n * 2)
The Zen of Python

"There should be one — and preferably only one — obvious way to do it." That way is the comprehension.

🧰 Advanced Tips & Tricks

Flatten a matrix

matrix = [[1, 2], [3, 4], [5, 6]]
flat = [x for row in matrix
           for x in row]
# → [1, 2, 3, 4, 5, 6]

String of digits

digits = [int(c) for c
          in "49371"]
# → [4, 9, 3, 7, 1]

Readability rule

One line or break it. If a comprehension doesn't fit on ~80 chars, split it across lines:

result = [complex(x, y)
          for x in xs
          if x > 0
          for y in ys
          if y != 0]

Don't nest too deep

3+ for clauses? Use a regular loop instead — comprehensions over 80 chars hurt readability.

🏋️ Practice Exercises

Try each one yourself before checking the solution.

Exercise 1 — EASY

Create a list of squares of numbers 1–20 that are divisible by 5.

[x**2 for x in range(1, 21) if x % 5 == 0]
# → [25, 100, 225, 400]

Exercise 2 — MEDIUM

Given a list of words, create a list of (word, length) tuples for words longer than 3 characters.

words = ["cat", "apple", "dog", "elephant", "hi"]
result = [(w, len(w)) for w in words if len(w) > 3]
# → [('apple', 5), ('elephant', 8)]

Exercise 3 — HARD

Flatten a list of dictionaries of lists: [{"nums": [1,2]}, {"nums": [3,4,5]}] → [1, 2, 3, 4, 5]

data = [{"nums": [1, 2]}, {"nums": [3, 4, 5]}]
result = [n for d in data for n in d["nums"]]
# → [1, 2, 3, 4, 5]

📝 Key Takeaways

1. Syntax

[expression for item in iterable if condition] — the one pattern to remember.

2. Filter vs. Transform

Use if after for to filter items out. Use a ternary inside the expression to conditionally transform.

3. When to avoid

Don't use comprehensions for side effects (e.g., print) or when the logic needs 3+ for clauses. Use a regular loop instead.
