🌳 Trees

Traversals · DFS vs BFS · Binary Search Trees

Trees are the most common non-linear data structure in interviews. Master the three primary traversals — iterative and recursive — and you can solve 80% of tree problems. 15 minutes. Go.

Trees The Three Core Traversals

Inorder (Left → Root → Right)

BST inorder = sorted order. Most common interview traversal.

Preorder (Root → Left → Right)

Tree serialization, copy construction.

Postorder (Left → Right → Root)

Delete tree, evaluate expression trees.

class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

def inorder_recursive(root):
    # Base case
    if not root:
        return []
    # Recursive leap: left → visit → right
    return (inorder_recursive(root.left) +
            [root.val] +
            inorder_recursive(root.right))

Recursive is elegant; iterative (stack) handles deep trees.

The recursive pattern is identical for all three — only the position of the [root.val] append changes. For inorder it goes between the recursions. For preorder it goes before. For postorder it goes after. Memorize that one template and rotate the visit line.

Trees Iterative Inorder — Stack Pattern

Recursion-free traversal for production code and avoid stack overflow.

def inorder_iterative(root):
    res, stack = [], []
    curr = root
    # Push all left, pop & visit, go right
    while curr or stack:
        while curr:
            stack.append(curr)
            curr = curr.left     # Go as far left as possible
        curr = stack.pop()
        res.append(curr.val)     # Visit
        curr = curr.right        # Now check the right subtree
    return res

Pattern: Stack simulates the call stack. Outer loop runs while either pointer or stack is non-empty.

Preorder iterative: Visit on push, not on pop. Swap res.append to the inner while loop.

Postorder: Two-stack or one-stack + visited flag. Harder — practice it last.

The iterative inorder is the pattern you should be able to write from muscle memory. If you blank, remember: while curr or stack, push all left, pop & visit, go right. That's the whole algorithm.

Trees BFS — Level-Order Traversal

Queue-based. Process level by level — key for "right side view", "zigzag", "max depth per level".

from collections import deque

def level_order(root):
    if not root:
        return []
    res, q = [], deque([root])
    while q:
        level_size = len(q)
        level = []
        for _ in range(level_size):
            node = q.popleft()
            level.append(node.val)
            if node.left:  q.append(node.left)
            if node.right: q.append(node.right)
        res.append(level)
    return res

When to use BFS

  • Shortest path in unweighted tree
  • Level-based problems
  • Tree is wide (avoid deep recursion)

When to use DFS

  • Path existence / path sum
  • Serialize / deserialize
  • Tree is deep (prefer iterative stack)
BFS uses a deque — always import deque, not list as a queue. Popping from the front of a list is O(n). The level_size trick is essential so you know where each level boundary is.

📚 Stacks

Monotonic Stack · Expression Evaluation · Bracket Matching

Stacks are LIFO — last in, first out. But the power comes from monotonic patterns: increasing or decreasing stacks that answer "next greater element" in O(n).

Stacks Monotonic Stack — Next Greater Element

Classic O(n) pattern. Stack maintains elements in decreasing order. When we see a larger element, we pop and record.

def next_greater_elements(nums):
    n = len(nums)
    res = [-1] * n
    stack = []  # stores indices

    for i in range(n):
        # While current > top of stack,
        # current is the "next greater"
        while stack and nums[i] > nums[stack[-1]]:
            idx = stack.pop()
            res[idx] = nums[i]
        stack.append(i)

    return res

# nums = [2, 1, 5, 3, 4]
#   →  [5, 5, -1, 4, -1]

Monotonic stack variants:

  • Decreasing (shown) → next greater element
  • Increasing → previous greater / next smaller
  • Circular — iterate 2×n, index = i % n

Key insight: The stack stores indices, not values. You compare by nums[stack[-1]].

Practice problems:

  • Daily Temperatures (LeetCode 739)
  • Largest Rectangle in Histogram (84)
  • Trapping Rain Water (42)
The monotonic stack is one of the most elegant O(n) patterns. Most people reach for O(n²) brute-force. The trick is recognizing when "next bigger/smaller" relationships apply — temperature maps, stock spans, histogram areas.

Stacks Bracket Matching — Validation Pattern

Push opening brackets, pop and match on closing. Fail early on mismatch or empty stack.

def is_valid(s: str) -> bool:
    pairs = {')': '(', ']': '[', '}': '{'}
    stack = []
    for ch in s:
        if ch in pairs:          # closing bracket
            if not stack or stack.pop() != pairs[ch]:
                return False
        else:                     # opening bracket
            stack.append(ch)
    return not stack              # all must be matched

# Valid: (), {}[], ({[]})
# Invalid: (], ([)], )(

Variations: Minimum add to make valid, remove invalid brackets, longest valid parentheses.

This is the entry-level stack problem. Don't dismiss it — the same pattern extends to more complex parsers, HTML tag matching, and expression validation. Key: return not stack at the end, not True.

🔁 Recursion

Divide & Conquer · Backtracking · Memoization

Recursion is the mental model for trees, graphs, divide-and-conquer algorithms, and backtracking. If you can write a recursive function without a whiteboard, you're ahead of 70% of candidates.

Recursion The Three Laws

1. Base Case

The simplest input — return without recursing. Without this, infinite recursion.

# Fibonacci base
if n <= 1: return n

2. Recursive Leap

Assume the function works for smaller inputs. Call it with a reduced problem.

return fib(n-1) + fib(n-2)

3. Combine Results

Merge subproblem answers into the final answer.

# Tree height
1 + max(left_h, right_h)

Recursive leap of faith: Don't trace through all calls. Verify the base case, then trust that your recursive call returns the right answer for a smaller input.

The recursive leap of faith is the hardest concept for new programmers. Your brain wants to trace the stack. Don't. Verify the base case is correct, then trust the function. It's like induction — base case + inductive step = all cases.

Recursion Backtracking — Explore, Try, Undo

Template for permutation / combination / subset problems.

def subsets(nums):
    res = []

    def backtrack(start, path):
        res.append(path[:])     # snapshot of current path
        for i in range(start, len(nums)):
            path.append(nums[i])        # choose
            backtrack(i + 1, path)      # explore
            path.pop()                  # undo (backtrack)

    backtrack(0, [])
    return res

# nums = [1,2,3] → [[],[1],[1,2],[1,2,3],[1,3],[2],[2,3],[3]]

Variations: permutations (use visited set), combinations (k-sized), combination sum (with repeat).

Backtracking is DFS on a decision tree. The template is always: add candidate → recurse → remove candidate. The res.append(path[:]) — the slice copy — is the most common bug. Without [:] you append a reference that keeps mutating.

Recursion Memoization — Don't Recompute

Cache results of expensive recursive calls. Top-down DP in disguise.

def fib_memo(n, memo={}):
    if n in memo:
        return memo[n]
    if n <= 1:
        return n
    memo[n] = fib_memo(n-1, memo) + fib_memo(n-2, memo)
    return memo[n]

O(n) time vs O(2ⁿ) naive. The cache turns exponential into linear.

When to memoize:

  • Overlapping subproblems
  • Same inputs computed multiple times
  • Optimal substructure exists

Pattern: Pass memo as dict parameter. Check before compute. Store after compute.

Memoization is the bridge from recursion to dynamic programming. Start with recursive, add a cache. If the cache keys can be sorted into a table, you have a DP problem. Practice: climb stairs, coin change, edit distance.

🌳 Tree

Given BST, find the lowest common ancestor (LCA) of two nodes.

Hint: In a BST, LCA is the first node where the two target values split — one on left, one on right. 
def lca(root, p, q):
    while root:
        if p.val < root.val and q.val < root.val:
            root = root.left
        elif p.val > root.val and q.val > root.val:
            root = root.right
        else:
            return root

📚 Stack

Given array, return array where each element is the number of consecutive smaller elements before it (stock span).

Hint: Monotonic decreasing stack of (value, span). When you pop, accumulate span. 
def stock_span(prices):
    res, stack = [], []
    for p in prices:
        span = 1
        while stack and stack[-1][0] <= p:
            span += stack.pop()[1]
        stack.append((p, span))
        res.append(span)
    return res

🔁 Recursion

Generate all valid combinations of n pairs of parentheses.

Hint: Backtrack with open and close counters. Only add '(' if open < n, only add ')' if close < open. 
def generate_parentheses(n):
    res = []
    def backtrack(s, open_c, close_c):
        if len(s) == 2 * n:
            res.append(s)
            return
        if open_c < n:
            backtrack(s + '(', open_c + 1, close_c)
        if close_c < open_c:
            backtrack(s + ')', open_c, close_c + 1)
    backtrack('', 0, 0)
    return res
These three problems cover tree property exploitation, monotonic stack, and recursive backtracking — three core patterns. The goal is to solve each in under 10 minutes without hints. Track your times.
