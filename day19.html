The Skeleton of Every HTML Page

Every HTML document follows the same 5-element skeleton:

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>My Page</title>
  </head>
  <body>
    <h1>Hello, World!</h1>
    <p>This is my first page.</p>
  </body>
</html>

<!DOCTYPE html>

Tells the browser: "This is HTML5." Always the first line.

<html>

The root element. Everything lives inside. lang="en" helps accessibility.

<head>

Metadata: title, character encoding, styles, scripts. Not displayed.

<head> vs <body> — What Goes Where?

<head> — The Brain

Invisible to the viewer. Contains:

  • <title> — appears in the browser tab
  • <meta charset="UTF-8"> — character encoding
  • <meta name="description"> — for search engines
  • <link> — stylesheets

<body> — The Body

Visible to the viewer. Contains:

  • <h1>–<h6> — headings
  • <p> — paragraphs
  • <a> — links
  • <img> — images
  • <ul>, <ol> — lists

📌 Rule of thumb: If the user sees it on the page, it goes in <body>. If it's information about the page, it goes in <head>.

Headings & Paragraphs

HTML gives you six levels of heading and one type of paragraph. Use them to structure your content — this is called semantic markup.

<h1>Main Title</h1>
<h2>Section Title</h2>
<h3>Subsection</h3>
<h4>Detail</h4>
<p>A block of text.
Use it for paragraphs.</p>
<p>Another paragraph.
Browser adds spacing.</p>

Rules of thumb

  • Only one <h1> per page — the main title
  • Don't skip levels: h1 → h2 → h3
  • Headings are for structure, not size — CSS handles size later
  • <p> automatically adds margin above and below
  • Multiple spaces collapse into one (line breaks too)

Links — The Web's Superpower

The <a> tag (anchor) creates hyperlinks. The href attribute tells it where to go.

<a href="https://example.com">
  Visit Example
</a>

<a href="about.html">
  About Us
</a>

<a href="#section2">
  Jump to Section 2
</a>

<a href="mailto:hi@me.com">
  Email Me
</a>

Link destinations

  • Absolute: https://... — links to other sites
  • Relative: about.html — links within your site
  • Anchor: #id — jumps to another part of the same page
  • Email: mailto:address — opens the user's mail app

Open in new tab

Add target="_blank" and rel="noopener" for safety:

<a href="..."
   target="_blank"
   rel="noopener">
  Link
</a>

Images — <img>

The <img> tag is self-closing — no closing tag needed. It uses attributes to specify the image source and alternative text.

<img
  src="photo.jpg"
  alt="A description of the photo"
  width="600"
  height="400"
>

Sources can be relative or absolute URLs:

<img src="images/logo.png"
     alt="Company logo">

<img src="https://picsum.photos/200"
     alt="Random photo">

⚠️ Important rules

  • Always include alt — screen readers use it; search engines rank by it; it shows when the image doesn't load
  • width and height prevent layout shift while the image loads
  • Use descriptive alt text: "Golden Gate Bridge at sunset" not "image.jpg"
  • Decorative images can have alt="" (empty) — screen readers skip them

Attributes — How Tags Carry Extra Info

Attributes go inside the opening tag and follow name="value" syntax.

<a    href="page.html">Link</a>
       ↑          ↑
     name    "value"

<img   src="photo.jpg"
       alt="Photo">

<html  lang="en">

<div   id="header"
       class="container">

<p     style="color:red">Red text</p>

Common attributes

hrefLink destination (for <a>)
srcImage/file source (for <img>)
altAlternative text (for images)
idUnique identifier — one per page
classGrouping label — many elements can share it
langLanguage of content
styleInline CSS (use sparingly)

Attributes are key-value pairs: name="value". Always use quotes (double or single).

Ordered & Unordered Lists

Lists are everywhere on the web. Two types: <ul> (bullets) and <ol> (numbers).

<!-- Unordered: bullet points -->
<ul>
  <li>Apples</li>
  <li>Bananas</li>
  <li>Cherries</li>
</ul>

<!-- Ordered: numbered -->
<ol>
  <li>Mix flour and eggs</li>
  <li>Add milk</li>
  <li>Cook on low heat</li>
</ol>

Nesting lists

Put one list inside another for sub-items:

<ul>
  <li>Frontend
    <ul>
      <li>HTML</li>
      <li>CSS</li>
      <li>JS</li>
    </ul>
  </li>
  <li>Backend
    <ul>
      <li>Node.js</li>
      <li>Python</li>
    </ul>
  </li>
</ul>

Exercise 2: Hyperlink Hub

Create links.html — a simple directory of useful links organized into sections.

  • Use <h1> for the page title
  • Use <h2> for each category (e.g., "Search", "Social", "Learning")
  • Each category is an <ul> of 2-3 links
  • Add a table of contents at the top with anchor links (#search, #social, etc.)
  • Give each category section an id attribute
<h1>Useful Links</h1>
<ul>
  <li><a href="#search">Search</a></li>
  <li><a href="#learning">Learning</a></li>
</ul>
<h2 id="search">Search</h2>

🎯 Key Takeaways

1. Structure is everything

Every HTML page has the same skeleton: DOCTYPE → <html> → <head> + <body>. Learn it once, use it forever.

2. Tags describe meaning

<h1> means "this is the main heading." <a> means "this is a link." Choose the right tag for the job — that's semantic HTML.

3. Attributes customize tags

href, src, alt, id, class — these key-value pairs make tags do what you want. They live inside the opening tag.

4. Build to learn

Open your browser, create .html files, experiment. The best way to learn HTML is to write it and see what happens.
