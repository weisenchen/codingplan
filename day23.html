What Is Responsive Design?

Responsive design means your page responds to the user's screen size, orientation, and capabilities — without separate mobile/desktop sites.

✓ Fluid layouts

Elements resize relative to the viewport instead of using fixed pixel widths.

✓ Adaptive content

Navigation, images, and text rearrange or hide based on available space.

Think of it like water — it fills whatever container it's poured into.

Step 1 — The Viewport Meta Tag

Without this tag, mobile browsers render the page at a desktop width (usually 980px) and then zoom out — making everything tiny.

<meta name="viewport" content="width=device-width, initial-scale=1.0">

width=device-width

Sets the layout width to the device's physical screen width (e.g., 375px on an iPhone).

initial-scale=1.0

Prevents default zoom so 1 CSS pixel = 1 device pixel.

⚠ Every HTML project from now on should include this tag in <head>.

Step 2 — rem & em Units (Not px)

Fixed px values don't scale. rem and em are relative units that respect the user's font-size preferences.

rem — Root em

Relative to the root <html> font-size. Default is 16px in most browsers.

/* 2rem = 32px when root is 16px */
html { font-size: 16px; }
h1 { font-size: 2.5rem; }  /* 40px */
p  { font-size: 1rem; }    /* 16px */

em — Parent em

Relative to the parent element's font-size. Can compound — use with care.

.card {
  font-size: 1.25rem;
  padding: 1em;  /* 20px if parent is 1.25rem */
}

Rule of thumb: Use rem for font sizes & spacing. Use em for properties that should scale with the element's own font-size (padding, border-radius).

rem vs px — A Visual Comparison

If a user increases their browser's default font-size to 20px, rem-based layouts grow gracefully; px-based layouts stay tiny.

Fixed with px ❌

h1 { font-size: 32px; }
p  { font-size: 16px; }
.card {
  width: 960px;
  padding: 20px;
}

Breaks on small screens. Doesn't respect user preferences.

Relative with rem ✅

h1 { font-size: 2rem; }
p  { font-size: 1rem; }
.card {
  width: 90%;
  max-width: 60rem;
  padding: 1.25rem;
}

Scales everywhere. Respects accessibility settings.

Step 3 — Media Queries

A media query applies CSS only when a condition is true. Most common: check the viewport width.

/* Apply these rules ONLY on screens narrower than 768px */
@media (max-width: 768px) {
  .sidebar {
    display: none;          /* hide sidebar on small screens */
  }
  .main {
    width: 100%;             /* full width instead of 70% */
  }
}

max-width: 768px

Targets screens at or below 768px — "desktop-down" approach.

min-width: 769px

Targets screens at or above 769px — "mobile-first" approach (recommended).

Common Breakpoints

Breakpoints are the widths where your layout changes. Don't target specific devices — target content behavior.

📱

Small

< 640px — phones, portrait

Single column, stacked

📟

Medium

640px – 1024px — tablets, landscape phones

2 columns, sidebar appears

🖥️

Large

> 1024px — desktops

Multi-column, full layout

/* Mobile-first: start with base styles, then layer up */
/* Base (no query) = default phone styles */

@media (min-width: 640px) {
  .nav { display: flex; }         /* horizontal nav on tablet+ */
}
@media (min-width: 1024px) {
  .container { max-width: 70rem; } /* constrain width on desktop */
}

Step 4 — Mobile-First Design

Start with the narrowest layout as your base CSS, then add min-width queries to enhance for larger screens.

/* 1. BASE — mobile (no media query needed) */
.layout {
  display: grid;
  grid-template-columns: 1fr;
  gap: 1rem;
}

/* 2. TABLET — 640px+ */
@media (min-width: 640px) {
  .layout {
    grid-template-columns: 1fr 1fr;
  }
}

/* 3. DESKTOP — 1024px+ */
@media (min-width: 1024px) {
  .layout {
    grid-template-columns: 1fr 1fr 1fr;
  }
}

✅ Why? — It's easier to add things to a simple layout than to override a complex one. Phone users get the leanest experience; desktop users get richer layouts.

Walkthrough — Making a Page Responsive

Let's take a basic page and apply everything we've learned.

<!-- Step 1: viewport meta tag -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

/* Step 2: Use rem for sizing */
html { font-size: 100%; }  /* 1rem = 16px (user-respectful) */
.card { padding: 1.25rem; }

/* Step 3: Flexible containers */
.container { width: 100%; max-width: 70rem; margin: 0 auto; }

/* Step 4: Mobile-first media queries */
.card-grid {
  display: grid;
  grid-template-columns: 1fr;       /* phone: single column */
  gap: 1rem;
}
@media (min-width: 640px) {
  .card-grid { grid-template-columns: 1fr 1fr; }
}
@media (min-width: 1024px) {
  .card-grid { grid-template-columns: 1fr 1fr 1fr; }
}

Responsive Navigation Pattern

A common pattern: horizontal nav on desktop, hamburger or stacked on mobile.

HTML

<nav class="navbar">
  <a href="#" class="logo">MyApp</a>
  <ul class="nav-links">
    <li><a href="#">Home</a></li>
    <li><a href="#">About</a></li>
    <li><a href="#">Contact</a></li>
  </ul>
</nav>

CSS (Mobile-First)

/* Mobile: stacked, centered */
.nav-links {
  list-style: none;
  display: flex;
  flex-direction: column;
  gap: 0.5rem;
  text-align: center;
}
/* Desktop: horizontal row */
@media (min-width: 768px) {
  .navbar {
    display: flex;
    justify-content: space-between;
  }
  .nav-links {
    flex-direction: row;
    gap: 1.5rem;
  }
}

Debugging Responsive Layouts

DevTools is your best friend. Open it and toggle responsive mode.

🔍 Chrome DevTools

  1. F12 or right-click → Inspect
  2. Click the mobile icon (📱) in the top-left
  3. Drag the handles — see your layout change live
  4. Pick a device from the dropdown for exact dimensions

💡 Pro Tips

  • Use outline: 2px solid red on elements to see their boundaries
  • Use * { background: rgba(0,0,0,.05) } to visualize spacing
  • Check the Computed tab — it shows the final value after all units resolve

Resize your browser while the page is open — you'll see media queries fire in real time.

✏️ Practice Exercise

Exercise (20 min)

Build a responsive card grid:

  1. Create an HTML page with the <meta viewport> tag
  2. Add 6 cards (each with a title, short description, and a colored box)
  3. Use rem for all font sizes, padding, and gaps
  4. Make it mobile-first: 1 column on phone, 2 on tablet (≥640px), 3 on desktop (≥1024px)
  5. Add a responsive nav that is stacked on mobile, horizontal on desktop

💡 Hints

  • Use display: grid with grid-template-columns
  • Start with 1fr — no media query needed
  • Add min-width queries for larger grids
  • Test by resizing your browser

✅ Check yourself

  • At 375px width — 1 column, stacked nav
  • At 768px width — 2 columns, horizontal nav
  • At 1280px width — 3 columns
  • No px for sizing — only rem and %

🎯 Key Takeaways

📐

Viewport

Always include <meta viewport> — without it, responsive CSS doesn't work on phones.

📏

rem & em

Use rem for spacing and type. Use em for element-relative values. Avoid px for layout.

📱→🖥️

Mobile-first

Write base styles for the smallest screen, then layer enhancements with min-width media queries.

One sentence to remember:
"Start small, use relative units, and enhance with media queries."

Responsive Design

You build it for one screen — it works on all of them.
