🔍 Step 2: Search & Filter — Backend

Add a search route to app.py that filters bookmarks by title, URL, or tags.

# In app.py — add a search endpoint
@app.route('/search')
def search():
    q = request.args.get('q', '').strip()
    tag = request.args.get('tag', '').strip()
    
    query = Bookmark.query
    if q:
        query = query.filter(
            Bookmark.title.contains(q) |
            Bookmark.url.contains(q)
        )
    if tag:
        query = query.filter(Bookmark.tags.contains(tag))
        
    bookmarks = query.order_by(Bookmark.created_at.desc()).all()
    return render_template('index.html', bookmarks=bookmarks, q=q, tag=tag)

Design note: We use .contains() which maps to SQL LIKE '%term%'. For a production app you'd want full-text search (FTS5), but this is fine for a portfolio project.

Explain the search route. We pass q and tag back to the template so the search box retains its value and we can highlight active filters. Note: contains() is case-sensitive in SQLite by default for non-ASCII, which is fine for this project.

🔧 Step 2c: Tag Context Processor

Add a context processor to app.py so tag filters are available on every page.

# In app.py — after the route definitions
@app.context_processor
def inject_tags():
    """Collect all unique tags from every bookmark."""
    all_bookmarks = Bookmark.query.all()
    tags = set()
    for bm in all_bookmarks:
        if bm.tags:
            # Tags stored as comma-separated string
            for t in bm.tags.split(','):
                tags.add(t.strip())
    return dict(all_tags=sorted(tags))

# Also update the index route to pass search params
@app.route('/')
def index():
    bookmarks = Bookmark.query.order_by(
        Bookmark.created_at.desc()
    ).all()
    return render_template('index.html', bookmarks=bookmarks, q='')

Jinja tip: Context processors are great for injecting data (tags, user info, config) into every template without repeating it in every route.

Context processors avoid duplicating the tag-collection logic in every route. The tags are stored as a comma-separated string — we split and deduplicate with a set, then sort alphabetically. This is a common pattern for simple tag systems.
