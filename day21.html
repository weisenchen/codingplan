CSS Basics

Three ways to add style · The cascade · Selectors

Concept

Inline · Internal · External

🚫 Inline

<p style="color: red;">
  Hello
</p>

Hard to maintain. Avoid except for quick tests.

⚠️ Internal

<style>
p { color: blue; }
</style>

Good for single-page demos. Not reusable.

✅ External

<link rel="stylesheet"
      href="style.css">

Best practice. One file controls many pages.

Rule of thumb: always start with an external stylesheet. Keep styling separate from structure.

Concept

The Cascade & Specificity

CSS stands for Cascading Style Sheets. When multiple rules target the same element, the browser uses a priority system:

/* Which color wins? */
p           { color: red; }      /* specificity: 0,0,1 */
.intro      { color: blue; }     /* specificity: 0,1,0 — wins! */
#main       { color: green; }    /* specificity: 1,0,0 — wins all */

Priority order (low → high)

  1. Browser defaults
  2. External / internal stylesheets
  3. Class selectors (.class)
  4. ID selectors (#id)
  5. Inline style="" (highest)

Remember

More specific = more priority. The !important flag overrides everything — use it only as a last resort.

Walkthrough

Element · Class · ID Selectors

/* Element selector — targets ALL <p> */
p {
  font-size: 16px;
}

/* Class selector — targets class="highlight" */
.highlight {
  background: yellow;
}

/* ID selector — targets id="banner" (use ONCE) */
#banner {
  width: 100%;
  height: 300px;
}

When to use each

SelectorUse for
elementDefault styles (all paragraphs)
.classReusable component styles
#idOne-off unique elements

Pro tip: Prefer classes. IDs are hard to override and should be reserved for anchors and JavaScript hooks.

Colors & Typography

Named colors · Hex · rgb() · Fonts · Alignment

Concept

Colors in CSS

Three common ways to specify color:

/* 1. Named colors (140+ built-in) */
h1 { color: tomato; }

/* 2. Hex codes — #RRGGBB or #RGB */
h1 { color: #ff6347; }     /* tomato in hex */

/* 3. rgb() / rgba() — red, green, blue, alpha */
h1 { color: rgb(255, 99, 71); }  /* same color */
h1 { color: rgba(255, 99, 71, 0.8); } /* 80% opaque */
tomato #ff6347
#4ade80 green shade
#7c6ff0 purple accent

Walkthrough

Fonts, Size, Weight & Alignment

body {
  font-family: Arial, sans-serif;
}

h1 {
  font-size: 32px;
  font-weight: 700;   /* bold */
  text-align: center;
}

.lead {
  font-size: 18px;
  line-height: 1.6;  /* 1.6× the font size */
  color: #666;
}

Common font properties

PropertyValues
font-family"Georgia", serif; Arial, sans-serif
font-size16px, 1.2rem, 2em
font-weight400 (normal), 700 (bold)
text-alignleft, center, right
line-height1.4, 1.6, 2.0

Tip: Always list a fallback font (sans-serif or serif) after your chosen font.

The Box Model

Content · Padding · Border · Margin

Concept

Margins, Padding & Borders

Every HTML element is a rectangular box. From inside out: content → padding → border → margin.

.card {
  width: 300px;
  padding: 20px;              /* space INSIDE the border */
  border: 2px solid #333;   /* visible edge */
  margin: 16px;               /* space OUTSIDE the border */
  background: #f5f5f5;
}

Shorthand rules

/* All sides */
margin: 10px;

/* top/bottom left/right */
padding: 10px 20px;

/* top right bottom left */
margin: 10px 5px 15px 5px;

Individual sides

margin-top: 10px;
margin-right: 20px;
padding-left: 15px;
padding-bottom: 5px;

Concept

CSS Units — px, em, rem, %

Different units for different jobs:

/* Absolute — fixed size, predictable */
h1 { font-size: 32px; }
.box { width: 400px; }

/* Relative — scales with parent or root */
.child { width: 50%; }       /* half of parent */
.text { font-size: 1.5em; }    /* 1.5× parent font-size */
.text { font-size: 1.5rem; }   /* 1.5× root font-size */

px

Pixels. Fixed size. Best for borders and small precise values.

em

Relative to parent's font-size. Compounds when nested.

rem

Relative to root (<html>) font-size. Predictable.

Practice Time

Two exercises to cement today's concepts

Exercise 1

Style a Profile Card

Create an HTML page with a profile card and style it:

<!-- Starter HTML -->
<div class="profile-card">
  <img src="avatar.png" alt="Avatar">
  <h2 class="name">Alex Chen</h2>
  <p class="bio">Full-stack developer</p>
  <p class="location">📍 Toronto, ON</p>
</div>

Requirements

  1. The card has a 2px solid border and rounded corners (border-radius: 12px)
  2. The name is 24px, bold, centered
  3. The bio is 16px, gray color (#666)
  4. Add 20px padding inside the card and 16px margin outside
  5. Center the image with text-align: center

💡 Use an external style block or inline — try both to see which is easier to maintain!

Exercise 2

Build a Color Palette Grid

Create a 3×2 grid of colored boxes, each labeled with its hex code:

<!-- Starter HTML -->
<div class="palette">
  <div class="swatch" style="background:#ff6b6b">#ff6b6b</div>
  <div class="swatch" style="background:#4ecdc4">#4ecdc4</div>
  <div class="swatch" style="background:#45b7d1">#45b7d1</div>
  <div class="swatch" style="background:#96ceb4">#96ceb4</div>
  <div class="swatch" style="background:#feca57">#feca57</div>
  <div class="swatch" style="background:#ff9ff3">#ff9ff3</div>
</div>

Requirements

  1. Use CSS Grid or Flexbox to arrange swatches in 3 columns
  2. Each swatch is at least 100px tall
  3. Text is white, centered, with a subtle text-shadow for readability
  4. The .palette has 16px padding and 8px gap between swatches

💡 display: grid; grid-template-columns: repeat(3, 1fr); gap: 8px;

Summary

Key Takeaways

1
3 ways to add CSS: inline (avoid), internal (quick tests), external (best practice). Keep styles separate from HTML.
2
Selectors & specificity: element → class → ID. More specific selectors override less specific ones.
3
Box model: content → padding → border → margin. Master this and you understand CSS layout.
4
Units: px for fixed sizes, % for fluid layouts, rem for accessible typography.
5
Colors: named, hex, and rgb(). Pick a palette early and stick to it.
