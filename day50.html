⥮ Array Reversal

The simplest foundation for two-pointer thinking

Slide 5

Reverse: Built-in vs In-Place

Python offers a convenient one-liner, but interviews want the in-place version:

Slicing (creates copy)

def reverse_copy(arr):
    return arr[::-1]
# O(n) time, O(n) space
# New list — original unchanged

In-Place (two-pointer)

def reverse_inplace(arr):
    l, r = 0, len(arr) - 1
    while l < r:
        arr[l], arr[r] = arr[r], arr[l]
        l += 1
        r -= 1
# O(n) time, O(1) space
# Mutates original — preferred in interviews
💡 Key insight: The in-place reverse is the simplest two-pointer pattern. Pointers start at both ends and move toward each other, swapping elements.
Slide 6

Walkthrough: Reverse In-Place

Trace through reverse_inplace([1, 3, 5, 7, 9]) step by step:

# Initial:  [1, 3, 5, 7, 9]   l=0, r=4
# Step 1:   [9, 3, 5, 7, 1]   l=1, r=3   ← swap 1↔9
# Step 2:   [9, 7, 5, 3, 1]   l=2, r=2   ← swap 3↔7, l ≥ r → stop

def reverse_inplace(arr):
    l, r = 0, len(arr) - 1
    while l < r:
        arr[l], arr[r] = arr[r], arr[l]  # swap
        l += 1
        r -= 1
    return arr

# Try it: reverse_inplace([10, 20, 30, 40]) → [40, 30, 20, 10]

Complexity: O(n) time, O(1) space — optimal. Each element is visited exactly once.

⇄ Two-Pointer Technique

Same-direction or opposite — two pointers unlock O(n) solutions

Slide 8

The Two-Pointer Pattern

Two pointers iterate through the array, often at different speeds or from different ends:

Opposite Direction

# One from left, one from right
# Used for: reverse, palindrome check,
# two-sum on sorted array
l, r = 0, len(arr)-1
while l < r:
    # ... compare / swap / sum ...
    l += 1
    r -= 1

Same Direction (fast/slow)

# Both start at left, different speeds
# Used for: remove duplicates,
# linked-list cycle detection
slow = 0
for fast in range(1, len(arr)):
    # ... conditionally advance slow ...
    if arr[fast] != arr[slow]:
        slow += 1
        arr[slow] = arr[fast]
🎯 Rule of thumb: Two-pointer works when the array is sorted or when you need to compare elements at different positions in a single pass.
Slide 9

Walkthrough: Two Sum (LC Easy #1)

Problem: Given a sorted array and target, find two indices whose values sum to target.

def two_sum(nums, target):
    l, r = 0, len(nums) - 1
    while l < r:
        curr = nums[l] + nums[r]
        if curr == target:
            return [l, r]          # found it
        elif curr < target:
            l += 1                 # need bigger sum
        else:
            r -= 1                 # need smaller sum
    return []                      # no solution

# Example: nums=[2, 7, 11, 15], target=9
#   l=0(2), r=3(15) → curr=17 > 9 → r=2
#   l=0(2), r=2(11) → curr=13 > 9 → r=1
#   l=0(2), r=1(7)  → curr=9  → return [0, 1] ✓

Complexity: O(n) time, O(1) space.

⚡ Unsorted input? Use a hash map instead (also O(n) but O(n) space). Two-pointer works only on sorted arrays.

Walkthrough: Remove Duplicates (LC Easy #26)

Problem: Remove duplicates from a sorted array in-place, return new length.

def remove_duplicates(nums):
    if not nums:
        return 0
                                    # slow = last unique position
    slow = 0                       # ← slow pointer
    for fast in range(1, len(nums)):  # ← fast pointer scans ahead
        if nums[fast] != nums[slow]:
            slow += 1
            nums[slow] = nums[fast]  # overwrite next unique position
    return slow + 1                # length = last index + 1

# Example: nums=[0, 0, 1, 1, 1, 2, 2, 3, 3, 4]
#   After:  [0, 1, 2, 3, 4, ...] → return 5
#   The array is modified in-place; elements beyond index 4 don't matter.

Complexity: O(n) time, O(1) space. Fast pointer visits every element exactly once.

⊞ Sliding Window

Expand, contract, track — O(n²) becomes O(n)

Slide 12

The Sliding Window Pattern

Maintain a contiguous subarray (the "window") and slide it through the array:

Fixed Window

# Window of size k slides right
# Each step: remove left, add right
window_sum = sum(nums[:k])
max_sum = window_sum
for i in range(k, len(nums)):
    window_sum += nums[i]
    window_sum -= nums[i - k]
    max_sum = max(max_sum, window_sum)

Variable Window

# Window grows/shrinks based on
# a condition (e.g., sum ≤ target)
l = 0; curr = 0; best = 0
for r in range(len(nums)):
    curr += nums[r]                # expand right
    while curr > target:          # contract left
        curr -= nums[l]
        l += 1
    best = max(best, r - l + 1)
💡 Key insight: Each element enters the window once and leaves once → O(n) time instead of a naive O(n²) nested loop.
Slide 13

Walkthrough: Maximum Subarray (Kadane's Algorithm)

Problem: Find the contiguous subarray with the largest sum. A classic sliding-window / DP hybrid.

def max_subarray(nums):
    max_ending_here = max_so_far = nums[0]
    for i in range(1, len(nums)):
        # Either extend the window or start fresh
        max_ending_here = max(nums[i], max_ending_here + nums[i])
        max_so_far = max(max_so_far, max_ending_here)
    return max_so_far

# Example: nums = [-2, 1, -3, 4, -1, 2, 1, -5, 4]
#   i=0: mah=-2, msf=-2
#   i=1: mah=max(1, -2+1)=1,      msf=max(-2,1)=1
#   i=2: mah=max(-3, 1-3)=-2,     msf=max(1,-2)=1
#   i=3: mah=max(4, -2+4)=4,      msf=max(1,4)=4
#   i=4: mah=max(-1, 4-1)=3,      msf=max(4,3)=4
#   i=5: mah=max(2, 3+2)=5,       msf=max(4,5)=5
#   i=6: mah=max(1, 5+1)=6,       msf=max(5,6)=6  ← answer: [4,-1,2,1]

Complexity: O(n) time, O(1) space. One pass, no extra memory.

Key Takeaways

⥮ Array Reversal

Two pointers from opposite ends, swapping until they meet. O(n) time, O(1) space. Foundation for the two-pointer pattern.

⇄ Two-Pointer

Opposite or same direction. Use on sorted arrays or when comparing elements at different positions in one pass.

⊞ Sliding Window

Expand right, contract left. Each element enters/exits once → O(n). Fixed-size or variable-size depending on constraints.

📊 Kadane's Algorithm

Track max ending here and max so far. Classic maximum subarray problem. O(n) time, O(1) space.
